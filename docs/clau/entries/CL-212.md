# CL-212 — ci.yml lint rule mis-specified in Step 11

=== WHAT WAS WRONG ===
Original VT-3.4 PR 2/3 brief Step 11 specified the ci.yml lint rule as: 'fail CI if any apps/team-orchestrator Python file imports from langchain.agents WITHOUT also importing from langgraph (per CL-179 lesson — create_agent migration).'

This rule would red-fail CI on the repo's own valid code. orchestrator_agent.py and sales_recovery_stub.py both legitimately use `from langchain.agents import create_agent` and neither imports langgraph (nor needs to). The post-CL-179-migration import IS `from langchain.agents import create_agent` — that's the correct pattern, not the violation pattern.

=== WHAT THE RULE SHOULD BE ===
The class-of-bug to prevent is regression TO deprecated langgraph.prebuilt agent constructors after the migration. CL-134 surfaced create_react_agent as deprecated; CL-179 corrected create_agent's signature. The lint guards against re-introduction of the deprecated symbols, not against the legitimate post-migration imports.

Corrected rule: Fail CI if any Python file under apps/team-orchestrator/ imports create_react_agent (or any other deprecated langgraph.prebuilt agent constructor) from langgraph.prebuilt. Grep target:
   from langgraph.prebuilt import.*create_react_agent
plus any other langgraph.prebuilt agent constructors confirmed deprecated.

=== HOW IT WAS CAUGHT ===
Claude Code Step 11 pre-write check. Recognized that the literal brief spec would break the repo's own valid code, declined to commit a CI rule that fails legitimately-correct imports. Surfaced for Clau confirmation before writing.

=== ROOT CAUSE ===
Clau conflated 'must import langgraph' (wrong test) with 'must not import the deprecated thing' (right test) when drafting the original brief Step 11. Sloppy phrasing of intent. The CL-179 link in the brief annotation was correct in spirit (the rule exists because of the migration), but the rule body inverted the polarity.

=== STANDING IMPLICATION ===
None new. The general 'verify brief lint rules against actual repo code' check is a sub-case of CL-194 (ground-truth verification of repo-state assumptions). Folded.

=== STATE AT TIME OF WRITE ===
ci.yml lint rule deferred to post-Landmine-2 session along with Step 11(a) thin wrapper + Step 12 PR open. Corrected rule body specified in CL-211 follow-up paste-block.
