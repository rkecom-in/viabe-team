# VT-295 — Ops Console — Assignment (VTAdmin)

## Why
VTAdmin controls which VTR is responsible for which businesses — the assignment that
every other page's RLS scoping reads from.

## What
- VTAdmin-only page: all-business view; assign / reassign businesses to VTRs inline.
- The assignment is the source of truth for VTR→tenant RLS scoping across the console.
- Deep view → overlay.

## Acceptance criteria
- Only VTAdmin can access/mutate assignments (role gate; fail-closed).
- Reassigning a business immediately rescopes that VTR's visibility (RLS canary: VTR's
  assigned set changes; real-PG).
- Every assignment change audit-logged (Pillar 7).

## Out of scope
- The per-page listings that consume the assignment (VT-291..294, VT-296).

## Notes
Build deferred behind launch queue. Foundational for multi-VTR scoping but VTAdmin-only,
so sequenced after the VTR core pages. One PR (CL-424).
