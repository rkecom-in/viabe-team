# VT-562 — per-run campaign outcome report to the owner

## Problem (audit finding, PARTIAL-confirmed core)
On owner approval `runner.py:565` calls `resume_run()` and DISCARDS its terminal state — which carries `campaign_execution_summary` {sent, skipped_opt_out, skipped_complaint_freeze, failed, killed}. `supervisor.py:483` edges `campaign_execute → END` with no owner-report node; `compose_owner_output` never runs on the resume path. Real customer sends happen and the owner is never told what happened. Vision: "reports outcome — end to end", "EVERY outcome reaches the owner".

## Fix
Capture `resume_run`'s terminal state on the resume/close path, compose an honest outcome message from `campaign_execution_summary` (sent/skipped/failed counts, no inflated claims), send to the owner via the existing guarded owner-send seam (within-24h window — the owner just replied; goes through the dev send-guard chokepoint; recorded in `owner_message_audit`). Send failure is fail-soft for run close but alerts. Flagship campaign path in scope; the l2_send agent-surface path only if a clean seam exists (else flag honestly).
