# CL-142 — Revised dep-audit migration sequence

That's wrong ordering — every PR in steps 1, 2, 4 needs to know whether to apply `==` or `>=` to its new/changed deps. Doing it third means steps 1-2 ship under whatever convention happens to be in flight, then step 3 normalizes them, then step 4 has to relitigate.

1. sarvam-ai-client → sarvamai rename (HIGH).
2. langgraph create_react_agent → langchain.agents.create_agent (HIGH).
3. tailwind-merge 2 → 3 (HIGH per CL-141 correction).
