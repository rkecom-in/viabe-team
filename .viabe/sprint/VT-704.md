## What shipped (V1)

New per-tenant Ops Console page `/team/ops/tenants/[tenantId]/activity-flow` — the e2e-sim
flow-diagram visual language as a product surface. One time-ordered, day-grouped story of the
last 30 days:

- **Chat lane**: owner ↔ Manager bubbles (conversation_log).
- **Manager decision cards**: summary + decided/did/result from tm_audit_log (the VT-690 spine).
- **Sub-agent cards**: manager_tasks dispatches (objective, assigned_function, outcome).
- **Execution errors**: pipeline_steps rows with error set (healthy steps = volume, not signal).
- **Approvals**: ask + resolution as separate timeline moments.
- **Comms**: delivered/dropped owner_comms_queue items (dropped reason surfaced).
- **Concerns**: incidents + tenant_alerts, severity-striped.

Data layer `lib/ops/activity-flow.ts` (fetchTenantFlow + groupByDay): serverSecretClient reads
with the VT-290 app-side scoping pattern — fail-closed on empty assigned set + canAccessTenant
inside the fetch AND at the page; per-source caps surfaced in the header (truncation is never
silent). Page guarded per-page (requireOpsOperator → login redirect; deny render when
unassigned). Cross-linked from the tenant dashboard.
