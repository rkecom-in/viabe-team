# CL-141 — tailwind-merge should be HIGH, not MEDIUM

Context7 verified: tailwind-merge v3 explicitly dropped Tailwind CSS v3 support; v2.6.0 IS the Tailwind v3 compatibility line; v3.x supports Tailwind 4.0–4.2 only. Quote from /dcastil/tailwind-merge docs: "Support for Tailwind CSS v3 has been dropped in favor of support for Tailwind CSS v4." Paired guidance: "For projects utilizing Tailwind v3, users should rely on version 2.6.0 of tailwind-merge."

apps/team-web is on Tailwind v4 + tailwind-merge ^2.6.0. That's the documented Tailwind-v3 compat library running against a Tailwind-v4 codebase. Every twMerge()/cn() call is potentially producing wrong class precedence right now — not a future risk, a present silent bug. Severity in dep-audit-2026-05-18.md should move MEDIUM → HIGH.

Migration: bump tailwind-merge to ^3.x, smoke-test class merging across shadcn/ui surfaces (Button, Card, Dialog primitives are highest-fan-in). One-line dep bump + visual diff sweep.
