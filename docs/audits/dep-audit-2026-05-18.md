# Phase 1 Dependency Audit — 2026-05-18

VT-Dep-Audit recon (CL-137). Recon only — **no version bumps, no removals, no
code changes** in the PR that carries this doc. Migration PRs follow review.

## Methodology

- **Inventory**: every `pyproject.toml` / `package.json` in the repo (6 files; `node_modules` excluded).
- **Usage**: each manifest dep checked against actual `import` statements under each app's `src/`.
- **Context7**: `resolve-library-id` run for the 12 material Python runtime libraries; deep `query-docs` run for the 8 highest-signal Python runtime deps (langgraph, dbos, fastapi, pydantic, supabase-py, psycopg, anthropic-sdk-python, twilio-python).
- **Transparency note**: per the Context7 budget guard, the remaining Python deps and **all JS deps were assessed by version knowledge + import-grep, not a separate Context7 deep-query**. JS findings (recharts, tailwind-merge, lucide-react) are flagged MEDIUM on a knowledge basis and should be confirmed against the npm registry during migration-PR planning. No npm/PyPI `outdated` run was performed (no lockfile-install in a doc-only PR).
- The langgraph `create_react_agent` deprecation is confirmed by the **live package** (`LangGraphDeprecatedSinceV10` runtime warning observed during VT-3.9), not just Context7 — Context7's indexed docs still show the old import as current.

### Recommended migration sequence

1. **`sarvam-ai-client` → `sarvamai`** (HIGH, CL-92) — rename in `apps/team-ingestion-worker/pyproject.toml`. It is wrong code on disk and blocks the worker's `uv.lock`. Trivial, do first.
2. **`langgraph` `create_react_agent`** (HIGH, CL-134) — migrate `langgraph.prebuilt.create_react_agent` → `langchain.agents.create_agent` **before** any langgraph V2 bump. Needs an API-shape check (`create_agent` kwargs differ) + a new `langchain` dependency. Schedule as its own PR.
3. **Python pin discipline** (MEDIUM) — decide one policy: `==` everywhere (Type-2 governed) or `>=` minimums. Currently only the langgraph/langchain trio is `==`-pinned; everything else is bare or `>=`.
4. **Frontend majors** (MEDIUM) — `recharts` 2→3 and `tailwind-merge` 2→3 (align with Tailwind v4). Bundle into one frontend-deps PR; both are behind-a-major.
5. **UNUSED cleanup** (housekeeping) — separate PR: drop the 8 unused deps (or, for `langgraph-supervisor`, leave it — VT-3.4 will use it imminently).

**langgraph — `==1.2.0` — HIGH (CL-134)**

- **Action**: migrate to `langchain.agents.create_agent` in a dedicated PR before any langgraph V2 bump. `create_agent`'s kwarg shape differs — verify `model` / `prompt` / `name` mapping. Adds a `langchain` dependency.

- At: `apps/team-orchestrator/pyproject.toml:6-21`.
- **Action**: pick one policy repo-wide. Recon only — no change here.

**supabase / pgvector / anthropic — UNUSED** (see UNUSED appendix).
