# VT-324 — enforce the no-direct-tenant-db-access gate + vestigial-param cleanup

ROW 3 of Fazal's build-both grant (after VT-306 #320 merged). The wrapper migration
(VT-306) shrank the `no-direct-tenant-db-access` allowlist to the 11 structural
residuals (gate EXIT 0). This row makes the gate an ENFORCED blocking check and
folds in the vestigial-param cleanup VT-306 flagged.

## Scope
1. **Wire the gate into the pre-push hook** (`scripts/git-hooks/pre-push`) so it
   blocks pushes. Post-VT-245 CI status checks don't gate merges — the pre-push
   hook is the real safety gate; the gate was CI-only before. The script already
   `return 1`s on a violation; this makes it enforced where it matters.
2. **Drop the stale "report/allowlist mode" labels** (gate docstring + CI step
   name + CI job comment) — it's now enforced against the structural residual.
3. **Vestigial-param cleanup** (Cowork: fold in, no fresh row): `pool` on
   customer_registry / get_recent_campaigns / get_attribution_data (+ the discarded
   get_pool checkout), `cur` on send_whatsapp_template/message — retained for caller
   stability during VT-306, now removable.

## Done when
- Gate runs in the pre-push hook + aborts the push on a synthetic violation.
- Labels reflect enforced state.
- Vestigial params dropped, callers + tests updated, full suite green.
- Cowork review → Pillar-7.
