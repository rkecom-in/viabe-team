# CL-122 — PR-fix-7 architecture locked on option (C)

Fazal locked option (C) without prior spike, on Claude Code's code-level evidence. Rationale: Claude Code has read the actual code (graph.py, _app_db.py:273, migration 000b verbatim, the 14-site inventory, DBOS schema management). That's a stronger foundation than a synthetic spike of SET ROLE + pool reuse. If (C) breaks at PR review, fallback to (A) is well-documented in the writeup and the throwaway cost is the wrapper module + 1 migration + 12 mechanical edits — not the spike's stated 200 LOC, because (A)'s 12 site edits would mostly be identical, only the wrapper's internals and graph.py plumbing diverge.

ARCHITECTURE LOCKED (CL-120 superseded by this):
- Single privileged pool (unchanged from today).
- New module `orchestrator/db/tenant_connection.py` with context manager primitive.
- `tenant_connection(tenant_id)` does: SET ROLE app_role → set GUC `app.current_tenant` (NOT `app.current_tenant_id`) → yield → finally: clear GUC + RESET ROLE.
- One migration creates `app_role` (NOLOGIN, granted DML on all tenant-scoped tables + rate_limit_buckets, NO BYPASSRLS) + GRANT app_role TO <existing privileged role>. No `app_user`, no DATABASE_URL swap.
- 12 tenant-scoped sites migrate to use `tenant_connection`. Sites 13 (_lookup_tenant) and 14 (_within_rate_limits) stay on direct `get_pool()` — they are the intended service-role path per 000b.
- DBOS and LangGraph PostgresSaver untouched.
- New CI test: one tenant writer under `SET ROLE app_role` asserts a cross-tenant write fails. The 'one forcing test' the writeup says doesn't exist today. This is the real isolation proof.
- Decorator deferred. @DBOS.step arg-serialization breaks connection-injecting decorators (writeup §5). Context manager primitive only in this PR. If a decorator pattern emerges later (around @DBOS.step boundary, not inside it), separate decision.

OUT OF SCOPE (re-confirmed):
- JWT/COALESCE policy variants (CL-88).
- B1/B2 auth-to-tenant mapping (Fazal owns; gates first dashboard schema).
- VT-3.3d RLS attack tests (separate PR).
- SECURITY DEFINER functions (option B; not adopted).
- Two-pool plumbing in graph.py (option A; not adopted).
