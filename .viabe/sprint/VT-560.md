# VT-560 — VT-557 remediation

## Problems (audit findings; both CC-shipped this week)
1. **Retry ladder inert:** `reap_stalled_manager_tasks` candidate query only matches planned/running/verifying; the RETRY branch flips a stalled task to `'blocked'`+`next_retry_at`, but `blocked` is never re-swept and `next_retry_at` never re-read ⇒ zero auto-retries, `dead_letter` unreachable via the ladder (only human redrive works).
2. **Boot-only sweeps:** `reap_orphan_runs`, `reap_stalled_manager_tasks`, `detect_silent_terminal_runs` run once in the FastAPI lifespan (`main.py:~186`), never as `@DBOS.scheduled` — a long-lived process never re-sweeps.

## Fix
(1) Include due-blocked tasks (`status='blocked' AND next_retry_at <= now()`) in the sweep: flip back to runnable with attempt/backoff via `decide_retry`, reaching `dead_letter` at max_attempts. (2) Register all three as `@DBOS.scheduled` sweeps in `scheduled_triggers.py` (cadence matching existing patterns), keep the boot invocation.
