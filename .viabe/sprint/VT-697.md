## What

1. **Typing indicator** (`twilio_send.send_typing_indicator`): Twilio v3 Indicators endpoint
   (Public Beta) — POST /v3/Indicators/Typing.json referencing the INBOUND MessageSid. Marks
   the owner's message READ (blue ticks) + shows "typing…" until our reply lands or 25s.
   Fired at `twilio_ingress` for owner inbounds AND the signup branch, on a fire-and-forget
   daemon thread: zero hot-path latency, fail-soft everywhere (missing creds / mock mode /
   HTTP error → log only). **Canary-proved 2026-07-23 (`canaries/vt697_typing_indicator_canary.py`,
   real 200 {"success":true} against Fazal's live inbound): `channel` MUST be uppercase
   "WHATSAPP" — lowercase 400s.**
2. **Deterministic TAP fast-path** (`journey._handle_reply_with_turn_brain` entry): a
   quick-reply tap echoes the suggestion text VERBATIM; when the body exactly matches a
   suggestion of the presented GAP question, record + advance + present the next question
   deterministically — no LLM on the hot path (sub-second vs the ~4-8s compose_turn round
   trip). Exact-match only (Pillar 1: no fuzzy interpretation); typed free text, corrections,
   Skip taps, and ALL confirm questions keep the full brain. Card-priority parity kept (a
   pending GST card outranks the queue head on this path too). The fast-path result carries
   the FULL next question so its suggestion buttons / card formatting ride the normal send.
