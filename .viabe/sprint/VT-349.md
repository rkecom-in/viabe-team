# VT-349 — 3 owner acks: template → free-form in-window reply

Fazal ruling 2026-06-06: a WhatsApp template is only required to message OUTSIDE the 24h
customer-service window. Three owner-facing messages are DIRECT REPLIES to a message the owner
just sent → in-window → they send as FREE-FORM session messages (`send_freeform_message`/VT-44),
not Meta templates. This dissolves the VT-348 all-variable-body Meta-reject problem entirely.

## Shipped
- `owner_surface/freeform_acks.py` — bilingual copy (`ACK_COPY`, EN+HI, Fazal-approved),
  `resolve_owner_locale` (COALESCE(preferred_language, language_preference, 'en')), and
  `send_freeform_ack` (best-effort + fail-safe: a window-closed 63016 or any send error is
  swallowed — the owner-action already landed and must not be unwound).
- 3 send sites rewired: `edge_cases_router._send_edge_ack` (handler text, already locale-aware),
  `support_bot._send_handoff_ack` (bilingual + run ref), `refund_executor._notify_owner` SPLIT
  (refund_processing = free-form in-window; **refund_completed STAYS a template** — out-of-window,
  up to 5 business days; ₹ amount via `money_inr`, Indian-grouped).
- Removed `team_edge_case_ack`, `support_handoff`, `refund_processing` from
  `config/twilio_templates.yaml` (no longer templates).

## Verified
canary: ack_body bilingual+fallback, send_freeform_ack 63016/error-swallow + no-phone, the 3
sites send free-form, refund split (processing free-form / completed template), resolve_owner_locale
