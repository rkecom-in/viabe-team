# CL-69 — Post-VT-3.3a validation plan

Fazal requested post-VT-3.3 validation of cumulative implementation. Recommended approach: 3-layer validation, sequenced.

## Layer 1 — Fresh-session architectural audit (DO AFTER VT-3.3a SHIPS)
Spin up a separate Clau session with no execution context. Inputs:
- Resurrection file v2 (update first if not already current)
- Clau_Session_Log Session Start (Standing + Open) view
- The actual codebase at current main HEAD
- A specific audit prompt covering: Pillar adherence, Notion-vs-shipped drift, scope-creep instances, architectural inconsistencies, schema-vs-spec drift, env-var hygiene, RLS coverage, CI gate coverage, test coverage gaps.

Fresh session reads with no prior context, surfaces issues that current-session-Clau is blind to. Outputs a structured findings report. Each finding becomes either: (a) immediate fix in next PR, (b) logged tech debt, (c) accepted trade-off with reasoning.

This is the highest-leverage validation we can do BEFORE Tier 3 dev testing exists (which requires VT-3.4 to be useful).

## Layer 2 — Live execution via Tier 3 dev testing (post-VT-3.4)
Already captured in earlier dev testing decision. ngrok + Twilio sandbox + real WhatsApp messages → watch full pipeline. Catches integration bugs and real-world surprises.

## Layer 3 — Third-party AI code review (CodeRabbit or similar)
Run over cumulative codebase (or last N PRs). Specific issue-level findings on security, common bug patterns. Noisy; needs filtering. Lower priority than Layers 1-2.

## Sequence
1. VT-3.3a ships (orchestrator FastAPI server + signature verification + DBOS ingress + migrations + synthetic webhook fixtures)
2. Update resurrection file v2 to reflect current main state
3. Run Layer 1 audit (fresh-session Clau)
4. Address findings (fix immediately or log as tech debt)
5. VT-3.3b ships (team-web route + tenant lookup + rate limiting)
6. VT-3.4 ships (langgraph_supervisor wiring)
7. Set up Layer 2 (ngrok + Twilio sandbox)
8. Consider Layer 3 (CodeRabbit) closer to soft launch

Layer 1 audit takes ~30-60 min of fresh-session Clau work + my time to act on findings. Cost: 1 session. Benefit: catches drift we are currently blind to.
