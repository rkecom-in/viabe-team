# CL-265 — DECISION: VT-50 tool return type conforms to VT-36's lean on-main SelfEvaluator 

VT-50 pre-flight reconciled the tool's output schema against the SelfEvaluator Protocol VT-36 shipped on main (PR #35, apps/team-orchestrator/src/orchestrator/agent/self_evaluate.py). The merged Protocol is deliberately lean: SelfEvaluator.evaluate(draft, criteria) -> SelfEvaluateVerdict; SelfEvaluateVerdict = outcome (SelfEvaluateOutcome enum PASS|REVISE) + feedback (SelfEvaluateFeedback|None); SelfEvaluateFeedback = four fields schema/pillar/consistency/legal, each str|None. The 2026-05-04 VT-50 page specified a much richer return: per-category CategoryFeedback objects (verdict + issues list[str<=200] + severity), overall_severity enum, and model_used/tokens_used/cost_paise telemetry. DECISION per VT-50 brief item 2 (Protocol on main wins): VT-50's tool RETURN TYPE conforms to the lean on-main Protocol — the richer page schema is NOT the contract and is cut from the return type. Telemetry (tokens_used, cost_paise, model_used) may still be logged internally by the tool but does not ride on the Protocol return. The 2026-05-04 page schema is superseded by the as-built VT-36 Protocol.
