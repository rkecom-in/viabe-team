# CL-197 — Brief-drafting must mark repo-state assumptions

STANDING DISCIPLINE arising from CL-193 (brief drafted on captured-prose foundations).

For every brief Clau drafts: identify what Clau cannot directly verify, then choose ONE explicitly:

(a) Fazal pastes the relevant ground-truth (file contents, table existence, recent merge SHAs, branch state) into the conversation before brief ships. Default when the unknown is small + bounded.

(b) Brief instructs Claude Code 'first action: verify X, Y, Z assumptions against current repo; stop and report if any diverge.' Default when the unknowns are many but the brief structure is sound conditional on them.

(c) Brief is rescoped to not depend on the unknown state. Default when verification cost > rescope cost.

NEVER: brief proceeds on captured prose ('per CL-X, the module looks like Y') treated as current verification. CL-X is a historical record, not a live source of truth.

MECHANICALLY in the brief-drafting protocol:
- Brief-drafting prep checklist (currently informal) gets a new mandatory step: 'List every assumption about current repo state. For each, mark (a) verified by Fazal pre-flight, (b) deferred to Claude Code first action, or (c) eliminated by rescope.'
- The brief's metadata section (§12 in current template) gets a new field: 'Ground-truth verification mode for repo-state assumptions: [a/b/c with rationale per item].'
- Briefs without this field default to (b) for everything Clau cannot directly read, and the §2 pre-flight section MUST include rg/grep/git-log commands that Claude Code runs and reports back on before touching code.

This is a STANDING DISCIPLINE addition. Same load-bearing weight as CL-183 (adjacent verification) and CL-178 (Context7 pre-brief gate). The three together form the 'don't ship a brief you cannot fact-check' triad.
