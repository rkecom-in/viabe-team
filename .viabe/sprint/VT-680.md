# VT-680 — §7C impact-judge (design-first)

## The gap (objective O4)
The delegation validate-return seam verifies COMPLETION (did the specialist produce the artifact;
VT-611 verify-floor: executed campaign must PASS verification) — but not QUALITY/IMPACT against the
step's defined outcome (manager_step_desired_outcome / acceptance_criteria already thread through
the framework context since VT-667 iter-2). A completed-but-poor outcome passes silently.

## Design questions (brief to write BEFORE build)
1. Judge seam: the validate-return point in the delegation loop (workflow.py verify leg) — add an
   LLM judge call (metered tier; VT-619) scoring result vs desired_outcome/acceptance_criteria.
2. Verdict handling: score below floor → retry-with-feedback (bounded, 1 retry) vs surface-to-owner
   honest partial vs accept-and-log. NEVER silently re-run money steps (idempotency + approval rails
   bind).
3. Logging: verdicts into tm_audit + VT-514 pipeline observability (reasoning_ref joinable — the
   §7D substrate exists).
4. Measurement interplay: the OFFLINE judge (tier_rescore) stays the gate; this is the ONLINE
   in-loop judge — keep rubrics aligned or the two disagree.

## Gate
Design brief ratified → build → unit + a dev-drive where a deliberately-poor specialist result gets
caught (fixture) + a good one passes → ×3 full-pack re-drive (brain-touch).
