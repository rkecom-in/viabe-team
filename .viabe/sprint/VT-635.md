## Problem

In the enforce manager loop, when manager_review returns the `ask_owner` outcome (the specialist/
manager needs a clarification from the owner before it can proceed), `pending_questions.ask()` only
INSERTs a `pending_questions` row. **No code path composes or sends the owner_question text to the
owner over WhatsApp.** The manager_task_workflow's ask_owner wait loop then waits (with the >24h
`team_reengage` stale nudge as the only owner-facing signal) for an answer to a question the owner
never received.

Owner experience: "I'm on it" → silence → (24h later) a re-engage nudge with no question attached.
= a latent **loop_stall / ignored_speech_act** Tier-1 class (§2.1) on every task that routes
through CLARIFY.

## Evidence

- Mapper audit (T9 inc-3 design workflow, 2026-07-11): the async manager_task_workflow has exactly
  five owner-facing emission points (approval 2a chat-summary, approval 2b template, campaign
  outcome report, terminal-outcome closure, >24h team_reengage). The ask_owner branch is NOT one of
  them — `manager/review.py` ask_owner → `pending_questions.ask()` = INSERT only.
- `workflow.py` ask_owner wait loop polls for an answer; `stale_resume.reengage_stale_task` fires
  only after >24h and sends the generic re-engage template, not the question.

## Fix sketch (to be planned)

At ask_owner arm time, send the owner_question text via the owner freeform seam (send_freeform_ack
path, records its own assistant turn → D1 suppressed) — mirroring how approval 2a delivers the
chat_summary. Idempotency: one send per pending_questions row (guard on the row, not the send).
Money-safety: read-only question, no approval semantics — low risk, but the send must be fail-soft
(a send failure must not kill the workflow; the row + reengage nudge remain the backstop).

## Acceptance

- A scenario driving a CLARIFY outcome shows the question text delivered to the owner in-turn (or
  within the D1 wait budget), with a `pending_questions` row armed.
- Answer path unchanged (owner reply resolves the row as today).
- Legacy/non-enforce paths untouched.
