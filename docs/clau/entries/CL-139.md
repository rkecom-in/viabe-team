# CL-139 — dep-audit work permanently lives under VT-CI pseudo-parent

Locked decision: VT-CI is the canonical parent for all dep audits, lib version bumps, EOL package migrations, deprecated-API migrations, and any cross-cutting infra/build work that isn't a Foundation-class refactor. VT-Foundation remains for repo-wide refactors (monorepo restructures, base-schema changes, etc). No third pseudo-parent without explicit Decision entry overriding this one.

Concrete implications:
- PR #22 stays titled (VT-CI) — correct.
- Future dep audit PRs: (VT-CI) parent, fix-N suffix where stacking applies.
- Future EOL migration PRs (e.g. create_react_agent → create_agent): (VT-CI) parent.
- pr-title regex unchanged. No ci.yml widening micro-PR needed.
- The CL-125 entry itself is not retroactively edited; this entry clarifies and extends it.

Supersedes nothing. Clarifies CL-125.
