# VT-548 — B3 graph-wiring slice 1 (observe-only)

## The seam (recon)
`decide_next_action` (VT-526, manager/decision.py) is the pure manager reasoning half; `SpecialistReturn` (roster.py) is the documented upward payload — but the two were never connected to the LIVE graph. A lane's `push_back_to_manager` emitted the pushback envelope as a tool result and nothing ran the manager decision loop on it; the seam was "documented, per-lane TODO."

## Built (0 migrations) — additive, observe-only
- **`agent/specialist_return.py`** — `parse_specialist_return(envelope) → SpecialistReturn | None` (recognizes the `pushback` envelope + a generic action envelope) and `observe_specialist_return(envelope, *, agent) → ManagerDecision | None`, which runs `decide_next_action` on the REAL return and records the decision to tm_audit (`event_layer='decides'`, `event_kind='manager_decision'`, `status='observed'`). Roster import is **lazy** (kept off the module surface → dep-less-smoke-safe).
- **Wired live at `sales_lane.push_back_to_manager`** — every real pushback now runs the decision loop + records "manager would REVISE/ESCALATE" as a greppable event, reading the ambient `_observability_context` (run_id/tenant_id). Fail-soft: an observability failure never touches the tool return or the brain's turn.
- **OBSERVE-ONLY on purpose:** routing is UNCHANGED — the manager model still drives the turn. The flip to letting the decision steer routing is a later, explicitly-gated slice (the codebase's build-the-rail-then-flip pattern, as OC1 `enforce_policy`). This slice makes the loop RUN + be observable on live data without the blast radius of rewiring `Command` routing across 9 lanes.

## Verify
ruff clean; module imports dep-less (no langgraph pulled at import); the bridge test file importorskip-skips under the dep-less smoke; **30/30** full-dep — parse (pushback/action/unrecognized), REVISE-with-proposed + ESCALATE-no-proposed decisions recorded observed, no-context → decision-but-no-emit, unrecognized → None — PLUS the pre-existing sales_lane + decision tests unchanged.
