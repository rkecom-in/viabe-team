# VT-451 — GST-verify: one auto-retry on a transient Sandbox failure

Built (commit 2f1d0d0). `sandbox_kyc.search_gstin`: a TRANSIENT GST lookup failure (httpx timeout/connect/read,
or a 5xx/504 gateway) gets ONE automatic retry before surfacing `vendor_down` — so a momentary Sandbox blip
(what Fazal hit live) doesn't bounce the owner to a manual retry. A 401 re-auths once (separate path); a 4xx /
parse error is terminal (no retry, fail-closed). Bounded: exactly one retry, no loop. `_is_transient()` helper.
Tests: transient→retry→ok (2 lookups); transient×2→fail-closed (2 lookups). 17 sandbox tests green.
