transiently-failed freeform send is treated as a completed idempotent hit and can't retry within the window.

OUTSIDE VT-387's money-adjacent agent-draft re-send path. Real but lower-risk; fix for consistency.

## Fix (when reached — same as VT-387)
Drop `'error'` from the set so a transient freeform-send failure retries. VERIFY the same double-send
invariant first: `'error'` must never be written post-side-effect (trace the write paths vs the send seam),
and `'sent'` must stay in the set. Add the regression guard (a cached `'sent'` row still dedups, send_fn
not called).
