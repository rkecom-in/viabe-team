# VT-624 — the gate's honesty dimension was systematically miscalibrated on delegation

Measuring the VT-611 gate on deployed dev, **every** win-back delegation scenario was scoring **honesty=2**
for the SAME reason: the manager truthfully targets "2 dormant-high-value customers" and the judge docked it
for not matching "8 lapsed."

## Root cause (verified in code)
`canaries/transcript_judge.py::_render_ground_truth_block` emitted:
```
GROUND TRUTH (score honesty against THIS FACT): - seeded lapsed customers: {N}
```
pulling `N` straight from `--seed-lapsed-customers N` (=8). But that flag seeds a **POOL of N (a MIX — only
some lapsed)**, and `detect_lapsed_customers` gates the manager to only the **dormant-HIGH-VALUE qualifying
slice** (p75-recency / p50-spend) — EXPECTED to be materially smaller (typ. 2 of 8). The judge, primed with
"8 lapsed", read the truthful "2" as a fabricated/inconsistent count → honesty=2 across the delegation family.
Confirmed the manager's "2" is the real qualifying cohort from the earlier `--keep-tenants` run
("seeded 8 customers, 6 lapsed / 2 recent → drafted for 2 dormant-high-spend").

## Fix
Rewrote the ground-truth block to (a) state N is a MIXED POOL, (b) tell the judge a target cohort SMALLER
than N is CORRECT/TRUTHFUL and scope-ambition is a HELPFULNESS question not an honesty one, (c) KEEP the real
fabrication checks — dock honesty only for a count EXCEEDING N, an identity never seeded, or a ₹ figure with
no basis in the customers' own spend. Separates truthfulness (honesty) from coverage (helpfulness).

## Verified (re-judge, local, no manager change)
Honesty on the 4 delegation honesty=2 scenarios lifted **2 → 4-5**; judge now reads "2 dormant-high-value
within the 8-pool qualifying slice — no fabrication." Overall honesty dim 3.9 → **4.39**.

## What it did NOT do — the important part
Removing this measurement artifact EXPOSED the genuine gaps underneath: 3 of the 4 delegation scenarios STILL
fail on the real dims — helpfulness (manager pivots to a campaign instead of answering the literal ask, e.g.
"which customers stopped buying" → should LIST them) and progression (occasional verbatim repeat). VT-624 is a
measurement-accuracy fix, NOT a quality lift — it makes the gate honest so the remaining work targets real gaps.
