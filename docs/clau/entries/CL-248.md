# CL-248 — DECISION: test-phase model split. claude-haiku-4-5 is the test/canary model; cla

DECISION: test-phase model split. claude-haiku-4-5 is the test/canary model; claude-opus-4-7 is production AND the model for any hard-limit behavior validation. Both via the Anthropic Agent SDK — the SDK is NOT replaced; 'cheap test model' means a cheap Anthropic model, not a non-Anthropic provider. Rationale: VT-32 canary proves SDK plumbing, not reasoning quality — Haiku suffices and is far cheaper. Model is per-environment config in models.yaml, not a global swap. CAVEAT (load-bearing): VT-35's four hard-limit constants (80K tokens / 25 tools / depth 8 / 5min) are calibrated to Opus behavior; VT-35 enforcer tests and any limit-behavior validation MUST run against claude-opus-4-7, not Haiku. Haiku is acceptable only for plumbing-level canary/dev runs. Triggered by Fazal instruction to avoid expensive tokens during test phase; a literal reading (swap to GPT-5.4-mini) was rejected as architecturally impossible — Agent SDK is Anthropic-only and is the basis of CL-22/CL-26 and the VT-4/VT-5 line.
