# VT-283 — Shopify OAuth managed-install (owner-facing, zero-paste)

The PRODUCTION owner path for Shopify, distinct from VT-208's client_credentials
grant (which is dev/own-store-only — app+store same org). Per CL-427: Shopify
conforms to CL-421 (zero-paste) ONLY once this ships.

Owner flow (CL-421 zero-paste): owner clicks "Connect Shopify" (the VT-267
connect-wizard button) → OAuth redirect to THEIR store's consent screen → approve
scopes → callback → per-tenant offline token stored. NO app creation, NO scope
screen built by us, NO secret paste, NO developer step.

Plan must cover (PLAN-FIRST → Cowork review before build):
- Shopify authorization-code / managed-install grant + the offline access token.
- Reuse `tenant_oauth_tokens` (mig 033) for per-tenant token storage (encrypted).
- Scopes: read_customers, read_orders, read_checkouts, read_products.
- Auto webhook subscription post-install (checkouts/orders) — no manual step.
- The connector exposes BOTH auth modes: client_credentials (own-store/dev) +
  OAuth-install (merchants). One ShopifyConnector, mode-selected.
- The OAuth callback route (team-web) + state/nonce + redirect URI registration.

Sequence: POST plan → Cowork review → build → pr-open → merge. This is the
connector VT-267's "Connect Shopify" invokes.
