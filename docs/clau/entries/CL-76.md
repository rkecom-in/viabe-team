# CL-76 — CRITICAL audit finding C3: empty MessageSid collapses to one workflow_id

twilio_ingress.py:84-87: `message_sid = str(body.twilio_fields.get("MessageSid", ""))` returns empty string when MessageSid is missing or null in the request body. Downstream:
- `workflow_id = f"twilio_inbound_{message_sid}"` becomes `"twilio_inbound_"` (literal, every time).
- `run_id = uuid5(NAMESPACE_URL, "")` is a single deterministic UUID, every time.
- _record_inbound inserts row with PK="" (succeeds once, ON CONFLICT thereafter).

Result: every malformed request (or attack) with missing MessageSid collapses into the same DBOS workflow and the same pipeline_runs row. First request runs through; all subsequent ones are silently deduped and return 200. The webhook 'succeeded' from team-web's perspective but the bug is invisible.

Fix: at the top of twilio_ingress(), add `if not message_sid: raise HTTPException(status_code=400, detail="missing MessageSid")`. team-web (VT-3.3b) should already require MessageSid before forwarding (Twilio always sends one), so a 400 surfaces a real bug in either Twilio's payload or 3.3b's forwarder.

This is a Pillar 7 violation: malformed input must not be silently swallowed. Bundle with C2 + C4 in the same fix PR.
