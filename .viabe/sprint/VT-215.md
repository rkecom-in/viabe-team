# VT-215 — VT-210 ingestion scheduler workflow registration retrofit

## Why this row exists

VT-200 (PR #96) shipped the pattern of `DBOS.workflow()(fn)` BEFORE `DBOS.scheduled(cron)(fn)` for `purge_workflow_inputs_scheduled`. VT-210's `register_ingestion_scheduler` has the same shape and would emit the same `DBOSWorkflowFunctionNotFoundError` warning every cron tick once that path is exercised. Closing the gap before it fires in production.

CC's plan-ready for VT-200 flagged this explicitly:
> "Same pattern I already used for VT-210's register_ingestion_scheduler (which I should also retrofit, but per brief scope-discipline that's its own row — flag it)."

Cowork locked it out-of-scope for VT-200; this is the row that closes it.

## Scope

- `apps/team-orchestrator/src/orchestrator/integrations/scheduler.py` — apply `DBOS.workflow()(ingestion_scheduler_body)` (or equivalent function) BEFORE the `DBOS.scheduled(cron)` wrap, matching VT-200's `register_purge_scheduler` shape.
- If `ingest_one_connector` is the body — verify which function is the scheduled entrypoint vs the per-tenant fan-out worker. Read the file before deciding which needs the decorator.
