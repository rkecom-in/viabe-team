# VT-293 — Ops Console — Activity / Pipelines

## Why
To steer or debug an owner-agent, the VTR needs its live activity stream — what it's
doing, why, and the ability to intervene.

## What
- Listing of owner-agents; open one → right-drawer live activity stream overlay.
- Stream entries timestamped with trigger / reason / guardrail / tool / retry.
- Overlay actions: Pause / Override / Steer-train / Escalate.
- Reads canonical pipeline_steps / Logfire (no parallel admin state; Pillar 8).

## Acceptance criteria
- VTR sees only assigned-tenant activity (RLS fail-closed; real-PG canary).
- Stream shows real timestamped trigger/reason/guardrail/tool/retry data.
- Pause/Override/Steer/Escalate each resolve to a real action through orchestrator paths + audit-logged.
- PII de-identified (CL-426).

## Out of scope
- Behaviour-quality measurement + training models (VT-294); watchdog detection (VT-296).
