# VT-588 (P1b) — the brain owns off-script turns

## Finding (deployed-dev harness re-run)
Once a Shopify connect is mid-flight (discovery, awaiting store address), the resume gate answered ANY non-domain inbound with a canned "That doesn't look like a Shopify store address, reply with that." — even a real question ("what do you charge?", "did you get my store address?"). The program end-state: the ONE brain owns every in-session turn; flows are context, not the microphone.

## Fix (two parts)
- P1b-1 (shopify_onboarding.py): the discovery gate scans the WHOLE body for a valid domain (not just the first token — handles "here it is: x.myshopify.com"); a malformed domain ATTEMPT still gets the specific reprompt; ANYTHING ELSE (question/proceed/chat) FALLS THROUGH to the manager brain (return None). _is_domain_attempt keys off a DOTTED HOST, not the bare word 'shopify' (so "how do I use shopify?" reaches the brain). Opt-out/DSR still short-circuits first.
- P1b-2 (dispatch.py): _build_onboarding_state_block injected into dispatch_brain — tells the brain which step it's mid-way through (awaiting store address / awaiting 'done') so it ANSWERS the off-script message AND guides the owner back, never fabricating progress. Best-effort, per-turn SystemMessage (cache holds), no PII in the block.
