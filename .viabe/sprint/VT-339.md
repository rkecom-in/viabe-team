# VT-339 — session-scoped conftest migration fixture

The `_dbpool` integration fixture is copy-pasted across test files (test_owner_dashboard,
test_edge_cases, test_edge_cases_pr2). #352 fixed an order-dependency by adding an idempotent
`apply_migrations.apply` to each copy. Cleaner: a single session-scoped conftest fixture that
applies migrations once + exposes the pool, so new integration tests inherit the schema
guarantee without copy-paste. LOW — zero deploy impact; the files self-migrate today.

## Phase 1 (DONE — bounded, Cowork 20260606T014800Z)
`tests/orchestrator/conftest.py` gains a session-scoped `_migrated_db` (migrations once,
idempotent) + a FUNCTION-scoped `_dbpool` (re-establishes the pool each test — a dbos-using
test may shutdown_dbos and clear `graph._pool`; matches the original per-file behaviour). The
4 owner_surface + api files I authored (test_edge_cases, test_edge_cases_pr2, test_support_bot,
test_owner_dashboard) drop their local copy + inherit it. NOTE: must be FUNCTION-scoped — a
session-scoped pool breaks when a later test clears `graph._pool`.

## Phase 2 (REMAINING — supervised follow-pass)
Migrate the other 9 `_dbpool` copies (billing ×7 + observability ×2: test_cost_dashboard,
test_pipeline_log). Deferred to avoid an unsupervised broad shared-test-substrate change.
