# CL-220 — STANDING DISCIPLINE: every brief whose verification step depends on CI running m

Brief default in Step 11 verification step 4 said 'push branch, confirm CI picks up the new job.' Pre-flight by Claude Code surfaced: ci.yml is `on: pull_request: branches: [main]` only. Push-to-feature-branch does not fire the workflow. Implication: for PRs gated by CI, CI green is only observable AFTER PR open, not before. This is the binding signal collapse — PR-open and first-CI-run are the same event. Going forward, every brief whose verification depends on CI running must first check the workflow's `on:` trigger configuration. Adjacent to CL-178 (Context7 pre-brief gate) + CL-194 (mark every repo-state assumption) + CL-183 (adjacent verification anti-pattern). Specific case: CI trigger config is a repo-state assumption that must be verified, not assumed from 'we have CI.'
