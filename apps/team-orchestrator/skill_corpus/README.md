# L4 skill-corpus seed (VT-313)

This directory is the hand-authored **L4 skill corpus** — the domain knowledge the
autonomous growth agent *retrieves* at reasoning time (Pillar 4: retrieve, don't
calculate; Pillar 5: RAG context, not fine-tuned weights). It is the real content
seed that VT-70 built the pipeline for.

One `NNN-slug.md` per note (numbered by the source playbook's note number). Each file
is YAML frontmatter + body, parsed by `orchestrator.knowledge.l4_corpus.parse_doc`
and embedded + UPSERTed into the global `l4_documents` table by `seed_l4_corpus`.

## Authorship + posture

- **`authored_by: fazal`** — the body of every doc is Fazal's playbook note, verbatim.
  This is operating expertise for an autonomous growth *function*, not owner advice:
  the agent runs self-driven; the owner is config + approval gates only.
- **Seed prior, not ground truth.** Every doc body opens with:
  *Seed prior — validate against this tenant's real attribution before treating as
  fact.* Fazal's magnitudes (timings, percentages, thresholds) are starting priors to
  validate against each tenant's real attribution data — never asserted as fact.
- **Edits via PR — not LLM-generated.** These docs are human-authored content. They
  are revised by a reviewed PR that edits the markdown, not regenerated by a model.

## Gated levers

Some notes describe levers the agent must **never** execute autonomously (price moves,
discount depth, tier/anchor/plan changes, public review replies, the complaint freeze).
Those carry a second italic *Gated lever* header and the `gated` tag. The note is **not**
the enforcement — enforcement is the structural guardrail (no direct send/price tool +
`request_owner_approval`, Pillar-7; for the complaint freeze, the VT-321 cohort
exclusion, fail-closed). The header records that the prose is a retrievable prior, not
a control.

## Locked tag taxonomy

`tags` are drawn ONLY from this locked set (no new terms — keeps retrieval facets from
sprawling):

`timing, festivals, whatsapp, sms, email, local, in_person, retention, churn, win_back,
referral, reviews, pricing, discount, bundling, loyalty, cadence, onboarding, complaint,
gated`

`applies_to_business_types` is present ONLY on notes the playbook restricted to specific
verticals (`gym, salon_membership, subscription_box, services, multi_tier_retail,
online_seller`); a universal note OMITS the key entirely (= applies to all).
