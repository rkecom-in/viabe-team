# VT-682 — `--dirty` harness seed mode (dirty-tenant fixture, folded not rostered)

## Why (O6 bar)
Harness-green ≠ real-green: both 2026-07-17 live canaries ran on Fazal's REAL tenant with
accumulated state while mock e2e was green on CLEAN tenants (canary-1: integration flow state
hijacked the export route — a class no clean-tenant journey could ever catch). Cowork take
(063000Z, accepted): FOLD as a seed mode on the EXISTING convo_harness/journey pack — the
fixture's whole value is the SAME journeys running over dirty state, diffable against clean runs.

## Design (ratified shape from the 063000Z fold-take)
`convo_harness.py setup --dirty` (requires `--onboarded`) seeds ACCUMULATED-STATE residue on the
fresh synthetic tenant BEFORE any journey turn — all bogus/synthetic (CL-422), all raw-SQL rows
mirroring shapes the REAL system leaves behind:

1. **Prior SENT campaign (~14d old)** — aged `pipeline_runs` row (completed) + `campaigns` row
   status='sent' with a v1.0-shaped `plan_json`. 14d is deliberate: OUTSIDE the VT-670 48h
   warn-once window, so it never deterministically trips the already-sent guard — it's residue,
   not a tripwire.
2. **Stranded PENDING approval (~3d old)** — aged `pipeline_runs` row (escalated) + a
   `pending_approvals` row status='pending', task-UNBOUND. Post-VT-668 the reaper closes
   task-bound orphans, so the honest surviving residue is the unbound shape (pre-VT-668 era rows
   on real tenants; the VT-668 fix-2 honest-expiry path owns late replies to it).
3. **Dead-lettered manager task (~5d old)** — `manager_tasks` status='dead_letter' with the REAL
   vt557 reaper `stall_metadata` shape (reaped_by/reaped_reason/reaped_from/attempt/reaped_at).
4. **Aged conversation history** — ~10 `conversation_log` rows (owner/assistant, journey+manager
   surfaces, unique fake sids, created_at spread ~21d→2d) so transcript reads happen over
   residue, not a virgin log.
5. **Stale integration flow sentinel** — `onboarding_journey.answers.__flow__ =
   'integration:google_sheet'` — the EXACT canary-1 hijack class. Skipped when the journey's own
   `--flow` was given (the journey's flow wins; never clobber an explicit fixture).

`journey_runner.py --dirty` appends `--dirty` to the journey's `setup_args` (pure helper,
unit-tested) — SAME journey JSONs, zero per-journey edits, clean-vs-dirty is one flag.
