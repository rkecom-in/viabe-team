# CL-201 — context_budgets.yaml per-section budgets are advisory only

Location: apps/team-orchestrator/src/orchestrator/context_builder.py + config/context_budgets.yaml (shipped in VT-3.4 PR 2/3).

SITUATION: The yaml declares per-section token budgets (business_profile 500, customer_ledger_summary 1500, etc.) and a total_cap (8000). The constructor's truncation loop only enforces total_cap (with *SAFETY_MARGIN → effective 6400). Per-section budgets in the yaml are advisory — they govern truncation ORDER (which section gets trimmed first) but NOT individual section size limits.

IMPLICATION: A real bundle where one section is dramatically oversized but other sections are tiny can pass the total-cap check untruncated. E.g. customer_ledger_summary at 4000 tokens (vs 1500 budget) + everything else near 0 = total 4000, under 6400, bundle ships. Agent receives a structurally unbalanced bundle that's 80% ledger.

NOT VISIBLE IN PR 2/3 because substrate is empty (all _build_* return safe-empty/0 tokens). Becomes visible when VT-7.1 + VT-4 land and real reads return data.

FIX (deferred to VT-7.1-bundling PR):
Either (a) document per-section budgets as advisory in yaml + docstring, or (b) enforce per-section caps in the constructor BEFORE the total-cap check. (a) keeps current behavior (matches what's shipping). (b) is the safer long-term design — per-section caps would cap the ledger at 1500 even if total is under, producing a balanced bundle.

RECOMMENDATION at VT-7.1 integration: implement (b). The truncation loop already has the bucket-trim logic for top_spenders and recent_campaigns; extend the same pattern to enforce per-section limits as a pre-pass before the total-loop. Order: per-section pre-truncation (each section to its budget) → total check → cross-section truncation loop if still over.

ALSO related: meta.token_count is the sum of 5 sections, excluding meta itself. Downstream readers that compare token_count to 8000 are comparing apples to a subset. Same fix point.

NOT BLOCKING PR 2/3. Will be visible (and immediately addressable) when VT-7.1-bundling PR lands and the first real bundle exercises substrate reads.
