# VT-239 — Enforce numeric VT-<N> reference in PR-title CI gate

## Why
Fazal directive 2026-05-30: **"All PRs must have a real numeric VT row."** Triggered by PR #142 (docs bootstrap refresh), whose title gate accepted a fabricated text suffix `(VT-Foundation)` because no numeric row existed and the gate only checked for `VT-*`. That collides with the standing rule "VT-IDs numeric only — never invent text-suffix IDs like VT-FOO." The gate must require a real numeric VT reference so chore/docs PRs get a real row instead of a fabricated tag.

## What CC must produce
1. Update the PR-title check in `.github/workflows/ci.yml` (the job that currently asserts a `VT-*` reference) to require a **numeric** reference matching `VT-[0-9]+` and to **reject** non-numeric suffixes (e.g. `VT-Foundation`, `VT-MCP-Tools`, `VT-FOO`).
2. Document the standing rule in `docs/clau/decisions-ledger.md`: "All PRs must reference a real numeric VT row (`VT-<digits>`) in the title; docs/chore PRs included. Text-suffix VT tags rejected by CI." Attribute to Fazal 2026-05-30.
3. No change to feature PRs already using numeric ids (VT-43, VT-48, etc. pass unchanged).

## Acceptance criteria (incl. canary — Rule #15 N/A: CI-config, validated by sample titles)
- Title `feat(x): something (VT-43)` → PASSES.
- Title `docs: something (VT-Foundation)` → FAILS the gate.
- Title with no VT reference → FAILS.
- ci.yml change is the only workflow touched; existing required-check names preserved (don't break branch protection's 11-check contract).
- Ledger entry added.
