# VT-174 — Retroactive Brief Artifacts amendment for VT-101/102/103/104

## Origin

Surfaced 2026-05-26 ~12:30 IST during VT-28 unblock audit. After VT-171 (Logfire migration hot-fix, merged at sha `39ea42c`) shipped, the VT-101/102/104 sprint file bodies still describe their work in LangSmith framing — even though the actual observability backend on `main` is now Pydantic Logfire.

Per **CL-176 Rule #9** (supersession discoverable from both ends): when a Standing decision supersedes prior work, the supersession must be findable from BOTH the new entry AND the older artifacts pointing to it. VT-171's brief cites VT-101/102/103/104 — discoverable from "new" side. But VT-101/102/103/104 sprint files don't yet cite VT-171 — broken from "old" side. A future Claude / Cowork session reading VT-101.md cold could miss that the LangSmith framing is historical.

Fazal authorized 2026-05-26 ~12:30 IST.

## Why this is a "documentation-retrofit" not a code change

The substantive work shipped under VT-171. This row is purely about the audit trail — adding a "Brief artifacts addendum" section to each of the four sprint files so anyone reading them post-VT-171 sees at-a-glance:

- the original LangSmith framing is HISTORICAL
- the observability backend is now Pydantic Logfire (`logfire-eu.pydantic.dev`)
- the migration shipped under VT-171 sha `39ea42c` on 2026-05-26
- token-format contracts (phone_tok_HEX, body_tok_HEX, <redacted:customer_name:len=N>, etc.) are byte-identical preserved
- the redactor seam (`redact_for_log` canonical name) survives the migration; the deprecated alias `redact_for_langsmith` will be dropped in VT-173
