# VT-559 — OTP send-guard bypass

## Problem (audit finding, CONFIRMED)
`auth/twilio_verify.py` builds its own raw Twilio `Client` (lines ~125-138) and sends real WhatsApp OTPs via `verify.v2...verifications.create` — it never passes through `utils/twilio_send._client()`, so `dev_send_guard.maybe_wrap_for_dev` never wraps it. Fail-OPEN on dev: signup pointed at any real number sends a real OTP. Contradicts the single-chokepoint / non-bypassable rail principle.

## Fix
Route the verify client through the same guard chokepoint (wrap with `maybe_wrap_for_dev` identically to `twilio_send._client()`), fail-closed on dev (empty `DEV_SEND_ALLOWLIST` ⇒ mock). Tests: dev OTP to non-allowlisted number is mocked; allowlisted passes through; prod posture unchanged.
