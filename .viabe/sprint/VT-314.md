## 1. The dev voyage key is FREE-TIER (3 RPM / 10K TPM, no payment method)

payment method ... reduced rate limits of 3 RPM and 10K TPM`. Implications:
- **Prod L4 is throttled to absurdity.** L4 retrieval embeds the owner query on
  EVERY dispatch (Composer `_build_l4_skills`) + the corpus is embedded at seed.
  3 RPM ⇒ ~3 dispatches/min ceiling — a launch-blocker for L4 at any real volume.
- **The DR-15 real-call canary rate-limits** when run repeatedly (e.g. iterative
  dev, or several embed calls in one suite run). VT-70 mitigated this in the
  canary (synthetic-embedding logic tests + ONE real-call test) + added bounded
  rate-limit backoff in `embeddings.embed_texts`, but the real fix is a paid key.

**Action (Fazal):** add a payment method to the Voyage organization. "Even with
payment methods entered, the free tokens (200M for Voyage series 3) still apply"
— so it's rate-limit relief, not necessarily new spend until the free tokens are
used. Standard limits apply a few minutes after.
