# VT-482 — business-type reconcile crashes on list-valued GST nature_of_business

## Defect (found driving 63211ce5 to onboarding-complete)
The VT-478 lazy recompose passes the draft's `nature_of_business` to `reconcile_business_type`
as `gst_nature`. The GST verify (`sandbox_kyc.GstinLookup.nature_of_business`) declares that field
`list[str]` — the real shape is e.g. `['Supplier of Services', 'Others', 'Warehouse / Depot']`.
`_best_keyword_key` did `t.lower()` on each input → `AttributeError: 'list' object has no attribute
'lower'`. The reconcile is fail-soft (swallowed), so `_draft_with_reconciled_type` returned the draft
UNCHANGED → the live confirm set kept the raw GBP `category` ("Telecommunications service provider")
→ staleness never detected → the VT-478 recompose silently NO-OPed. The owner is still asked
"We found you're a Telecommunications service provider — is that right?".

The forward path (`auto_discovery_sources._reconcile_type`) never passed `gst_nature`, so VT-475
forward composition + the unit tests (string-only gst_nature) never hit the crash — it only bit the
VT-478 recompose against real GST-verified drafts.

## Fix
`business_type_reconcile.py`: normalize `gst_nature` to one keyword-matchable string at the public
`reconcile_business_type` boundary (`_coerce_nature`: join a `list[str]`, pass a `str` through,
empty → None). Widened the type hint to `str | list[str] | None`. All downstream consumers
(`_deterministic_reconcile`, the LLM-leg thinness check + prompt) now receive a string.

## Proof
- `reconcile_business_type(..., gst_nature=['Supplier of Services','Others','Warehouse / Depot'])`
  → `business_type='services'` (was: AttributeError → no-op → telecom re-surfaced).
- Live (read-only against dev DB, 63211ce5): live confirm set now yields a `business_type='services'`
  confirm; the queued `category` confirm is now detected `STALE=True` → recompose heals it.
- 2 regression tests added (list shape beats mis-category; empty list = no signal). 14/14 reconciler
  tests + 124 onboarding tests green.
