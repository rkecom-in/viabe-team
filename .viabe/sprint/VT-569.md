# VT-569 — the onboarding turn brain

## Live-drill defects (evidence: drill screenshots)
(1) Bare "No" to a confirm re-presents the IDENTICAL question (journey.py treats bare-negative as not-yet-answered with no re-prompt) — dead-end loop. (2) Every walker reply is a fixed template; inside the open 24h session the bot repeats canned copy instead of conversing. Vision + CL-443: the conversational agent is the primary surface; no-fixed-playbook: LLM composes within guardrails.

## Fix
Journey table stays the durable spine (queue/cursor/answers — deterministic bookkeeping + resume). The TURN is LLM-composed: given journey state + the discovered draft + the owner's message, the turn brain acknowledges, extracts any answers present (multi-field from one message), handles rejection intelligently ("No" → ask what's right, offer the precise discovered alternatives), decides phrasing + whether to use in-session interactive quick-replies (≤3 buttons, Meta in-session rules), and answers owner questions instead of steamrolling the script. Deterministic floors stay binding: field promotion via confirm_draft only; consent/opt-out/DSR pre-filter untouched; sends via the guarded chokepoint; fail-soft to the deterministic walker on any LLM failure; env-gated (dev ON).
