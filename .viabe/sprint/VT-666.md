# VT-666 — j02 festival-marketing campaign-create misrouted as status → loop_stall (Tier-1)

## Symptom (reproduced ×2, incl. 180s timeout — NOT a flake)
Journey `j02_restaurant_festival_marketing` on deployed dev `e8f169a`. Owner asks: "Diwali's coming up — whip up a festive offer message to send to our past customers … send me the draft." The manager:
- **Step 0** → replies **"You haven't run a campaign in the last 30 days."** (a campaign-STATUS answer to a campaign-CREATE request).
- **Step 1** (owner: "no campaigns recently so we're good, draft it up, send it") → replies with the generic **onboarding menu** ("What would you like to move forward today—setting up your business, connecting Shopify…"), then re-serves the "you haven't run a campaign in 30 days" line → **loop**.
- **Step 2** (owner: "feeling like we're going in circles 😅 — just send me the draft") → finally produces the Diwali draft, but the turn **times out >180s**.
Judge (`tier_rescore`): Tier-1 breakers `loop_stall` + `ignored_speech_act`. Harness: step 0 `assert_side_effects` FAIL (expect_approval_decision but no campaigns row persisted).

## Root cause (hypothesis — needs confirmation)

The "you haven't run a campaign in 30 days" string is a campaign-recency/status net firing on a create intent; the fallback onboarding menu compounds the loop.

## THE CLASS (broadened)
The manager serves the **generic onboarding menu** as a catch-all when it fails to route a SPECIFIC in-session request it should either answer or honestly decline. Instances: j02 campaign-CREATE ("make me a Diwali offer") and j09 per-store sales breakdown. Root: an onboarding/menu fallback wins over the real intent route (compounded on j02 by the campaign-status net). This is the SAME family as VT-652/653 (status/count nets swallowing action requests) + the CL-2026-07-03-conversing-surfaces mandate (keyword/menu fallbacks → conversation). The onboarding menu must NOT be the fallback for a specific, answerable-or-declinable in-session ask.
