# VT-205 — Connector registry framework

## Why this row exists

The Integration Agent needs a structured knowledge base. One entry per supported tool. Tells the agent: how to authenticate, what scopes to ask for, sample-pull pattern, field-mapping hints. Without this registry, the agent's "knowledge" is ad-hoc prompt engineering — won't scale past 2-3 connectors.

## Scope

### `apps/team-orchestrator/src/orchestrator/integrations/registry.py`

Pydantic-typed registry. Each entry:

```python
class ConnectorSpec(BaseModel):
    connector_id: str            # "google_sheet", "shopify", "paper_book"
    display_name: str            # "Google Sheets"
    category: Literal["digital", "manual", "scrape"]
    auth_flow: AuthFlow          # oauth2 | api_key | service_account | manual_upload | none
    auth_scopes: list[str]       # e.g. ["https://www.googleapis.com/auth/spreadsheets.readonly"]
    auth_walkthrough_url: str    # public doc the agent can show owner
    sample_pull: SamplePullSpec  # how to fetch first 50 rows for field-mapping
    canonical_fields_hints: dict[str, list[str]]  # "phone" → ["Phone", "Mobile", "Customer Phone"]
    rate_limits: RateLimitSpec   # per minute/day per Anthropic prompt-safety
    push_supported: bool         # webhook receiver available?
    pull_default_cadence: str    # cron-like, e.g. "0 9 * * *" (9am IST daily)
```

### Initial registry entries (file-only; implementations come in VT-207+)

- `razorpay` — API key (placeholder spec; impl later)
- `amazon_seller_central` — SP-API (placeholder spec; impl later)

- VT-6 family stubs (paper_book, contacts, upi_export, kot_pos, cash_book, qr_opt_in, apify_*, owner_typed) — minimal spec referencing existing sub-rows VT-52..VT-63

## Acceptance criteria

1. Registry module exports `get_connector(connector_id)` and `list_connectors(category=None)`
2. Each entry validates against `ConnectorSpec` schema; CI gate fails on schema drift
3. Initial registry has ≥10 entries (2 implementable now, 6 digital placeholders, 8 manual stubs)
4. Integration Agent prompt can list available connectors deterministically from registry — no hard-coded names in prompt

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt205_connector_registry.py` — 4 assertions: schema validation, get/list helpers, prompt-listing determinism, manual+digital coverage.

## Out of scope

- Concrete connector implementations (VT-207+ own those)
- Auth token storage (existing secrets pattern handles per-tenant API keys; framework just declares what kind is needed)
- Marketplace ops / Finance / ERP categories (Phase 3+)
