## What CoderC must produce
The `get_recent_campaigns` MCP tool. Returns the tenant's recent campaign history so the agent can avoid repeating recently-tried approaches and learn from outcomes.
1. **Tool implementation** at `apps/team/src/agent/tools/get_recent_campaigns.py`:
	Input schema:
	- `limit: int = 5` (max 20)
	- `status_filter: list[Literal['proposed', 'approved', 'sent', 'failed', 'cancelled']] | None`
	- `since: datetime | None` (only campaigns with `created_at >= since`)
	- `cohort_label_substring: str | None` (substring match on `target_cohort.cohort_label` for finding similar past attempts)
	Output schema:
	- `campaigns: list[CampaignSummary]` — each item:
		- `campaign_id: UUID`
		- `proposed_at: datetime`
		- `status: Literal[...]`
		- `cohort_label: str`
		- `cohort_size: int`
		- `template_id: str`
		- `attribution_status: Literal['pending' | 'closed' | 'never_sent']`
		- `attribution_close_at: datetime | None`
		- `arrr_paise: int | None` (only when attribution_status == 'closed')
		- `attribution_rate: float | None` (attributed customers / cohort_size; only when closed)
		- `customer_ids_in_cohort: list[UUID]` — the cohort, so the agent can detect overlap with current proposal
		- `summary_for_agent: str` (200 chars max, factual: "Sent 2026-04-15 to 87 60-90d dormants; 12 attributed transactions worth ₹18,500")
	- `total_count: int`
2. **Implementation**:
	- SQL JOIN over `campaigns` + `attributions` (aggregated GROUP BY).
	- `summary_for_agent` is templated text generated by Python f-string from numeric fields, NOT an LLM call.
	- Pagination is simple LIMIT (no cursor needed for max-20 results).
3. **Tenant scoping**: ctx.tenant_id only; cross-tenant attack tested.
4. **Tests**:
	- Happy path: synthetic tenant with 10 historical campaigns at various statuses; correct subset returned.
	- Cohort overlap detection: a campaign whose cohort overlaps with another in the result set is correctly returned (the cohort lists are accurate, agent can compute overlap).
	- Attribution aggregation: synthetic attributions → correct ARRR + rate.
	- `attribution_status='never_sent'` for campaigns with status `cancelled` or `failed` (no attribution data exists).
	- Substring filter: `cohort_label_substring='90d'` returns only campaigns with that substring.
	- Cross-tenant attack: tenant_A cannot see tenant_B campaigns.

## Architectural rules to enforce in this subtask
- Pillar 2: pure SQL + aggregation; no LLM.
- Pillar 4: `summary_for_agent` is templated text, not LLM-generated. CodeX confirms by reviewing the f-string.
- Cohort overlap is a data field; the agent decides what to do with it. CodeX rejects fields like `is_cohort_overlap_significant` — that's reasoning.
- Attribution rate is computed deterministically. CodeX rejects probabilistic models.
- The 20-campaign cap is Phase 1; raising requires Type 2 (cost: more campaigns = larger context).
