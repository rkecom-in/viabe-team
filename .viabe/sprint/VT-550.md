## Built (mig 155)
- **`agent_memory` table** — the learnable-memory store. **GLOBAL** rows (`tenant_id IS NULL`, `memory_scope='global'`) = archetype seeds, a shared head-start every tenant may READ but **only the SERVICE path writes** (the INSERT policy is tenant-scoped → a tenant can never write a global seed). **TENANT** rows = the tenant's own seed/learned memory. Content **PII-redacted at write**. Retrieval-gate cols **default-CLOSED** (Phase-2 activation, no ALTER later) — same posture as C3 corrections.
- **`agent_memory.py` mechanism** — `upsert_seed` (global via service / tenant via conn), `upsert_learned` (OVERWRITES the seed for the same key, version+1 = **grow beyond the seed**), `get_active_memory` (retrieval interface, learned-over-seed precedence, **DEFAULT-CLOSED → returns [] until activated**), `mark_retrievable` (the Phase-2 activation flip).
- **DSR** — `agent_memory` in `_PURGE_ORDER`; the `WHERE tenant_id=…` purge erases TENANT rows and naturally skips GLOBAL seeds (archetype knowledge, not a subject's data).
