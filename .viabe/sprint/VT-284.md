# VT-284 — UPI ingestion ease (forward-statement-to-WhatsApp)

Connector-audit ease follow-up on VT-57 (CL-427). VT-57 ingests a UPI export the
owner uploads; this minimizes the export friction: the owner FORWARDS their
PhonePe / GPay / Paytm statement (PDF/CSV email or in-app share) straight into
WhatsApp, and the ingestion path picks it up — no manual "export → download →
upload" dance.

Scope (to design): the WhatsApp media/document inbound path → route to the VT-57
upi_export parser; handle the providers' email-statement formats; owner just
forwards. Reuses VT-57's parser + the two-surface seam. Backlog — UX ease, not a
launch-gate.
