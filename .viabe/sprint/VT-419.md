## Contract (Cowork 2026-06-24, from VT-417 PR-1 deferral). BUILD post-e2e — NOT blocking.

VT-417 PR-1's Shopify `backfill_orders` caps the initial Admin-API orders pull at **250** (one page; Link-header
cursor pagination deferred). Fine for the dev-store canary + most SMBs, but a store with >250 historical orders
would silently miss the older sales on initial connect.

## Scope
- Add Link-header cursor pagination to `connectors/shopify.py` `pull_orders`/`backfill_orders` so the full order
  history ingests on connect (page through `rel="next"` until exhausted), feeding the same `ingest_customer_rows`.
- Idempotency already covers re-pulls (the ledger `entry_key` UNIQUE) — a re-run is safe.
- Test: a paginated mock (3 pages) ingests all rows, no dupes.

## Gate
BUILD (low-risk, additive). Self-merge on green when reached — OR Cowork gate if it touches the ingest seam.
