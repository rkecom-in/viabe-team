# VT-176 — Replace VT-28 shell trigger bodies with real implementations

## Origin

VT-28 (Done sha `05837e3`) shipped Path-(b) infrastructure: 4 `@DBOS.scheduled` workflows registered with deterministic cron + workflow_id derivation + idempotency + pipeline_log emission. Bodies are shells: each emits a `*_shell` event with `payload.status: skipped_schema_pending`. Reserved completion event names (`attribution_closed`, `day39_evaluated`, `day39_continue`, `day39_refund_triggered`, `monthly_impact_started`) are deliberately NOT emitted yet — per VT-28 review §Condition 2 (phantom-Done prevention, CL-318/319/380 failure-class).

VT-176 replaces the shells with real bodies once VT-175's schema + day-39 evaluator land.

## Why Critical / Sprint 1

Same Pillar 1 Type-3 commitment driver as VT-175. The triggers were registered in VT-28 but they emit shells. Until VT-176 ships, no actual day-39 evaluation runs, no actual attribution closes, no actual monthly impact prep happens. The infrastructure exists; the feature doesn't.

## Dependencies

- **VT-175** — MUST be merged. Provides `attributions` table + cadence columns + `day39_evaluator.evaluate_day39()` + `attribution_close.close_attribution()`.
- **VT-28** (Done sha `05837e3`) — scheduled-triggers infrastructure; this row only modifies the body callables.
- **VT-9.6** (or whatever the post-cutover VT-N is for the monthly impact PDF generator) — monthly impact trigger calls into it; if not yet shipped, monthly impact body stays partially shell. Surface at brief-ready time.
