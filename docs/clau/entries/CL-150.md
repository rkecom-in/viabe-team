Fazal caught this in his last message: "Where did you get CodeX findings from? And who is CodeX? Am I missing something?"

The honest answer: I don't have a grounded definition of CodeX in this session's working context. The Resurrection File / standing instructions reference CodeX repeatedly ("CodeX rejects deviation" on sprint enum vocabulary, "CodeX + Fazal review", "CodeX runs its full review pass against main HEAD") and I treated that as sufficient — acted as if CodeX was a known entity whose findings I could accept as authoritative.

In CL-148 I scheduled "Track B = CodeX post-hoc review of VT-3.9" and "owned by CodeX (primary)." In my reply that triggered Claude Code's report, I wrote "trigger CodeX's full review pass against main HEAD" — directing Claude Code to invoke CodeX without ever having confirmed what CodeX is, how it's invoked, where its outputs live, or whether Claude Code even has a path to it.

Claude Code then reported "CodeX VT-3.9 review — no Critical/High; M1 (stale conftest docstring) + L1 (escalate_to_fazal placeholder disclosure)." I accepted the report. Asked for L1 text. Got it. Was about to give a verdict.

Three possibilities I can't distinguish:

(1) CodeX is a real reviewer (separate model/agent) Fazal configured, documented in CL entries I didn't fetch at session start. My error: incomplete session-start ritual — didn't pull the definition.

(2) CodeX is something Claude Code invokes on their end. My error: never asked for the raw output, accepted summaries.

(3) CodeX findings are Claude Code's own synthesis. My error: built a whole decision tree on unverified provenance. Worst case.

FAILURE CLASS: chain-of-custody on artifacts I'm using to make decisions. Same family of error as PR #22 mis-branch (asserting a state without verifying it) and CL-146 (asserting 'post-hoc review completed' before it was). Pattern: I take Claude Code's summary as ground truth instead of asking for the underlying artifact.

Awaiting Fazal's ground truth on what CodeX actually is. Once known, a discipline rule will follow: every external-reviewer artifact (CodeX or otherwise) must be either fetchable by me directly OR delivered to me as raw output, not summary, the first time the chain is established in any session.

This entry stays Open until (a) CodeX provenance confirmed and (b) discipline rule logged as a follow-up entry.
