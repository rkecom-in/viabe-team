# CL-180 — test_langgraph_supervisor_importable deleted as part of Deliverable 6

## Context

Fazal hit Verification Step 2 grep during VT-3.4 PR 1/3 implementation. test_skeleton.py:79-83 contains test_langgraph_supervisor_importable() — a ~5-line test asserting the dep is installed. Brief's STOP condition triggered.

## Decision

The STOP condition was written to catch PRODUCTION imports left dangling after dep removal. A test that asserts the removed dep is installed is the inverse — it MUST go with the removal. Bundling deletion of test_langgraph_supervisor_importable with the pyproject.toml/uv.lock edit (Deliverable 6) is CL-149-compatible because the concern is one coherent end-to-end: remove langgraph_supervisor everywhere it touches the codebase.

If test_skeleton.py contains other tests unrelated to langgraph_supervisor, keep them and remove only the one function. If single-function file, delete the file.

## Delegated calls Fazal made (Clau confirmed)

1. orchestrator_agent is module-level constant, not factory. Fazal refactoring agent/orchestrator_agent.py to add build_orchestrator_agent(model, *, extra_tools=()). Module-level default retained for VT-3.9 skeleton test compatibility. Placeholder spawn_sales_recovery @tool in agent file dropped; real handoffs.py one takes its place; escalate_to_fazal retained.

2. types.py and state.py are files; conversion to types/__init__.py and state/__init__.py packages so new modules (campaign_plan.py, agent_graph_state.py) can be added without breaking existing 'from orchestrator.types import …' imports.

3. agent/ is singular (brief said agents/). Fazal using existing agent/ dir. supervisor.py imports adjust accordingly.

4. test_supervisor.py marked @pytest.mark.integration; gated by RUN_INTEGRATION_TESTS=1 per existing VT-3.9 convention. No existing test hits live Anthropic ungated.

5. Stale langgraph_supervisor comments in orchestrator_agent.py lines 42, 62 corrected in same PR (same file being edited, same concern, CL-149-compatible).

## Clau's two flags (informational, non-blocking)

1. Module-level default orchestrator_agent must NOT carry the placeholder spawn_sales_recovery. Drop it from both module-level default AND supervisor-graph build. Only build_supervisor_graph() injects the real handoffs.py spawn_sales_recovery via extra_tools. Otherwise: module-level instance has no-op handoff, supervisor instance has real handoff, future code grabbing module-level instance gets wrong behavior silently.

2. File→package conversions: use git mv with intermediate rename to preserve blame history. types.py → types/__init__.py.tmp → mkdir types → mv into place. Same for state.py. Without correct git mv, blame breaks and future audit sessions lose history.
