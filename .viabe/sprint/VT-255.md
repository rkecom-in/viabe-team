# VT-255 — CI Actions-diet (reduce GitHub Actions spend)

## Why
Fazal directive: PR-run failures + redundant triggers are burning Actions minutes. Extends the VT-245 trigger-diet. Data (last 40 runs): ~17% PR failure rate, all attempt=1 (real failures from code that skipped the full local suite, not flaky reruns); ci.yml was triggering on `edited` so every title/body edit re-ran the ~17-job suite (CL-423 title churn amplified this).

## Shipped — PR #177 (safe, merge under housekeeping authority)
- ci.yml: drop `edited` trigger.
- Move PR-title check to its own 1-job `pr-title.yml` on [opened, edited, reopened] → title edit cost ~17 jobs → 1.
- (cancel-in-progress already ON; superseded pushes die.)

## Deeper levers — plan-first, NOT yet built (CL-424 big/risky)
1. **Per-path job gating** (biggest saver): `dorny/paths-filter` + per-job `if` so orchestrator-only PRs skip web jobs and web-only PRs skip postgres/migrations. Risk: a mis-gate skips a job — lower stakes now that checks are advisory (VT-245 turned off required-status-checks).
2. **Pre-push hook DB coverage** (root-cause): hook currently runs only dep-less smoke, not the orchestrator(DBOS+pg)/migration jobs where failures originated. Extend to run them against local pg16+pgvector when reachable (graceful-degrade like the docker step) → failures stop reaching CI. Proven workable in VT-254.
