## Why

Fazal 2026-05-29 21:25 IST. Magic-link auth was VT-203's choice for zero-paste UX per CL-421. But for the OPERATOR persona (Fazal himself), hitting login 20+ times a day during testing:

1. Supabase project-wide email rate limit kicks in (default 30/hour)
2. Each magic link requires opening email inbox + clicking link + waiting for finalize redirect — slow
3. Even with VT-236's 7-day session, fresh logins (cookie cleared, new browser, etc.) still hit the email rate limit

Password-based auth solves all three. Magic link stays as fallback for password reset.

**Note:** This applies to the OPERATOR (Fazal) Ops Console login only. Customer-facing tenant onboard (`/team/onboard`) keeps the magic-link / Google OAuth flow per CL-421 (zero-paste).

### 3. `/api/ops/login` route — branch on `password` presence

```ts
export async function POST(req: Request) {
  const { email, password, next } = await getBody(req)

  if (typeof password === 'string' && password.length > 0) {
    return handlePasswordSignIn(email, password, next, req)
  }
  // existing magic-link flow continues
  return handleMagicLink(email, next, req)
}

async function handlePasswordSignIn(email, password, next, req) {
  const supabase = serverSecretClient()
  const { data, error } = await supabase.auth.signInWithPassword({ email, password })
  if (error || !data.session) {
    return NextResponse.redirect(
      new URL(`/team/ops/login?error=${encodeURIComponent(error?.message ?? 'invalid_credentials')}`, req.url),
      { status: 302 },
    )
  }

  // Same allowlist gate as VT-203 fix-2 / VT-233
  const userId = data.user.id
  const operatorAllowlist = (process.env.FAZAL_OWNER_UUID ?? '').trim()
  if (!operatorAllowlist || userId !== operatorAllowlist) {
    return NextResponse.redirect(
      new URL('/team/ops/login?error=not_authorized', req.url),
      { status: 302 },
    )
  }

  // Mint operator JWT + set cookie (reuse the logic from finalize-hash; extract to shared helper)
  const opJwt = await issueOperatorJwt(userId)
  const safeNext = computeSafeNext(next)
  const res = NextResponse.redirect(new URL(safeNext, req.url), { status: 302 })
  res.cookies.set('viabe_ops_jwt', opJwt, {
    httpOnly: true, secure: true, sameSite: 'lax',
    path: '/team', maxAge: 60 * 60 * 24 * 7,
  })
  return res
}
```
