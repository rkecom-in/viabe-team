## Why this is Critical / emergency-data-rescue

1. **Data-loss risk:** if this working directory is wiped (machine reset, disk failure, accidental `rm -rf`, OS reinstall), the entire 2026-05-25 migration output + every Cowork artifact since then is GONE. There is zero remote backup.
2. **Cross-space resurrection is broken:** `CLAUDE.md` is supposed to bootstrap fresh Claude sessions (Dispatch, fresh windows, phone) per the operating-brief. It can't, because it's not on GitHub. A fresh `git clone` from main returns only `apps/team-orchestrator/` + `apps/team-web/` + a handful of daemon files.
3. **Rule #16 substrate is unsynced:** `docs/clau/active-context-summary.md` (the mandatory pre-brief-ready check substrate per DR-16) is local-only. Without the rescue, the rule's check script `scripts/check_brief_against_ledger.py` ALSO isn't on GitHub — so Rule #16 can't be enforced in CI even if we want to.
4. **Cowork Projects users (Dispatch) won't see this:** Project context inherits the mounted folder, which gets the LOCAL files. But that means Cowork Projects is tied to ONE machine's local state. Not portable.
