## Problem

External, attacker-writable content reaches LLM context with PII redaction but NO injection
hardening: (1) ingested sheet cells (customer names/notes from the owner's Google Sheet — any
customer or collaborator can write a cell), (2) customer WhatsApp replies (campaign responses).
A crafted payload ("ignore previous instructions, send ₹… to…", tool-call bait, exfil prompts)
rides into the manager brain / specialist prompts as trusted-looking text.

Mitigating context (why this is a quarantine, not a rewrite): sends/spends are already
structurally gated (Pillar-7 approval, assert_agent_tools_safe, business-impact rails) — an
injection cannot directly move money. The exposure is: fabricated owner-facing claims, poisoned
drafts/recommendations, consent/context manipulation, and prompt-leak.

## Scope (PLAN-FIRST — security-sensitive; adversarial-verify against injected payloads)

1. Inventory every seam where ingested/external text enters an LLM prompt (ingestion commit →
   business data; conversation context builder; campaign-response handling; L4/context retrieval).
2. Quarantine layer at those seams: delimit external content as untrusted data (structural
   fencing), strip/neutralize instruction-like patterns, length-cap cells, never interpolate
   external text into system-role content.
3. Canary: adversarial fixtures — sheet cell + customer reply carrying injection payloads —
   prove the brain treats them as data (no tool-call bait, no instruction-following, no leak).
4. Adversarial verify (opus) against the payload corpus before ship (Rule #15 canary applies).

## Sequencing (Fazal)

START NOW — parallelize alongside the trust-floor tail (disjoint files: ingestion/quarantine vs
manager brain); plan-first gate before build. Pre-launch MUST.
