# VT-337 — CI-hygiene: green ci.yml so the Railway orchestrator deploy unblocks

ci.yml `lint` + `test` have been RED on main since ~#345 (VT-83). Per the CLAUDE.md deploy
topology, Railway native deploy has "Wait for CI" ON — a red ci.yml on a main push SKIPS
the orchestrator deploy. So VT-83/84/94/99/331/91/335 likely have NOT deployed to dev.
This is the deploy-unblocker (Critical, ahead of VT-87).

## Root cause
- **`test` job (dep-less `uv run --no-project`):** 3 test modules import a heavy dep at
  COLLECTION time (no skip guard): test_approval_reply.py + test_edge_cases.py
  (owner_inputs/__init__ -> writer -> `anthropic`), test_vt153_body_backfill.py (`psycopg`
  via a module-level import below a runtime-only `pytestmark` skip). The pre-push hook's
  dep-less smoke did NOT catch it because the LOCAL `--no-project` env still resolves the
  deps from the uv cache; CI's clean runner does not.
- **`lint` job ("No LLM in deterministic subtree"):** `grep -rniE 'anthropic|...'` matched
  the WORD "Anthropic" in pre-existing PII-disclosure COMMENTS (runner.py, consent
  handlers), not actual LLM usage.

## Fix (this row)
- `pytest.importorskip("anthropic")` at the top of test_approval_reply.py + test_edge_cases.py
  + test_edge_cases_pr2.py; `psycopg = pytest.importorskip("psycopg")` in
  test_vt153_body_backfill.py. They skip cleanly dep-less, run fully in the real-PG suite.
- lint gate grep → match `import (anthropic|openai)` / `from (anthropic|openai)` /
  `.messages.create` / `claude-[a-z0-9]` / `Anthropic(` / `OpenAI(` — real usage, NOT the
  bare word, so PII-disclosure comments stop false-tripping. Comments preserved.
- (follow-up) harden the pre-push hook dep-less smoke so this trap class can't ship again.

## Acceptance
ci.yml `test` + `lint` go GREEN on a real run → the accumulated dev deploys unblock.
