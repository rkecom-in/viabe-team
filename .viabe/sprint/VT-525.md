# VT-525 — B2 task/step persistence spine

## What
The durable, tenant-scoped spine for owner-intent the manager takes on. Sits ABOVE the two existing guarded effect pipelines (campaign-collapse AND the Gap-5 agent-draft path) as a supervisory/evidence layer — reconciled under it, NOT replaced. The manager decision loop that reasons over these rows = B3 (VT-526, next).

## Built
- **`manager_tasks` (mig 151)** — objective/acceptance_criteria (REDACTED JSONB), source_message_ref (pointer, not body), status machine `clarifying→planned→running→(waiting_owner|blocked)→verifying→(completed|failed|cancelled)`, `current_step_id`, `evidence_refs`, `idempotency_key` (redelivery-safe), `version` (CAS). RLS+FORCE + 4 tenant policies + operator-JWT SELECT (redacted state for the review loop; assignment-scoping = VT-533).
- **`manager_task_steps` (mig 152)** — ordered step plan (supersedes one-spawn-per-turn); polymorphic `evidence_kind {campaign_plan|agent_work_item|pipeline_run}` + by-value `evidence_ref` (NO hard FK — the two pipelines don't share a key); CAS `version`; REDACTED `detail`.
- **`manager/task_store.py`** — create/mint (parent-row lock serializes minters incl. cross-process; ON idempotency-hit return existing), add_step, `set_task_status`/`set_step_status` under the **CAS guard** (`status = ANY(expected_from)`, rowcount-0 → log-not-raise; never regress a terminal — the coordinator VT-374 pattern). All free-text/JSONB redacted at write (pii_redactor). Tenant-scoped via `tenant_connection` (RLS-enforced).
- **Orphan detector** — `orphan_reaper.reap_stalled_manager_tasks` (mirrors the VT-481 reaper shape: >1h age floor, service-role, best-effort, never-raise). Predicate: a task in planned/running/verifying with NO non-terminal step → the "runnable step / durable wait / explicit blocker" invariant is violated → flip to `blocked` + stall_metadata. Wired at boot next to `reap_orphan_runs`. Deliberate waits (clarifying/waiting_owner/blocked) excluded.
- **DSR** — `manager_task_steps` + `manager_tasks` added to `dsr_purge._PURGE_ORDER` (steps-before-tasks) in the SAME PR + erasure proven in `test_dsr_purge_substrate` (VT-518/VT-524 discipline).
