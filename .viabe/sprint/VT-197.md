# VT-197 — Day-39 → context-bundle reflection loop

## Why this row exists

VT-175 shipped the Day-39 self-evaluator schema + evaluator runner. The evaluator runs post-campaign and produces a reflection record. But that record's output goes nowhere yet — no consumer reads it. The reflection loop is open-ended.

For the moat to function, the evaluator's findings must feed back into future agent behavior: "campaigns of type X on customer-segment Y produced outcome Z; next time you see Y, factor that in." Without closing this loop, Day-39 is decorative.

## Scope

1. **Define the feedback contract.** What fields does Day-39 evaluator produce that the orchestrator-agent's next context bundle should include?
2. **Add a `context_bundle_priors(tenant_id)` query** that reads Day-39 evaluator records + summarizes them into a context block (e.g., "Last 30 days: 8 campaigns ran; success rate X%; pattern observed: customers Y respond best to Z").
3. **Integrate into Context Composer** — `assemble_context_bundle(tenant_id)` includes Day-39 priors in the agent's invocation context.
4. **Operator-visible.** Ops Console surfaces the Day-39 priors view per tenant — "what the agent currently believes about your business."

## Acceptance criteria

1. Day-39 evaluator record produces structured fields consumable by `context_bundle_priors()`
2. Agent invocation context includes the Day-39 priors block as a system-message segment
3. Agent response demonstrably references priors (canary asserts agent mentions or considers prior pattern when relevant)
4. Ops Console shows the priors block in a readable format per tenant

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt197_day39_reflection.py` — provisional 4 assertions exercising the loop end-to-end (seed a Day-39 record → fire a brain dispatch → verify context bundle includes priors → verify agent reasoning references prior).

## Out of scope

- Auto-tuning of campaign parameters from Day-39 data (defer; this row just CARRIES the data into context, doesn't act on it)
- Cross-tenant Day-39 aggregation (privacy locks; defer)
- Day-39 evaluator improvements (separate VT row when needed)
