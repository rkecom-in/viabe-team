# VT-204 — Integration Agent (epic, Sprint 2 anchor)

## Why this row exists

Without owner data, every downstream agent is decorative. Sales Recovery has nothing to recover; the moat work has nothing to learn from. The Integration Agent is the load-bearing first step of the entire product loop.

Fazal directive 2026-05-28: "An integration agent takes the owner through onboarding and integration, asks for first-time data share, suggests recurring options. The user just selects an option, the agent does the rest. Intelligent agent with brain + prior knowledge for all known popular tools (Shopify, Sheet, GA, CRM)."

Locks Sprint 2 sequencing: Integration Agent first → real data flows → Sales Recovery proven → moat substrate (L1/L0-writes/Day-39-feedback) ships on real data not synthetic.

## Use case (acknowledged by Fazal 2026-05-28 ~00:25 IST)

4 phases:
1. **Discovery** — agent asks non-technical questions; owner names their tools (or "I have nothing, here's a Sheet")
2. **First-time share** — agent picks connector, walks through OAuth, pulls sample, confirms field mapping
3. **Recurring** — agent offers push (webhook) or pull (schedule); runs hands-off forever
4. **Expand** — new tool added later → re-runs phases 1-3 + dedupes against existing data

RKeCom bootstrap: Google Sheet with dummy customer + drop-off data. Sheet first; Shopify next; framework supports all known popular sources.

## What this parent owns

- Connector registry — per-tool knowledge (auth flow, scopes, sample-pull pattern, field-mapping hints)
- Onboarding conversation flow — agent's discovery/setup/expand decision tree
- Field-mapping reasoner — Opus 4.7 over sample data; proposes mapping; asks owner only when unclear
- Schedule manager — DBOS scheduled workflows for pull cadence; webhook receivers for push
- Per-tenant tenant_config — encoded business rules (e.g., "skip cart_value < ₹300")
- Dedupe substrate — phone_hash across sources

## Acceptance criteria (whole epic; sub-rows have their own)

1. RKeCom owner can onboard via Sheet in <15 min without reading docs
2. First batch of drop-off rows lands in DB tagged `acquired_via='google_sheet'`
3. Recurring ingestion runs hands-off (push OR pull)
4. Adding a second source (Shopify or another) dedupes against existing customers via phone_hash
5. Owner sees integration status in Ops Console: "Sheet: connected, last sync 5 min ago, 12 new rows today"
