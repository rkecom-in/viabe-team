# CL-420 — Sprint 2 re-anchored: Integration Agent first; moat work follows

## Fazal directive (2026-05-28 ~00:25 IST)

> "We are handicapped until we have an Integration Agent. It takes the owner through onboarding and integration, asks for first-time data share, suggests recurring options. Intelligent agent with brain + prior knowledge for all known popular tools (Shopify, Sheet, GA, CRM). This part may have to come first, because without this, its a nightmare for us to get any data from the business."

> "Until then, the E2E test is useless."

## What Clau missed

Clau's VT-6 epic (9 ingestion methods, sub-rows VT-52..VT-63) covered the *manual / un-digital* tail:
- Paper book photo (vision LLM)
- Phone contacts list
- UPI export
- KOT/POS export
- Cash-book photo + voice
- QR opt-in
- Apify scrape (Zomato/Swiggy/GBP — context only)
- Owner-typed WhatsApp

Reasoning was "70% of Indian SMBs don't run a POS." Defensible for the target tier — but missed:
- **Shopify, WooCommerce, Amazon Seller Central** — the digital SMB tier (RKeCom-class)
- **Google Sheets** — universal bootstrap (every business has one)
- **GoHighLevel, HubSpot, other CRMs** — owners already running them
- **Google Analytics 4, Meta Pixel** — drop-off intent signals
- **Razorpay** — payment-reconciliation source

Clau had earlier even acknowledged this gap to Fazal: "Inventory, ERP, Accounting, Amazon Seller Central… those are explicitly Phase 3+." Locking the digital sources behind Phase 3 broke the Phase 1 value loop.
