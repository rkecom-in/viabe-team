# VT-296 — Ops Console — Monitoring / Watchdog board

## Why
Continuous autonomous monitoring surfaces agent crash / stall / misbehaviour as a board
so the VTR sees fleet-wide health without polling each agent.

## What
- Board view: continuous-monitoring detectors → status (crash / stall / misbehaviour),
  per assigned business, with severity.
- Detector definitions feed the VT-298 Telegram alerting path.
- Deep view → overlay (drill into the offending run/agent).

## Acceptance criteria
- VTR sees only assigned-tenant monitoring (RLS fail-closed; real-PG canary).
- A detector firing (crash/stall/misbehaviour, sandbox/mock) shows on the board with severity.
- Board items resolve to a real run/agent overlay (no dead-ends).

## Out of scope
- The Telegram alert delivery (VT-298); the live stream (VT-293).

## Notes
Build deferred behind launch queue. Pairs with VT-298 (board = the visual; VT-298 = the push). One PR (CL-424).
