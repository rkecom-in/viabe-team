LOCKED DESIGN (Fazal 2026-05-19, post Clau stress-test):

AgentGraphState extension in this PR adds 5 fields (total=False TypedDict pattern):
- tenant_id: UUID | None = None
- run_id: UUID | None = None
- trigger_reason: Literal['weekly_cadence', 'owner_initiated', 'edge_case_response', 'support_followup'] | None = None
- sales_recovery_context: SalesRecoveryContext | None = None (already in brief §3.9)
- terminated_without_spawn: bool = False (already in brief §3.6 / CL-188)

Clau refinement to Claude Code's proposal: trigger_reason state field defaults to None, NOT 'weekly_cadence'. The 'weekly_cadence' fallback lives only at the spawn tool's read site: state.get('trigger_reason', 'weekly_cadence'). Rationale:
- None at state schema = 'nobody set this upstream' is observable and loud at debug time.
- Literal fallback at read site = defensive at the only place that consumes it.
- A real literal as schema default would silently mask the no-upstream-source case (every bundle would say weekly_cadence even when owner_initiated was the real trigger; we'd never know).

SECURITY-RELEVANT BEHAVIOR (intentional, not a bug):
A graph invocation that forgets to set tenant_id will have state.get('tenant_id') == None. assert_tenant_scoped then has expected_tenant_id=None, every real DB row will mismatch (no row has tenant_id=None), and the guard will raise TenantIsolationError on every read. Tests catch it loudly. This is the correct fail-mode — missing tenant_id is a security-relevant bug and silent fallback would be worse than loud failure. Document this in _tenant_guard.py docstring.

UPSTREAM WIRING (where tenant_id, run_id, trigger_reason actually get set in production):
Deferred to a later PR (VT-3.3 / VT-3.5 / VT-3.8 runner integration). PR 2/3 ships the state schema + test wiring; real-runner wiring lands when the upstream nodes get their PR. Matches PR #26's deferral pattern.

TESTS (§4.5/§4.6) supply tenant_id, run_id, trigger_reason directly in test invocations of build_supervisor_graph(...).invoke({...}). §4.1/§4.3 monkeypatched tests pass them as constructor args directly (no state involvement).

RESOLVES: §2.1 verification stop-condition.
UNBLOCKS: Claude Code proceeds with full PR 2/3 build.
