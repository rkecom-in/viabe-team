# VT-632 — the manager doesn't OWN its owner-facing action (it's scraped text, not a tool call)

## Key finding (plan-first design, 4 read-only mappers + architect synth)
The Claude-Code shape is **~90% already present.** `build_orchestrator_agent` is a langchain create_agent ReAct
loop with real tools + VT-484 middleware keeping tool_use/tool_result valid, one graph.invoke in dispatch_brain.
**Single structural defect:** the owner reply is NOT a tool — it's the trailing `AIMessage.content`, reverse-
scraped by `_last_manager_reply_text` (dispatch.py:1119) then pushed by a deterministic ladder. The ENTIRE
lattice (L0-L5 + the runner D1 "I'm on it" net) exists ONLY to compensate for "the reply is inert text the loop
never executed." Fix = make the reply a TOOL, retire the lattice. Not a rewrite.

## Target (Claude-Code shape, legacy/shadow path = Fazal's primary today)
- Deterministic pre-model floor UNCHANGED + still pre-model (opt-out/DSR/dupe/consent/run-control) — never model-chosen (DPDP floor).
- Brain's ONLY owner channel = calling `reply_to_owner(text)`; only way to act = `delegate(...)` / `arm_owner_approval(...)`. Standard ReAct: reason → advisory/delegate tools → read tool_results → reply_to_owner → stop.
- "Did it reply/act" becomes a FACT (`terminal_state['owner_reply_emitted']`), not a heuristic (reverse-scan + difflib).
- `reply_to_owner` routes through the EXISTING `send_freeform_ack` chokepoint (records the turn). **Recipient resolved SERVER-SIDE from the tenant owner phone — model NEVER supplies a number** (closes the fabricated-number / send-to-customer risk of handing the brain a send tool). Customer sends stay forbidden.
- Sync/advisory `delegate()` returns the specialist envelope INTO the loop as a tool_result the model narrates → "delegate → get result → reply about it in-turn" finally exists → "I'm on it → silence" structurally impossible for sync work. Heavy/durable specialists stay ASYNC (manager_task_workflow) but every async terminal (blocked/escalate/empty-cohort/verified) MUST re-drive a one-shot manager turn ending in reply_to_owner.
