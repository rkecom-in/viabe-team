# VT-238 — Terminal-style stream view

## Why

Fazal directive 2026-05-29 22:25 IST: stream page needs redesign. Earlier directive 2026-05-29 18:30 IST clarified the desired aesthetic: "a running log in a terminal view, with options to filter by tenant, agent, other essentials."

Current `/team/ops/stream` (VT-201) ships card-style row layout. VT-235 styling pass applied gray-50/white-card baseline but kept card layout. For operator who's tail-following pipeline activity across all tenants, a continuous terminal-style log is faster to scan than discrete cards.

## Brief contract

**Visual layer replacement only.** Substrate from VT-201 (Supabase Realtime + filters + operator JWT) preserved. The Realtime subscription, filter state machine, and data shapes stay.

CC implements:

### 1. New `apps/team-web/components/ops/stream-terminal.tsx` (client component, replaces `<StreamRowList>` usage in `<StreamFeed>`)

Terminal aesthetic:
- Container: `bg-gray-900 text-gray-100 font-mono text-xs` (dark terminal background; not pure black to soften)
- Container scrolls; container takes `flex-1 overflow-y-auto` so it fills viewport with sticky top filter bar
- Each event = single line, columns separated by spaces:
  ```
  HH:MM:SS.mmm  <tenant_short>  <agent_short>  <step_kind>  <status>  <snippet>
  ```
- Color rules:
  - timestamp: `text-gray-500`
  - tenant_id (first 8 chars + ellipsis): `text-cyan-400`
  - agent_short: `text-amber-400`
  - step_kind: `text-blue-300`
  - status: green for `success`, red for `error`, yellow for `out_of_scope`, gray for `pending` (use Tailwind `text-green-400` / `text-red-400` / `text-yellow-400` / `text-gray-400`)
  - snippet: `text-gray-300` (truncated at 80 chars with `…`)

### 2. Auto-scroll behavior

- When user is scrolled to bottom (`scrollTop + clientHeight >= scrollHeight - 8px`), new rows auto-scroll. Sticky-to-bottom behavior.
- When user scrolls UP (away from bottom), auto-scroll PAUSES. Show a small "↓ Resume tailing" floating button bottom-right that snaps back when clicked.
- Pause/resume button is sticky `fixed bottom-4 right-4 bg-gray-800 text-gray-100 px-3 py-1 rounded-md shadow-lg`

### 3. Click-to-expand

- Click a row → expand inline (below the row) showing:
  - input_envelope JSON pretty-printed (using existing `<JsonPretty>` from VT-234)
  - output_envelope JSON pretty-printed
  - tool_calls if present (model_id + tokens + tool_name)
- Expanded rows: `bg-gray-800 border-l-2 border-cyan-400 pl-2 py-2`
- Click again → collapse
- Track expanded state per-row in React state; survives new rows arriving above

### 4. Filter bar (top)

Sticky top. Reuse the existing `FilterPills` / filter state from `<StreamFeed>` — DO NOT reinvent the filter logic. Just restyle the bar to match the terminal aesthetic:
- Bar: `sticky top-0 bg-gray-800 border-b border-gray-700 px-4 py-2 flex flex-wrap gap-2 items-center`
- Each filter: `bg-gray-700 text-gray-100 px-2 py-1 rounded text-xs font-mono` with an `×` to clear
- "Filter:" label `text-gray-400 text-xs uppercase tracking-wide`
- Free-text search: small input `bg-gray-900 border border-gray-700 px-2 py-1 text-xs font-mono w-64 ml-auto`

### 5. Page wrapper at `apps/team-web/app/(app)/team/ops/stream/page.tsx`

Update to use the new terminal-style component:
- Page: `h-screen flex flex-col bg-gray-900` (full-viewport terminal feel)
- Header (above filter bar): `bg-gray-800 px-4 py-2 border-b border-gray-700` with `<h1 className="text-sm font-mono text-gray-300">Ops Console — Live Stream</h1>`
- StreamFeed component renders below header (terminal feed + filter bar)

### 6. Tenant + agent friendly-name resolution

For the `<tenant_short>` and `<agent_short>` columns, resolve UUIDs to short names:
- Tenant: first 8 chars of UUID. If `business_name` is available from `fetchTopTenants()` (already loaded server-side), prefer the short name. Else show `f4e3...`.
- Agent: agent registry name (e.g., `sr-agent`, `integration-agent`, `orchestrator`). If absent, show `unknown-agent`.

### 7. Canary `apps/team-orchestrator/canaries/vt238_stream_terminal.py`

3-4 assertions:
- A1: GET `/team/ops/stream` with operator JWT → 200 + response body contains `font-mono` + `bg-gray-900` (terminal sentinels)
- A2: Response body does NOT contain `bg-white rounded-lg shadow-sm` (card-style sentinels — confirms layout swapped)
- A3: Response contains the existing filter pill keywords (tenant, agent, step_kind, status, free-text input present)
- A4: SSR with seeded data renders ≥1 expected step_kind label
