# VT-683 — session-first owner comms (Fazal ruling, supersedes template-per-message plans)

## The ruling (verbatim intent)
Owner-facing template whitelist = **auth OTP (Twilio-handled) + welcome + wake-up/re-engage**.
Everything else happens ONLY inside the 24h conversation session after the owner responds.
Product loop: make the owner respond once a day → session opens → Viabe Team works for him →
until then, ALL owner communications QUEUE and deliver slowly/asynchronously at idle time
during the session. (Kills the need for hing/hi variants of weekly_approval / monthly_report /
draft_approval — those become in-session sends.)

## Design questions (brief BEFORE build)
1. **Session-window truth**: track last owner-inbound timestamp per tenant → `session_open()`
   predicate (24h). Where: tenants column vs derived from conversation_log (derive first —
   no migration; index exists?).
2. **Owner-comms queue**: table (tenant, kind, payload_ref, queued_at, sent_at, priority) —
   approval asks rank over reports; money/approval semantics unchanged (an approval ask that
   queues must not expire silently — VT-668 honest-expiry interplay).
3. **Idle-pace delivery**: drain trigger = owner-inbound turn completion + a scheduled sweep
   inside open sessions; pace (e.g. one queued item per N minutes, never mid-conversation-turn);
   never interleave into an active back-and-forth (idle detection = no inbound for X min within
   the session).
4. **Daily wake-up**: send the wake-up template ONLY when (a) session closed, (b) queue
   non-empty OR daily-brief due, (c) ≤1/day, (d) owner hours (IST). Copy carries the queued
   count (DB-backed — numeric honesty). Route en/hing/hi by owner_locale.
5. **Template-send chokepoint**: template sends outside the whitelist FAIL-CLOSED (registry
   enforcement — ties into VT-681 capability truth; the whitelist is a capability declaration).
6. **Existing senders migration**: weekly_approval / monthly_report / draft_approval template
   paths → queue-or-in-session sends. Inventory-first (who sends templates today, VT-632-style
   sweep).
