## What CoderC must produce
DLT (Distributed Ledger Technology) registration for India + Twilio sender ID approval + production WhatsApp Business sender configured. DLT is the Indian regulatory framework for entity-level + template-level registration with TRAI-approved registries (Vodafone Idea, Jio, Airtel, BSNL run separate registries — Phase 1 picks one, typically Vodafone Idea Smartping as the most-used Trueconnect registry).
1. **DLT entity registration**:
	- Fazal owns: registers [Viabe.ai](http://Viabe.ai) entity with chosen DLT registry (typically Vodafone Idea Smartping). Required documents: PAN, GST, business registration, authorized signatory ID. Process takes 5-10 business days.
	- Output: Entity ID assigned by DLT registry. Logged in `apps/team/config/dlt_registration.yaml` (sensitive; encrypted via Vercel env vars in production).
2. **DLT template registration**:
	- Each Meta-approved WhatsApp template (from VT-13.3) MUST also be registered with DLT registry under one of the standard categories: Promotional / Service Implicit / Service Explicit / Transactional.
	- This is in addition to Meta's category — the two systems use different taxonomies. Mapping:
		- Meta MARKETING → DLT Promotional
		- Meta UTILITY → DLT Service Implicit
		- Meta AUTHENTICATION → DLT Service Explicit (OTP messages)
	- Template registration: ~24-48 hours per registry; can be batch-submitted.
	- Each registered template gets a DLT Template ID (different from Meta SID); both stored in `apps/team/config/whatsapp_templates.yaml` registry from VT-13.3.
3. **Twilio Sender ID approval**:
	- Twilio's WhatsApp Business sender (production phone number) requires Twilio's review + Meta's approval.
	- Phase 1: Twilio number approved as Verified Business Sender; display name 'Viabe Team'; about page updated; profile photo set; business verification documents submitted.
	- Approval takes 5-15 business days from submission.
4. **Production cutover**:
	- Production sender ID configured in `TWILIO_WHATSAPP_NUMBER_PROD` env var.
	- Sandbox sender retained for development.
	- VT-5.6/5.7 reads sender ID from env; CodeX confirms.
5. **Compliance verification**:
	- Pre-launch test: send a Meta-approved + DLT-registered template to Fazal's personal phone via the production sender. Verify delivery + correct sender ID display.
	- DND (Do Not Disturb) rules: India's National Customer Preference Register (NCPR) applies to promotional SMS but NOT to WhatsApp Business as of 2026. Document at `docs/team/india-compliance.md`.
6. **Renewal cycle**:
	- DLT entity registration: typically annual renewal.
	- Each template: re-registration if content changes substantially.
	- Documented at `docs/runbooks/dlt-renewal.md`.
