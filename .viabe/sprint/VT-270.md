# VT-270 — classify_owner_message no-consent-gate gap (latent CL-390)

`classify_owner_message` send the raw inbound message body to Anthropic Haiku for
classification on EVERY inbound message, with NO `_owner_inputs_enabled` consent
gate in front (unlike the L0 writer, which gates on it). This is shipped code.

Per the VT-52 review (Cowork, 2026-06-01): a latent CL-390 issue — the transmit-
to-sub-processor path is not consent-gated. NOT in scope for the VT-52 build (no
scope-creep); filed here for Fazal to rule on whether classify must be gated the
same way the vision pipeline is (`_owner_inputs_enabled`, fail-closed).

Fix sketch (pending ruling): add `_owner_inputs_enabled(tenant_id)` check before
the classify transmit; fail-closed → skip classification rather than transmit.
