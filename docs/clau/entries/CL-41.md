# CL-41 — Three-repo architecture; marketing repo deferred to July

Three-repo architecture confirmed: rkecom-in/viabe-reports (Reports product), rkecom-in/viabe-team (Team product), rkecom-in/viabe-marketing (apex + company-level pages ONLY). Marketing repo creation DEFERRED to ~July 2026 before Team soft launch.

CORRECTION 2026-05-12 (later same evening): the prior framing was sloppy. Each product OWNS its landing page in its own repo — the landing pages are product-specific (Team's founding counter widget reads from tenants table in viabe-team-prod; Reports' landing has Reports pricing). They are not 'product-agnostic marketing'.

Correct scope:
- viabe-marketing: apex viabe.ai/, /about, /careers, /blog, /legal/*, /contact. Pure company-level marketing. No DB access. No product internals.
- viabe-team-web: /team (landing page with founding-counter widget), /team/dashboard, /team/ops, webhooks. Team owns its surface end-to-end including landing page.
- viabe-reports-web: same pattern for Reports.

Vercel apex routes via rewrites: viabe.ai/ → viabe-marketing; viabe.ai/about, /careers, /blog, /legal/* → viabe-marketing; viabe.ai/team and viabe.ai/team/* → viabe-team-web; viabe.ai/reports and viabe.ai/reports/* → viabe-reports-web.

July work scope (REDUCED from prior framing): create viabe-marketing repo, port apex homepage from viabe-reports into it, add /about/careers/blog/legal pages, configure Vercel rewrites. The Team landing page STAYS in viabe-team-web forever; no migration needed for it. Estimated 1-2 days instead of 2-3.
