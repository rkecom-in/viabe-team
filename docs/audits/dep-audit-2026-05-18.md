Cited at `agent/orchestrator_agent.py:14` (import) and `:61` (call). Works on pinned 1.2.0; `==` blocks an accidental V2 bump. **This is a call-site rewrite, not an import rename**: Context7 (`/websites/langchain_oss_python_langchain`) confirms `create_agent` shipped in langchain v1.0 (2025-10-20) with a different signature — `create_react_agent(model, tools, …)` → `create_agent(model=…, model_provider=…, tools=…, system_prompt=…, …)`; `model_provider` is a new required kwarg, kwarg order changed, `system_prompt` is first-class. The migration-PR brief MUST carry before/after for `orchestrator_agent.py:14,61`.
- **Python pin discipline — MEDIUM (CL-142)** — only the langgraph/langchain trio is `==`-pinned; `dbos` is `>=2.0`; the other 11 runtime deps are bare. Resolved by migration step 0.
- **UNUSED**: `supabase`, `pgvector`, `anthropic`, `langgraph-supervisor` — see UNUSED appendix for per-item positions.
- LOW: dbos, langchain-core, langchain-anthropic, langgraph-checkpoint-postgres, pydantic (uses `model_dump()` — current; deprecated `.dict()`/`.json()` not used), psycopg (psycopg3; no psycopg2 anywhere — verified), psycopg-pool, pyyaml, fastapi (no deprecated APIs used), uvicorn, twilio.
