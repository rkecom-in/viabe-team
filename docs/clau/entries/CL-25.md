# CL-25 — Two-stage event filtering: deterministic pre-filter then orchestrator brain

Stage 1 (deterministic, ~$0): regex/signature checks catch obvious opt-outs ('STOP', 'UNSUBSCRIBE'), DSR keyword matches, status pings, ack messages, template error webhooks. Routes directly to handler without LLM reasoning. Stage 2 (orchestrator brain, Opus 4.7): everything else. Orchestrator reasons over context bundle (its own memory + tenant L1-L4 + current trigger) and decides: spawn specialist / run pipeline / respond directly / listen-only. Cost model: pre-filter handles ~70% of events at zero LLM cost. Orchestrator brain on ~30% at Opus 4.7 cost (~₹3-6 per invocation, well under the ₹50 per-run ceiling). VT-49 classifier (currently spec'd as Haiku-backed intent classifier) gets absorbed: its job becomes part of the orchestrator's reasoning rather than a separate step.
