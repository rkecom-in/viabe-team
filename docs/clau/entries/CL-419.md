## Volume projection (RKeCom dev estimate)

- Above threshold for sustainable unit economics at non-enterprise pricing tiers

## Levers identified (ordered by cost-impact / implementation-cost ratio)

1. **Prompt caching (Anthropic `cache_control: {type: ephemeral}`)** — 90% discount on cached tokens; 5000-token system prompt cached → ~50 paise vs ~500 paise per turn. **10x cost reduction, no capability loss, no prompt rewrites.** Sprint 2 anchor.
2. **STUB tool inventory pruning** — 5 STUB tools (write_l0_fragment_stub etc.) carry full schemas ~300 tokens each. Drop until real wiring lands. ~1500 token reduction = ~30% input savings. Bundled with prompt caching work.
3. **System prompt token reduction** — move WHEN-to-write-L0 guidance to per-task injection rather than always-on. ~1000 token reduction. Bundled with prompt caching work.
4. **Mixed-model brain (Pattern A: Haiku default + Opus self-escalation)** — Haiku ~1/4 cost of Opus on input + output. Per-dispatch ~225 paise blended. **Compounding lever on top of caching.** Sprint 2.5 follow-up if caching alone doesn't reduce enough.
5. **Provider abstraction (OpenAI fallback)** — Sprint 3+; not built speculatively. Trigger: Anthropic outage / price hike / specific tenant where Anthropic underperforms.
