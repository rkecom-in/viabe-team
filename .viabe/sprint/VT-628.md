# VT-628 — the manager AND its ruler both sampled at temperature 1.0

The stocktake found the top gate blocker is not (only) the model's intelligence — it's that **intelligence
was being measured with a rubber ruler**. `temperature` was set on ZERO LLM calls in the orchestrator → both
the manager brain and the eval judge sampled at the API default 1.0. So the ±1-2 per-scenario swing ("variance")
is partly the manager being non-deterministic and partly the JUDGE scoring an identical transcript differently
run-to-run. Cannot chase a quality floor with a moving ruler.

## Family gate (verified live 2026-07-08 — ALL current models)
`claude-haiku-4-5` **ACCEPTS** temperature. `claude-sonnet-5` **DEPRECATES** it (400 "temperature is deprecated
for this model"). `claude-opus-4-7` AND `claude-opus-4-8` **DEPRECATE** it (400). So on the current lineup ONLY
haiku accepts the param. `src/orchestrator/llm_config.py::sampling_kwargs(model_id)` → `{"temperature": 0.0}`
for haiku, `{}` for everything else. EVERY touched call routes through the helper (no hardcoded temp) so a model
swap can never re-introduce a 400.

### FIRST-CUT BUG (fixed same session)
Cut 1 gated on `"opus"` only + hardcoded `temperature=0.0` on triage/review (both sonnet-5) — I'd trusted a
stale "sonnet accepts temperature" memory (true for sonnet-4-6, NOT sonnet-5). Deployed → every sonnet turn
(brain-complex, triage, review, judge) 400'd. Caught because the gate re-run produced 0 judged files (judge
400'd). Lesson: Rule #15 — canary EVERY target model live BEFORE shipping a sampling change; don't trust memory
of model capabilities across releases. Fix = haiku-only gate + live-canary of all 6 paths (PASS).

## What got pinned (haiku only → temperature=0)
- **Brain — routine turns** (`dispatch.py::_resolve_model` when tier=haiku). The sonnet-complex turns get `{}`
  (can't pin). Covers the VT-593/594 compose fallbacks that route through `_resolve_model`.
- **Intent classifier** (`classify_owner_message.py`, haiku) — kills the tier-flip non-determinism (the
  haiku-vs-sonnet brain choice is a pure lookup on this signal → deterministic classifier → deterministic tier).

## What CANNOT be pinned (documented limit — temperature deprecated)
- **Brain-complex, triage, review** (all sonnet-5) — non-deterministic; irreducible via temperature.
- **The 10 opus specialist lanes** (sales/marketing/finance/tech/cost/accounting/onboarding-conductor/
  integration/orchestrator-agent + field_mapping) — opus, non-deterministic on specialist CONTENT (SR campaign
  wording). Pinning needs a model change, not a param — stocktake step 6 (Fazal's cost call).
- **Net:** determinism-via-temperature only helps the routine-haiku path + the classifier. The variance thesis
  needs a different lever (N-sample, or model choice) for the capable turns. Honest downgrade of step 1's reach.

## The ruler (eval judge)
`transcript_judge.py` STAYS `claude-opus-4-8` (most discerning). It CAN'T be pinned (opus deprecates
temperature) and haiku is too weak a rubric-scorer to trust as the ruler. So ruler STABILITY is MEASURED
empirically (score one fixed transcript N times); if residual noise is material, go N-sample-median — NOT a
weaker deterministic model. No rebaseline vs the old opus-judged ~43% (same judge model).

## Target (Fazal 2026-07-08 — reframed from a flat 95%)
"A manager not on par to run my business, I won't trust." → the bar is a TRUST/ACCEPTANCE bar, two-tier:
- **Trust-breakers = 0 (100% clean):** fabrication, wrong/dropped money action, verbatim-repeat loop, ignoring
  the speech-act asked, promising the impossible. Measured as a violation COUNT, not an average.
- **Quality acceptance ≥ 90%** to ship as trustworthy (95% = excellent). Not a stochastic 95% average on a
  rubric that scores a proactive campaign-draft as "overreach".
