# VT-619 — agent-level LLM metering + config-driven limits

## Why (Fazal, 2026-07-07)
Heavy Sonnet-5/Opus-4.8 consumption will be too expensive for the SMB product. The
concern is **API-call count from the code** (measured ~3.2 LLM calls/owner-message,
opus-4-8 60% of calls at the $5/M tier), not per-session tokens. Cost must be metered
per tenant + per agent, capped, and shown on the tenant ops page.

## Pricing model (Fazal, decided this session)
- **No base fee** (a base fee makes tenants uncomfortable).
- **Per specialised agent = ₹5000/month** (e.g. Sales Recovery). ₹5000 auto-covers
  **onboarding + integration** (setup processes — tracked internally, NOT billed
  separately) and the **Team Manager** conversational cost.
- **Team Manager is not a separate line** — its consumption is **reserved within**
  each specialised agent's budget via per-call agent attribution (a manager turn that
  serves/routes to sales_recovery bills sales_recovery).
- Limit shown to the tenant as a **%**; a **top-up** raises the cap (resets the %).

## Model policy (agreed; rewire is the objective-loop's first step, NOT this row)
- **haiku-4-5 = default for all agents.**
- **sonnet-5 = extensive-reasoning only, single-shot (no iterate)** → note: single-shot
  precludes tool use, so tool-using hard turns stay on haiku iterating.
- **opus dropped.** GPT-5.4-mini rejected for now (Anthropic prompt-caching — already
  live, cached-haiku input ~$0.10/M — beats GPT-mini without a 2nd-vendor tax:
  OpenAI tool-use format, cache loss, DPDP data-processor surface). Metering is
  vendor-neutral, so GPT stays a cheap future config option.

## Economics → caps (haiku-default ~₹1.1/msg, 30% COGS)
₹5000 agent → ₹1500/mo model budget (cap) ≈ ~2× expected usage (~₹755/mo, 15%).
Converted to raw meter units (the hard, rate-independent enforcement unit):
**max_api_calls=4000, max_tokens_in=6.8M, max_tokens_out=2.1M** per agent/month
(conservative — derived from old opus/iterating traffic; real haiku usage lands under).
`agent_cost_limits` seed values are a **policy placeholder** — Fazal to confirm the
COGS % and the top-up price/mechanics.
