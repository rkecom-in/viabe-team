# CL-149 — Sarvam ships two Python SDKs

Surfaced during step 1 brief drafting (sarvamai PyPI search). The `sarvamai` package on PyPI is Sarvam's general-purpose SDK — STT (Saarika), TTS (Bulbul), Translate (Mayura), Chat (Sarvam-M). That's what step 1 wires in for VT-3.3 ingestion.

SEPARATE package: `sarvam-conv-ai-sdk` (released 2026-03-27). Conversational AI / voice agents (LiveKit + Pipecat integrations). Different surface area, different use case.

Phase 2 implication: when we build live voice agents (real-time call handling, in-conversation tool calls per the sarvam-conv-ai-sdk description), we'll add `sarvam-conv-ai-sdk` ALONGSIDE `sarvamai`, not replacing it.

Phase 1 scope: Phase 1 ingestion uses `sarvamai` only (batch STT/TTS, no live voice agent flows). `sarvam-conv-ai-sdk` is NOT added in Phase 1.

Logged here so this isn't rediscovered in Phase 2 planning. When voice agents come online: (a) add sarvam-conv-ai-sdk via the CL-147 new-dep gate, (b) decide pin style (likely `==` since it's a runtime SDK in fast iteration), (c) confirm LiveKit vs Pipecat integration choice based on rest-of-stack decisions at that time.

Not a blocker, not urgent. Reference entry only.
