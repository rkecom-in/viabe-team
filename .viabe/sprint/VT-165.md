# VT-165 — Test hardening — SR-agent end-to-end test should import model constant, not hardcode 'claude-opus-4-7'

## Scope (from brief 20260530T184226Z-brief-ready-e2e-batch, Wave A)

The SR-agent real end-to-end test (`test_sales_recovery_end_to_end.py`) hardcodes
the model id `'claude-opus-4-7'` in its proof-of-call assertion. Replace the
hardcoded literal with the canonical model id derived from the single source of
truth (`config/models.yaml` via `orchestrator.agent.sales_recovery._resolve_model`),
so the assertion can never drift from the configured production model.

Exact change site:
`apps/team-orchestrator/tests/orchestrator/test_sales_recovery_end_to_end.py:244`

```python
assert first_call["model"] == "claude-opus-4-7", diag
```

### Plan-divergence note (read before implementing)
The brief says "import the canonical model constant." There is **no** Python
string constant for the production model id today — the model id lives ONLY in
`config/models.yaml` and is read at runtime by `_resolve_model()`. It is hardcoded
as a bare literal in ~6 source files (`dispatch.py:112`, `integration_agent.py:70`,
`orchestrator_agent.py:93`, `orchestrator_agent_driver.py:111`,
`field_mapping.py:155`, and the `cost.RATES` keys). The minimal in-scope hardening
is therefore to make the TEST derive the expected id from the canonical resolver,
not to introduce a new module-level constant (that would widen scope into source).
The test already sets `VIABE_ENV=production` before the call, so
`_resolve_model("sales_recovery")` returns exactly the production model id.

Trivial. No migration. No canary (no external/SDK/persistence touched by the
change itself — the real Anthropic call already exists and is unchanged).

## Acceptance
- The literal `'claude-opus-4-7'` no longer appears in the proof-of-call assertion.
- The assertion compares `first_call["model"]` against `_resolve_model("sales_recovery")`
  (or an equivalent expected-model local bound from it).
- `grep -n "claude-opus-4-7" test_sales_recovery_end_to_end.py` returns nothing.
- Existing test semantics unchanged (still asserts production model == Opus when
  `VIABE_ENV=production`).
