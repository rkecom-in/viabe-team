# VT-211 — Integration Agent web onboarding page (deferred from VT-206)

## Why this row exists

VT-206 shipped the Integration Agent specialist + WhatsApp entry path (pre_filter → brain → spawn_integration). The `/team/onboard` server-rendered web page was deferred from VT-206's PR-1 scope to keep that PR focused. This row closes that scope.

## Scope

- `apps/team-web/app/(app)/team/onboard/page.tsx` — server-rendered page
  - Reads current `tenant_integration_state.phase` for Fazal's tenant
  - Renders the agent's current prompt + a single textarea form
  - POSTs owner answer to `/api/onboard/answer`
  - Redirects to refreshed page (or shows confirmation if phase = phase_5_confirmed)
- `apps/team-web/app/api/onboard/answer/route.ts` — internal API
  - Auth: same `requireFazal()` gate as `/team/ops`
  - Validates input
  - Calls orchestrator's `/api/orchestrator/integrations/onboard-step` endpoint (new, server-side only, X-Internal-Secret-guarded)
  - Returns redirect target
- `apps/team-orchestrator/src/orchestrator/api/onboard_step.py` — orchestrator endpoint
  - Invokes integration agent via supervisor with owner's answer
  - Persists phase transitions
  - Returns the agent's next prompt

## Acceptance criteria

1. Owner opens `/team/onboard` → sees agent's current prompt (or "All set" if confirmed)
2. Owner submits answer → POSTs to internal API → orchestrator invokes agent → page refreshes with next prompt
3. Resume across browser sessions: closing tab + reopening shows the same phase
4. Fazal-only auth: unauthenticated requests redirect to /login (VT-203 substrate)
5. Page renders <1s for any state

## Rule #15 Canary

`apps/team-web/tests/e2e/onboard.spec.ts` — Playwright spec (auth-mocked):

1. Unauth GET /team/onboard → redirect to /login
2. Auth GET → renders current phase prompt
3. POST answer → next phase prompt rendered
4. Close + reopen → same state
5. Final phase shows "All set" + tenant_config row populated

## Out of scope

- Mobile UI (desktop-only)
- Multi-tenant operator (Phase 2)
- File upload (sheet ID is text input; OAuth flow is link-out per VT-207)
