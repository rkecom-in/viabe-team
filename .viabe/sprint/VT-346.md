# VT-346 — pre-push hook: drop rls_ckpt_tester (rerun-clean)

The pre-push hook (`scripts/git-hooks/pre-push` ~L178) drops `app_role app_operator_role
rls_tester` before the orchestrator suite, but NOT `rls_ckpt_tester`. An RLS-checkpoint test
creates that CLUSTER role + objects; a SUBSEQUENT run's teardown `DROP ROLE rls_ckpt_tester`
fails (`DependentObjectsStillExist`) because a leftover `viabe_*` test DB still has objects
depending on it. A single clean-cluster run passes; the leak only bites across runs — so the
LOCAL hook false-fails (CI on a fresh runner is unaffected). Cost me a full diagnostic detour
this session (mis-read as a code break).

**Fix:** add `rls_ckpt_tester` to the hook's role-drop loop (+ ideally `DROP OWNED BY … CASCADE`
first, since it owns objects). Pure test-hook cleanup — does NOT touch the RLS isolation /
SET-ROLE path (no subagent needed). See memory `rls-ckpt-tester-hook-leak`.
