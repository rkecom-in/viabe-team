# VT-182 — Anthropic Messages SDK observability callback

## Scope

Per design doc §3.3 (updated for the 2026 SDK + CL-249 direct-Messages-SDK pattern + CL-56 Logfire backend):

1. **Callback hook** that fires once per agent think/act/observe loop iteration, writing one `agent_reasoning_step` record via `write_step()` (VT-180).
2. **Captured fields per design doc §2.3 envelope:** context_bundle_hash, context_bundle_components, context_bundle_token_count, prior_tool_calls_count, prior_tool_calls_summary (input); think_text_redacted, action, action_target, action_args_summary (output); decision_rationale; tokens_input/output; model_used; cost_paise.
3. **PII redaction:** `think_text_redacted` flows through VT-104 redactor before persistence. Phone numbers → tokens.
4. **2026 SDK instrumentation surfaces:** capture prompt caching usage + defer_loading metadata (per VT-122 body's 2026-05-16 updates).
5. **Logfire span correlation:** the agent_reasoning_step record carries the Logfire trace_id so Ops UI can cross-link.

## Dependencies

- VT-180 (writer)
- VT-179 (envelope schemas)
- VT-104 (PII redactor)
- VT-171 (Logfire backend for trace_id correlation)
- VT-32 / VT-33 (Anthropic SDK skeleton + system prompt — both Done)

## Acceptance criteria

- Real Anthropic call in a synthetic agent run → exactly one agent_reasoning_step row per loop iteration.
- think_text_redacted has phone-token tokens, not raw digits.
- Logfire trace_id captured + queryable.
- Cost + tokens per step match Anthropic's response usage block.
- Real Anthropic canary < ₹1.
