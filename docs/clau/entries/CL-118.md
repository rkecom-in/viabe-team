# CL-118 — Briefs delivered as single copyable fenced block

Fazal 2026-05-18: all Claude Code briefs must be delivered as a single copyable code block (markdown fenced, no inline commentary interrupting the brief, no split sections). Reason: copy-paste friction — multi-section briefs with prose between them require scrollback hunting.

DISCIPLINE: when drafting a Claude Code brief, write any preamble/stress-test/clarifying questions BEFORE the brief, then deliver the brief itself as one fenced \`markdown block from title to discipline reminders. Any post-ship commentary goes AFTER the block.

Applies to: every Claude Code brief, every fix-PR brief, every audit-driven brief. Does not apply to ad-hoc code snippets shown for discussion (those stay inline as normal).
