# VT-269 — sub-processor consent for customer-PII transmission (DPDP)

Surfaced during VT-52 (vision extraction) plan review. The vision pipeline sends
the owner's CUSTOMERS' PII (names/phones on a ledger photo) to Anthropic (a
sub-processor). VT-52 builds + canaries on the `_owner_inputs_enabled(tenant_id)`
gate (owner-level consent, CL-342/390) against SYNTHETIC data only (CL-422).

**Open question (Fazal's call — privacy/legal authority):** does owner-level
consent cover the owner's customers' PII to a sub-processor under DPDP, or is a
DISTINCT flag / notice required before the vision path is enabled against REAL
customer data?

This is a PRODUCTION-ENABLEMENT gate, NOT a dev-build blocker. VT-52 ships +
canaries on synthetic data regardless. The vision path does NOT process real
customer data until Fazal rules here. Cowork escalated to Fazal 2026-06-01.
