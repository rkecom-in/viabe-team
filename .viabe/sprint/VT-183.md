# VT-183 — LangGraph node hooks

## Scope

Per design doc §3.1:

1. **Entry/exit hooks** on every LangGraph node that wrap node execution and call `write_step(step_kind='state_transition', ...)` automatically.
2. **Retrofit existing nodes:** pre-filter gate, supervisor, sales_recovery node, collapse, orchestrator_terminal — all get the hook wired.
3. **No-engineer-touches-write-path-manually:** the hook IS the integration. Adding a new node = inheriting the hook automatically.
4. **CI gate (paired with VT-186):** new LangGraph node without the hook = build fails. (Actual gate implementation lives in VT-186; this row sets up the convention + retrofits existing nodes.)

- VT-180 (writer)
- VT-179 (envelope schemas — `state_transition` schema)

## Acceptance criteria

- mypy --strict clean.
