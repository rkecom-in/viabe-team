# VT-285 — POS connect ease (OAuth-connect / upload fallback)

Connector-audit ease follow-up on VT-58 (CL-427). VT-58 ingests a KOT/POS export
the owner uploads; this prefers a real OAuth CONNECT where the POS vendor exposes
one (PetPooja / Posist / Slick etc — owner approves, zero-paste per CL-421), with
the file-upload path as the fallback for vendors without an API.

Scope (to design): per-vendor OAuth connect (where available) → auto-pull POS
exports; else fall back to VT-58's upload + header-sniff parser. Each vendor that
exposes OAuth must pass the connector audit (owner-only-approves). Backlog — ease,
not a launch-gate; the upload path (VT-58) already works.
