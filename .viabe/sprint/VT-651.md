`campaigns.plan_json target_cohort.cohort_size` came out **5, 3, 5** across 3 identical-seed runs; the owner
was honestly told "5"/"3"/"5" customers. So:
- **NOT a Tier-1 trust-breaker** — the blind tier_rescore judge scored 0 breakers + Tier-2 100% all 3 runs
  (the agent states the number it actually plans; no fabrication, no wrong-count-to-owner).
- **IS a real coverage + consistency defect** — a revenue-recovery win-back should reach ALL eligible lapsed
  customers (8), deterministically. Under-targeting 3-5 of 8, VARYING per run on a deterministic seed, means
  lost recovery + non-reproducible cohorts.
