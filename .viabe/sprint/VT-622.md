# VT-622 ‚Äî repair orphaned tool_use before a lane's model call (kill the dual-spawn 400)

## Root cause (proven from the deployed 400 message)
A dual-intent owner message ("connect my Sheets AND set up a win-back") makes the orchestrator
emit TWO handoff `tool_use` blocks in ONE assistant turn (`spawn_sales_recovery` +
`spawn_integration`). LangGraph executes the first handoff's `Command(goto=...)` and routes
away BEFORE the second `tool_use` receives its `ToolMessage`. The target lane's `create_agent`
then replays the parent history carrying an UNPAIRED `tool_use` ‚Üí the next Anthropic call:
```
messages.2: `tool_use` ids were found without `tool_result` blocks immediately after:
toolu_01Dtmv‚Ä¶  Each `tool_use` block must have a corresponding `tool_result` block ‚Ä¶
```
‚Üí `BadRequestError` in the integration lane ‚Üí VT-602 catches it ‚Üí the whole turn degrades to a
human `escalated` (observed: routing_dual escalates, reply never mentions "sheet").

Why SR survives: `_sales_recovery_node` runs from a clean `sales_recovery_context` bundle, not a
`create_agent` replay of parent messages. The `create_agent` lanes (integration, marketing, tech,
finance, cost_opt, accounting, onboarding_conductor) all share this latent bug.

Message-level analogue of VT-484 (`_tool_error_to_tool_result`) ‚Äî but VT-484 only covers tools
that RAISE; spawn handoffs RETURN a `Command` (never raise), so VT-484 does not fire.

## Fix (landed to dev)
`agent/tool_pairing.py`: `repair_orphaned_tool_use(messages)` inserts a synthetic error
`tool_result` immediately after the `AIMessage` for any `tool_use` id with no following
`ToolMessage`; **identity no-op when the conversation is already valid**, so it can never degrade a
healthy turn. Exposed as `repair_tool_pairs_before_model` (`@wrap_model_call` middleware), wired
into the integration lane's `create_agent(middleware=[‚Ä¶])`. Edits the MODEL INPUT only
(`request.override(messages=‚Ä¶)`), never persisted state. 5 unit tests (valid‚Üíidentity;
dual-spawn‚Üírepaired; multi-orphan; all-orphan; no-calls‚Üíidentity).
