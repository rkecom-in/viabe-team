# VT-264 — narrow query_customer_ledger's UndefinedColumn tolerance

From the VT-262/263 adversarial review (dropped-low). VT-257 broadened
query_customer_ledger's graceful-empty `except` to swallow UndefinedColumn (for
the forward-target customers.phone_token gap). That `except` wraps BOTH SELECTs +
result construction, so a future REAL column typo/logic bug in either query would
be silently masked as an empty result. Narrow it: scope the swallow to the
specific forward-target queries (or assert the expected missing column name), so
only the known gap is tolerated. Post-Sprint-2. NOT launch-gating.
