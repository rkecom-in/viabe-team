## Bug (CC flag during VT-382, twice; pre-existing, untouched there)

`send_whatsapp_template.py`: 'error' is in `_IDEMPOTENT_HIT_STATUSES`, and agent drafts
use the FIXED idempotency key `agent:{draft_id}` — so a TRANSIENTLY failed agent send
(Twilio 5xx, network blip) registers as an idempotent hit and is unretryable until the
24h key TTL expires. A retry within the window silently no-ops.

## Contract (when dispatched)

Either drop 'error' from the idempotent-hit set for retry-classed failures (terminal
Twilio errors stay idempotent-hit), or salt the retry attempt into the key. Gate-module
file → heightened care: zero behavior change to the success/dup paths; real-DB test for
transient-fail → retry-succeeds within the window + terminal-fail stays deduped.
