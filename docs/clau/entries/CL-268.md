# CL-268 — DECISION (Type 2): draft_message_variants is DEFERRED — not in v1. v1 LLM-backed

## The question

Should the Sales Recovery Agent have an LLM-backed tool (draft_message_variants) that generates alternative phrasings of campaign messages, in v1?

## Decision: Defer to Phase 1.5

draft_message_variants is NOT in v1. The v1 LLM-backed tool set is exactly two tools, both with hard consumers: self_evaluate (gates final output) and classify_owner_message (classifies inbound owner messages).

## Rationale

1. The agent already produces message_plan (template_id + template_params) inside CampaignPlan. That IS the v1 messaging contract. Nothing in Phase 1 consumes alternative phrasings.
2. Variant generation with no production data is generating noise — there is no signal to rank variants against until real campaigns have run. The capability wants production data first; that makes it a Phase 1.5 concern by nature.
3. It would be a third Opus-backed tool: more cost, more Pillar 2 surface to justify, for zero Phase 1 consumers.
4. Deferring keeps the v1 LLM-backed set minimal and every tool in it has a hard consumer.

## Include condition (not met)

Would only be included if there were a Phase 1 owner-facing reason the agent must offer message choices at launch. No such reason identified.

## Scope impact

- Does NOT block PR #36 (VT-39 tools framework) — the framework is decision-agnostic by design.
- VT-5.13 status-set is a separate ~2-tool-call follow-up after #36 merges: mark draft_message_variants deferred in the tool registry / status set.
- docs/team/llm-backed-tools-rationale.md must list only self_evaluate and classify_owner_message as locked, with draft_message_variants carrying a 'deferred — Phase 1.5' note (not a placeholder pending decision anymore — the decision is now made).
