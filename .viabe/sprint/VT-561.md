# VT-561 — correction store as trainable pairs (mig 160)

## Problems (audit findings)
- On reject/edit, `record_correction` keeps only the owner's prose; `redact_batch_close` sha256-destroys the proposal in the same txn ⇒ (proposal → verdict) pair broken.
- Clean approvals only bump `clean_approval_streak` — no labeled positive examples.
- `apply_agent_decision`/`record_correction` write `run_id=NULL` — chain not joinable.

## Fix
Migration 160: `agent_corrections` += `proposal_snapshot JSONB NULL` (PII-REDACTED draft template+params/body), `corrected_snapshot JSONB NULL`, `outcome TEXT NULL` (Phase-2 back-annotation), widen `correction_kind` CHECK += `'approve'`. Snapshot the (redacted) proposal at `record_correction` time BEFORE `redact_batch_close`; record approve-as-is as a positive lesson row alongside `record_approval_outcome`; thread `run_id` through `apply_agent_decision → record_correction`. Same redactor as `correction_text` (pii_redactor, never sha256). DSR purge already covers the table.
