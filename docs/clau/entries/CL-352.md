# CL-352 — BLOCKER (VT-150 / PR #47) — product-code defect, not just the test: the @DBOS.sc

BLOCKER (VT-150 / PR #47) — product-code defect, not just the test: the @DBOS.scheduled decorator on purge_workflow_inputs_scheduled runs at MODULE IMPORT of orchestrator.dbos_purge. main.py deliberately imports that module inside the lifespan (not from launch_dbos) specifically to keep the @DBOS.scheduled registration OFF keyless test fixtures — so app_version (DBOSRegistry.compute_app_version) stays stable for cross-process recovery match. But test_dbos_purge_substrate.py and test_dbos_purge_unit.py import the SAME module to reach purge_terminal_workflow_inputs, so the decorator fires for them too. The main.py isolation design does NOT hold — the registration side-effect is welded to the module, not the lifespan import. ALSO: dbos_purge.py module docstring claims the scheduler is 'wired through dbos_config.launch_dbos' — directly contradicts main.py, which does the opposite by design. Stale/wrong docstring. FIX (product code): make registration explicit — define purge_workflow_inputs_scheduled UNDECORATED; apply DBOS.scheduled(_PURGE_CRON)(fn) inside a register_purge_scheduler() function called from the main.py lifespan. Importing the module for its functions must NOT register. Correct the docstring. THIRD DEFECT — CI: the orchestrator job (ci.yml) provisions pg16 and runs pytest tests/orchestrator/ -v, but sets DATABASE_URL / INTERNAL_API_SECRET / TEAM_PHONE_HASH_SALT and NOT RUN_INTEGRATION_TESTS=1. test_dbos_purge_substrate.py gates on RUN_INTEGRATION_TESTS — so the 4 purge substrate tests SKIP in CI. Claude Code's 'PR #47 12 gates green / 162 passed' was green because the substrate tests were among 135 skipped. Corrects my earlier guess (CI does run substrate tests — but the env gate skips these four). Fold RUN_INTEGRATION_TESTS=1 into the orchestrator job (Clau-owned CI-config call) so the fix is actually gated. PR #47 HELD: fix registration leak + docstring + test clock race (prior blocker 368387c2-cc5a-816c) + CI env var, then re-run.
