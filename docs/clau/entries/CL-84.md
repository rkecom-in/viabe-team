# CL-84 — Create VT-3.3c: minimal handler-side Twilio send wiring

NEXT ACTION (Fazal 2026-05-18 in response to audit C4 / D-3.8.1-3 from CL-74 + CL-77): create Notion subtask VT-3.3c, scope minimal.

SCOPE:
- New helper: apps/team-orchestrator/src/orchestrator/utils/twilio_send.py with `send_template_message(tenant_id: UUID, template_name: str, params: dict) -> SendResult`. Wraps the Twilio SDK (already in pyproject), uses TEAM_TWILIO_* env vars from orchestrator/.env.example.
- Templates needed (4): opt_out_confirmation, dsr_acknowledgment, status_callback_failure_notice, status_ping_response. Maybe 5 if template_error_handler's escalation needs its own template.
- Wire each of the 5 existing direct handlers to call send_template_message. Remove TODO VT-3.3 comments. Update return contracts (no more lying `confirmation_sent: True` — actual send result).
- Wrapped in @DBOS.step so a send failure is durably retried via DBOS.
- Pillar 7 contract: handler returns send_result with status. No padding, no lies.
- Tests: each handler exercises the real send path with a mocked Twilio client. Plus one integration test against Twilio sandbox if practical (else dev-only).

NOT IN SCOPE (deferred to VT-5):
- General-purpose MCP outbound tools (send_whatsapp_message, send_whatsapp_template, etc.) that specialists call. VT-5 owns the broader MCP tool surface.
- Template content management (lives in Twilio Console for Phase 1).
- Outbound rate limiting (Twilio enforces; not our problem at this layer).

DEPENDENCIES:
- Schedule AFTER VT-3.4 (langgraph_supervisor wiring) so the orchestrator-agent's tool surface and the handler-send surface evolve consistently.
- Twilio DLT approval not blocking (VT-13.6); template send works against the trial sandbox.

PILLAR 7 IMPLICATION: until VT-3.3c ships, opt_out / dsr / status_ping flows are visibly broken at the user-facing contract level (handler returns lie). The audit's C4 finding doesn't fully resolve until VT-3.3c is Done.

ACCEPTANCE:
- 4-5 templates send via Twilio SDK.
- All 5 handlers' TODO VT-3.3 comments removed.
- Handler return contracts truthful (send_sent: actual bool from Twilio response, not hardcoded True).
- Tests cover send-success, send-failure-retry, handler-return-truthfulness.
- Notion subtask body explicit: scope is templates only, not the MCP tool surface.

Fazal to create the Notion VT-3.3c subtask under the VT-Orchestrator parent.
