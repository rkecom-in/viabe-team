# VT-298 — Autonomous watchdog → immediate Telegram VTR alerting

## Why
Fazal: "If [watchdog] found [an issue] report to VTR on Telegram immediately." A crash /
stall / misbehaviour must reach the assigned VTR without them watching a board.

## What
- Watchdog detectors (crash / stall / misbehaviour) → severity classification → IMMEDIATE
  Telegram alert to the **assigned** VTR.
- Detectors shared with the VT-296 board (board = visual; this = push).
- Routing uses the VTR↔business assignment (VT-295) to pick the recipient.

## Acceptance criteria
- A detector fires (sandbox/mock) → severity assigned → Telegram alert dispatched to the
  ASSIGNED VTR only (not unassigned VTRs).
- Alert path exercised in canary (sandbox/mock Telegram acceptable until bot token lands — DR-15, flag if needed).
- No PII in the alert payload (CL-426 / CL-390).

## Out of scope
- The board UI (VT-296); the interactive bot (VT-297).

Build deferred behind launch queue. Pairs with VT-296 + VT-297. One PR (CL-424).
