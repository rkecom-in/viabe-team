# CL-252 — DECISION: admin-bypass merge (gh pr merge --admin) is the standing merge method 

Fazal confirmed 2026-05-20: --admin bypass merge is intentional policy FOR NOW, rationale = solo developer, no other contributors, so branch-protection required-review rules have no second human to satisfy. SCOPE OF THE POLICY: --admin bypasses branch-protection rules (e.g. required reviewers) only. It does NOT license merging on a red/failing CI gate — CI green remains a hard precondition for every merge; the canary (where applicable) remains a hard precondition. The bypass is about the review rule being unsatisfiable with one human, not about skipping checks. REVISIT TRIGGER: this policy is Phase-1 / solo-dev specific. If a second developer joins, or branch protection is reconfigured, this decision is superseded and normal review-gated merge resumes. Logged so sessions apply it consistently and Claude Code does not re-infer 'per precedent' without a referent.
