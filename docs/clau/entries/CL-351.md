# CL-351 — BLOCKER (VT-150 / PR #47): test_old_terminal_rows_are_purged FAILS on live PG — 

BLOCKER (VT-150 / PR #47): test_old_terminal_rows_are_purged FAILS on live PG — assert cutoff_ms < now_ms (1779475738089 < 1779475736706, cutoff ~1.4s in the FUTURE relative to test's now_ms). Root cause: test captures now_ms once at top, then seeds 4 rows across a live Supabase pooler (~1.4s network I/O), then purge computes its own later 'now'; with retention monkeypatched to 1s — SHORTER than the test's own setup latency — cutoff lands after the stale now_ms. TEST bug, not product bug: the cutoff_ms<now_ms assertion is incidental and proves nothing; the test dies on it before ever asserting the 4 old rows were deleted. Fix (Quality/Test Engineer, test file only): drop the cutoff<now assertion OR re-snapshot now_ms after seeding; set retention to a stable value well above setup latency, not 1s. Product code stands. Substrate tests require RUN_INTEGRATION_TESTS=1 + live PG; near-certain CI does NOT provision these, so the substrate suite is in the 135 SKIPPED, not the 162 passed. CI green badge on PR #47 does NOT cover the substrate layer. Dead-canary pattern recurrence (cf CL-272, #39/#40 thread). Standing implication: substrate-layer correctness on any PR cannot be inferred from CI green — must be run locally against PG or CI must provision PG. PR #47 HELD from merge until test fixed AND re-run green.
