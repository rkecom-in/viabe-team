# VT-208 — Shopify connector

## Why this row exists

Shopify is the highest-impact digital source for the SMB tier above RKeCom. Ships AFTER Sheet so the framework (VT-205/206/209/210) is proven on the simpler connector first. Shopify validates that the framework handles real e-commerce data: orders, customers, abandoned checkouts, products viewed.

## Scope

### Auth flow

- Shopify Admin API access token (custom app pattern — owner generates token in Shopify admin → pastes into Integration Agent)
- Agent walks owner through: Shopify admin → Apps → Develop apps → Create app → API scopes → Install → copy token
- Token stored encrypted per-tenant (re-use Fernet substrate)

### Data sources from Shopify

- `customers` — name, email, phone, total_spent, orders_count
- `abandoned_checkouts` — drop-off equivalent; cart_value, line_items, customer info, abandoned_at
- `orders` (for attribution post-Sales-Recovery — was the customer recovered?)
- `products` (for "what was in the dropped cart")

### Adapter contract (same as Sheet)

`apps/team-orchestrator/src/orchestrator/integrations/connectors/shopify.py`:
- `start_setup`, `complete_setup`, `pull_sample`, `pull_full`, `setup_push`
- Push via Shopify webhooks: `checkouts/create`, `checkouts/update`, `orders/create`, `orders/paid`

### Canonical schema mapping

Shopify `customers` → our `customers`. Shopify `abandoned_checkouts` → our `drop_offs`. Tagged `acquired_via='shopify'`.

## Acceptance criteria

1. Owner pastes Shopify Admin API token; agent validates token works + has required scopes
2. `pull_sample` returns first 50 customers + 50 abandoned checkouts
3. Webhooks register correctly; orchestrator receives + processes checkouts/abandoned events
4. Order paid webhook triggers attribution: matches against past Sales Recovery campaigns; updates outcome
5. Dedupe against existing Sheet customers via phone_hash (VT-209)

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt208_shopify_connector.py` — 5 assertions: token validation, sample pull (customers + abandoned), webhook receipt, attribution match, dedupe.
