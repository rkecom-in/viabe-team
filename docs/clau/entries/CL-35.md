# CL-35 — REVERSE: Phase 1 will NOT ship with checkpointer-only durability

REVERSAL of CL-26 (Phase 1 checkpointer-only durability posture). Fazal cited the Viabe Reports CrewAI lesson: 'we discussed and started with crewai, thinking we will port to Langgraph after 10 customers, but we have got so deep into crewai implementation that porting to Langgraph will be a nightmare.' Same pattern applies here: deferring durable execution infrastructure to Phase 2 will create the same migration nightmare. If checkpointer-only is wrong at 200 subscribers, it's wrong at 10. Need to evaluate Temporal vs DBOS vs other durable execution options NOW and commit to one for Phase 1, accepting the upfront complexity. Pending architectural decision after Clau presents the comparison.
