# VT-557 — manager_task retry ladder → dead_letter → redrive (B6)

## Why
`backoff.compute_delay()` existed with ZERO callers; manager_tasks had no attempt/dead-letter concept — a repeatedly-stalled task parked at `blocked` forever needing a human. Wire the deterministic backoff into the reaper as a BOUNDED retry ladder + add a real `dead_letter` terminal + an operator redrive.

## Built
- **mig-158** `manager_tasks += attempt / max_attempts(5) / next_retry_at` + status CHECK gains `dead_letter` + a retry-gate partial index.
- **`manager/task_retry.py`** (pure, dep-less) — `decide_retry(attempt, max_attempts)` reusing `backoff.compute_delay` (ONE backoff policy, not two): RETRY (attempt+1 + backoff delay) until the budget hits `max_attempts` → DEAD_LETTER. Curve asserted exactly with injected `rand`.
- **Reaper (`orphan_reaper.reap_stalled_manager_tasks`)** now applies the ladder per stalled task: RETRY → `blocked` + `attempt+1` + `next_retry_at` (the reaper skips a task until its backoff elapses) → orphaned_task alert (VT-529, unchanged for the common single-stall); budget-exhausted → `dead_letter` (operator-redrivable, never auto-retried) → **dead_letter_task** alert (new). Best-effort, service-role, never raises.
- **`task_store.redrive_task()`** — CAS `dead_letter|blocked → planned`, attempt=0, next_retry_at cleared; `dead_letter` added to `TASK_TERMINAL`.
- **API** `POST /api/orchestrator/ops/run-control/redrive-task` — tenant DERIVED from the task row (VT-293/294 IDOR); transport auth verified BEFORE the derive (anti-probe, the cancel-override precedent); require_vtr_action assignment gate; audit-in-txn; 409 if not redrivable.
- **triggers.py** += `dead_letter_task` (warning).

## Verify
ruff clean; 5 pure retry-ladder tests (base/exponential-curve/exhaustion/jitter-bounds/clamp); 15 task_store realdb (6 new: attempt+backoff recorded, exhaustion→dead_letter+alert, backoff-gate skips, redrive resets, redrive no-op on completed) — existing single-stall reaper + orphaned_task tests still green; 14 DSR purge; 111 run-control suite green; ops_run_control imports with the redrive-task route.

## Note (honest scope)
The reaper is the failure DETECTOR — the coordinator sweep dispatches plan-items, not manager_tasks, so there is no separate task executor to auto-re-run. The ladder records bounded attempts + backoff across a task's life and escalates chronic stalls to `dead_letter`; the operator redrive is the human re-driver. No fabricated auto-executor.
