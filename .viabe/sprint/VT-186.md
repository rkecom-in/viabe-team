# VT-186 — CI gates for observability discipline

## Scope

Per design doc §3.1 + §3.2 + §2.2 — 3 structural CI gates that prevent observability-discipline regression:

1. **`gate-langgraph-nodes-have-observability-hook`** — grep all LangGraph node definitions; assert each is wrapped by VT-183's hook (or explicitly opt-out with a documented annotation). Fails build if drift.
2. **`gate-mcp-tools-have-observability-decorator`** — grep all `agent/tools/*.py` functions; assert each is wrapped by `@observability.tool_step` (per VT-181). Fails build if drift.
3. **`gate-envelope-schema-registered`** — at startup OR at CI: assert every step_kind string referenced in source has a corresponding Pydantic envelope registered (per VT-179). Drift = boot failure (which CI catches).

Pattern-match to existing gates: `gate-no-langsmith-imports` (VT-171), `gate-no-llm-in-deterministic-triggers` (VT-28 + extended in VT-175/VT-30).

## Dependencies

- VT-181 (decorator exists)
- VT-183 (LangGraph hooks exist)
- VT-179 (envelope schemas registered)

## Acceptance criteria

- All 3 gates added to `.github/workflows/ci.yml`
- Negative tests: each gate fails on an intentional violation (test in PR)
- Green on current main + all of VT-178..VT-185 once those merge
