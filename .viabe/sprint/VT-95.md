## What CoderC must produce
The public landing page at `viabe.ai/team`. Bilingual (Hindi + English) with language toggle. Hero, value prop, pricing tiers, founding indicator widget, FAQ, footer. SSR + edge-cached. Lighthouse >90.
1. **Page structure** at `apps/team-web/app/team/page.tsx`:
	- Hero: short headline + subheadline + primary CTA ("Start free trial" or "Join waitlist" depending on VT-11.3 mode).
	- Value-prop section: 3 cards explaining what Viabe Team does (Sales Recovery Agent, WhatsApp ledger, refund-if-no-results promise).
	- Pricing tiers: 3 cards (Founding ₹2,499 / Standard ₹4,999 / Pro ₹14,999) with feature lists; founding card highlighted with live counter widget (VT-11.5 — referenced here, implemented there).
	- Day-39 promise: dedicated section explaining the 2x ARRR refund commitment (Pillar 7 honesty).
	- Social proof section (VT-11.4 placeholder).
	- FAQ: 8-10 questions Fazal pre-authors.
	- Footer: links to privacy policy, DPDPA disclosure, contact, terms.
2. **Bilingual + i18n**:
	- Default language detection: browser `Accept-Language` header. Fallback English.
	- Manual toggle in header: EN \| HI.
	- All copy in `apps/team-web/locales/team-landing/{en,hi}.json`. CodeX rejects hardcoded strings.
	- Hindi script via `Noto Sans Devanagari` web font.
3. **SSR + edge cache**:
	- Server-side rendered for SEO + first-paint.
	- Static generation for the bulk of content; founding counter widget hydrates client-side via `/api/team/founding-status`.
	- CDN cache 5 minutes; founding counter widget refreshes every 60 seconds via client-side fetch.
4. **Performance budget**:
	- Lighthouse mobile: Performance >90, Accessibility >95, Best Practices >95, SEO >95.
	- Largest contentful paint <2.5s on 3G.
	- Total bundle size <250KB gzipped.
	- Images: all WebP/AVIF with fallbacks; lazy-loaded below fold.
5. **Accessibility**:
	- WCAG 2.1 AA: keyboard navigation, screen-reader labels, sufficient contrast, no auto-playing media.
	- Reduced-motion respected.
6. **Analytics**:
	- Cookie-free analytics (Plausible or similar; documented decision in `docs/team/landing-analytics.md`).
	- Tracks page views, language toggle clicks, CTA clicks, scroll depth. NO PII.
	- Source attribution via UTM params; passed to sign-up form (VT-11.2) as `referral_source`.
7. **Tests** at `apps/team-web/__tests__/team-landing/*.test.tsx`:
	- Page renders in EN and HI locales.
	- Language toggle persists via cookie or localStorage.
	- Founding counter widget displays correctly when API returns various states (under cap, near cap, at cap).
	- Lighthouse CI gate (synthetic): all 4 scores meet thresholds.
	- Accessibility: axe-core synthetic run passes WCAG 2.1 AA.
	- Hindi script renders correctly (Devanagari font loaded).
	- SSR: page HTML returned without JS execution.
