# CL-240 — DECISION: VT-29 scoped to wrap VT-3.3 webhook only; VT-3.5 scheduled-trigger wir

Claude Code VT-29 pre-flight (Step 1) found VT-3.5 (scheduled triggers) is NOT on main: grep across apps/team-orchestrator/ for DBOS.scheduled / crontab / cron_string / @DBOS.scheduled returns 0 matches. trigger_reason Literal carries weekly_cadence as a valid value but no upstream producer exists; in-code comments (agent_graph_state.py:34, orchestrator_agent.py:70, handoffs.py CL-195) confirm VT-3.3/VT-3.5/VT-3.8 producer wiring is explicitly later-PR work. VT-3.3 IS on main; VT-3.5 is not.

DECISION (Clau, owns subtask sequencing + brief scope): VT-29 proceeds NOW, scoped to wrap the VT-3.3 webhook handler only. The VT-3.5 scheduled-trigger wiring is NOT deferred-as-TODO inside VT-29 — it moves to VT-3.5's own scope. Rationale: VT-29's VT-3.5 dependency is a single wiring seam at one edge (1 of 5 build items); the framework itself (failures.py, strategies.py, error_router.py, backoff, circuit breaker, two-layer model, tests, docs) has ZERO dependency on scheduled triggers. You wire a handler when you build the handler — so VT-3.5's brief, when it lands, will include 'wrap handlers in the VT-29 error framework'. Blocking all of VT-29 on VT-3.5 (Option 1/3) would block ~90% dependency-clean work to avoid stubbing ~10%; that misreads a leaf-edge dependency as a root one.

UNBLOCKS VT-35: VT-35 needs the agent_hard_limit_breach failure TYPE (defined in failures.py), not the scheduled-trigger wiring. VT-29-scoped-to-VT-3.3 ships failures.py in full, so VT-35 is fully unblocked by this VT-29. Option 1 would have delayed VT-35 behind VT-3.5 for no dependency reason.

VT-29 brief updated: Step 2 item 5 wiring scoped to VT-3.3 only; VT-3.5 wiring removed from VT-29 scope and reassigned to VT-3.5. failures.py / strategies.py / error_router.py / tests / docs unchanged — full build.

DEPENDENCY CHAIN NOW: VT-29 (VT-3.3-scoped, buildable now) → unblocks VT-35 (hard-limit enforcement, fully specified, Backlog). VT-3.5 (scheduled triggers) is independent and carries its own error-framework wiring when built.
