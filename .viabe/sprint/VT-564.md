# VT-564 — delivery-callback reconciliation (mig 161)

## Problem (audit finding, CONFIRMED)
`customer_send.py:864-921` flips draft→'sent' + writes `agent_customer_contacts` + a `send_result` audit on transport ACCEPTANCE (message_sid), not delivery. A later `failed`/`undelivered` status callback is routed (`pre_filter_gate.py:176` — failed only, undelivered not at all) to `template_error_handler`, which never reconciles the failed message back to the draft/contact/audit rows. Claimed effect ≠ evidence of effect.

## Fix
Migration 161: durable delivery status on the customer-send ledger (e.g. `agent_customer_contacts.delivery_status` + timestamp). On failed/undelivered callback: resolve message_sid → draft/contact row, mark delivery-failed (distinct from send-accepted), emit a `send_result` status-update tm_audit, fire the reviewer alert (reuse the VT-534 outbound_failure pattern). Thread `run_id` into the customer `send_result` emits where resolvable (audit finding: currently NULL). Owner-facing messaging stays out of scope (VT-562 owns owner reporting).
