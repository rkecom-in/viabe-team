# CL-198 — VT-3.4 PR 2/3 brief inline-rescope resolved

## 2. §3.8 collapse path — STOP-condition fired

## 3. §4.1 + §4.3 — substrate-tests contradiction
Apply option (3c): monkeypatched bundle plumbing tests + defensive tenant_id cross-tenant assertion.

Monkeypatch `_build_business_profile` and other `_build_*` per-section helpers to return distinct non-empty data for tenant A vs tenant B. The constructor's plumbing (dispatcher routing, mixed empty/non-empty integration, data_completeness flag flipping) gets tested without requiring real L1/L2 substrate.

Defensive cross-tenant test: with safe-empty fallbacks active, assert the bundle's tenant_id field always matches input. Catches accidental tenant_id leakage in the safe-empty path. ~10 lines.

Keep `assert_tenant_scoped` as a pure unit test (rescued from §4.3).

Brief inline comment per CL-183: 'VERIFICATION TARGET: bundle plumbing tested via monkeypatched _build_* functions; real-substrate test deferred to VT-7.1-bundling PR.' Makes the limitation explicit instead of pretending we tested integration we didn't.

Reasoning for (3c) over (3a) cut-everything: cutting plumbing tests means PR 2/3 ships untested dispatch + data_completeness logic. First bug surfaces when VT-7.1 lands — exactly when we don't want to be debugging plumbing. Monkeypatch tests cover OUR code; future integration test covers the boundary. Different layers, both needed.

## 4. §3.9 trigger_reason source
Apply (4a) with stop-or-1-line-extend rule.

Brief §3.9 becomes:
1. Verify AgentGraphState has trigger_reason field (set by upstream init).
2. If yes: spawn_sales_recovery tool reads it via InjectedState and passes to build_sales_recovery_context.
3. If no AND adding it is a trivial 1-line state-schema extension: add it in this PR.
4. If no AND it requires more than 1 line (e.g., upstream wiring changes): STOP AND REPORT. Likely a scope conflict with VT-3.5 or VT-3.3 territory.

Keeps PR 2/3 from quietly absorbing upstream work; gives Claude Code permission to handle the trivial case inline.
