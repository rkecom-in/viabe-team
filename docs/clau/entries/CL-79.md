# CL-79 — Delta audit on VT-3.3b — 12 additional findings

Delta audit on VT-3.3b (commit 4106e29). VT-3.3b ships team-web webhook route + Twilio signature verification + rate limiting + tenant lookup. But VT-3.3b's actual architecture decision shifted from the audit-prompt summary: tenant lookup + rate limiting moved INTO the orchestrator (Pillar 8: single DB-access path), not split into team-web. team-web only does signature verification + forwarding. This was a defensible design call but isn't documented as a Decision in Clau_Session_Log.

DELTA CRITICALS:

DC1 (CRITICAL): orchestrator-client.ts forwards request body as `{ twilio_fields }` (no tenant_id), and the orchestrator's TwilioIngressBody Pydantic model at 277723c required `{ tenant_id, twilio_fields }`. Verified: the orchestrator at 4106e29 was correspondingly refactored to drop tenant_id from the request body (model has only twilio_fields). So the two sides are consistent at 4106e29 but the architectural shift wasn't logged.

DC2 (CRITICAL): env-var naming inconsistency. team-web uses TWILIO_AUTH_TOKEN (no TEAM_ prefix). orchestrator uses TEAM_TWILIO_AUTH_TOKEN. Same secret, two names. Lint rule explicitly whitelists TWILIO_AUTH_TOKEN in team-web. Operational footgun: Fazal must set both names in Railway with identical values; rotations must happen atomically. Convention break worth a Decision entry or rename.

DC3 (HIGH): _lookup_tenant in twilio_ingress.py opens a cross-tenant SELECT against tenants.whatsapp_number. By design service-role; bypasses RLS by necessity. But: no index on tenants.whatsapp_number, no UNIQUE constraint. Full-table scan on hot path. With ORDER BY created_at DESC LIMIT 1 ('most recent wins'), silent data loss if two tenants share a number. Add index + decide on UNIQUE.

DC4 (HIGH): _lookup_tenant runs BEFORE rate limit check. An attacker with INTERNAL_API_SECRET can probe arbitrary phone numbers as From values; unknown senders short-circuit before rate limit. Phone-number enumeration risk if INTERNAL_API_SECRET leaks. Add a workspace-wide rate limit BEFORE tenant lookup, keyed on a request-source identifier.

DC5 (HIGH): C1 (RLS/GUC) from prior audit got worse — 4106e29 adds 4 more uncovered call sites (_lookup_tenant, _bump_bucket x2, _within_rate_limits). _lookup_tenant legitimately needs service-role (cross-tenant). Rate-limit writes still need the GUC pattern.

DC6 (MEDIUM): rate_limit_buckets has no FK on tenant_id (documented in migration 013 as deliberate for the workspace sentinel UUID). Acceptable tradeoff but means typos in writes go uncaught.

DC7 (MEDIUM): _within_rate_limits increments bucket counts BEFORE checking. Rejected requests still consume bucket count. Cosmetic.

DC8 (MEDIUM): when per-tenant check passes but workspace check fails, both buckets got bumped. Tenant pays for workspace overflow. Minor accounting issue.

DC9 (MEDIUM): Twilio signature verification has no test that exercises the real verifyTwilioSignature with a non-null signature against actual env-var-set auth token. Tests mock the function. Env var typo or rename would not be caught by tests — only production failure.

DC10 (LOW): synthetic_webhook.py uses port 8000; everything else uses 8001. Doc/test consistency.

DC11 (LOW): C2, C3 from prior audit still present at 4106e29 (lost-message ordering, empty MessageSid collapse). Unchanged.

DC12 (LOW): H1 from prior audit got more acute — twilio_ingress.py is now significantly larger (tenant lookup, rate limit) but still missing from no-LLM CI grep.

NEW VALIDATIONS PASSED (VT-3.3b specific):
- Twilio SDK used for signature verification, not bespoke crypto.
- TwiML returned correctly (text/xml, empty Response).
- 5s timeout on orchestrator forward via AbortSignal.
- Pillar 7 holds: orchestrator non-200 / timeout / unknown_sender / rate_limit_exceeded all still return 200 TwiML.
- Rate limit tests assert behavior correctly under contrived seeds.
- New unknown_sender test covers the new path.
- Lint rule extended with WEB_ENV_WHITELIST for legitimate Next.js server-side route handler vars.
- CONVENTIONS.md documents the env exception.
