# VT-590 — the manager's final message is the owner reply

## Finding (deployed-dev, post-VT-589)
VT-589 transmits the trailing AIMessage, but the manager's trailing text was internal narration ("Let me proceed with the direct manager reply to keep momentum on the setup") or a half-answer ("Good question — totally fair to ask before moving forward"), because the prompt told it to reply via compose_owner_output_tool (which shapes canned text + isn't sent) — so its own final message was just reasoning wrap-up.

## Fix (prompt, orchestrator_agent_system.md — Type-1)
Handle-directly section + a top-level hard rule: the manager's FINAL message IS the WhatsApp reply, sent verbatim — write ONLY the finished owner-facing message, no meta-narration / half-sentences / empty turns; answer the question (honest 'I don't have that specific' + next step when no fact); if mid-onboarding, answer AND guide back. compose_owner_output_tool is explicitly NOT the send path for a conversational reply. Validated EMPIRICALLY on deployed dev (read the replies).
