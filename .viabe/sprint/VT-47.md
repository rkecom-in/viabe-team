## What CC must produce
The `request_owner_approval` pause primitive. It (a) sends the approval template via VT-45, (b) persists a `pending_approvals` row, (c) PAUSES the LangGraph run via `langgraph.types.interrupt()` (checkpointer-backed), (d) returns a typed pause result. Resume is driven by the Twilio inbound webhook (classify reply via VT-49 -> resolve decision -> `Command(resume=...)`). A timeout sweep (5th scheduled trigger) resolves stale pauses.

1. **Pause primitive** at `apps/team-orchestrator/src/orchestrator/agent/tools/request_owner_approval.py`:
   - `MCPTool` subclass; Pydantic IO (frozen). Input: `approval_type` (Literal campaign_send|cohort_size_exceeded|sensitive_data_access|other), `summary` (<=500), `details: dict`, `template_id`, `template_params: dict[str,str]`, `language: Literal['en','hi']='en'`, `timeout_hours: int = 48` (max 168). Output: `status` (paused|error), `approval_id: UUID`, `error_envelope`.
   - Order of effects (idempotent): (1) send template via VT-45 — if it errors, return ErrorEnvelope and write NO pending_approvals row; (2) INSERT pending_approvals (decision NULL); (3) call `interrupt({approval_id, approval_type, summary})` to halt the graph.
   - The tool does NOT poll. `interrupt()` raises `GraphInterrupt`; dispatch catches it and returns a typed pause `DispatchResult` so the DBOS workflow exits cleanly (not a timeout/error).

2. **Migration `052_pending_approvals.sql`** (assigned, CL-424): `id UUID PK`, `tenant_id UUID`, `run_id UUID` (FK pipeline_runs), `approval_type text`, `summary text`, `details JSONB`, `requested_at timestamptz`, `timeout_at timestamptz`, `resolved_at timestamptz NULL`, `decision text NULL` (approved|rejected|needs_changes|timeout), `owner_message_sid text`. RLS `tenant_isolation` via `app_current_tenant()` (CL-82/88, migration `000b` helper). Path-first per CL-81.

3. **Resume path** (in the Twilio inbound webhook, `api/twilio_ingress.py`): on inbound owner message, look up the open `pending_approvals` for the sender's tenant/run; run VT-49 `classify_owner_message`; map approval->approved / rejection->rejected / question|feedback->needs_changes; UPDATE the row (`decision`, `resolved_at`); resume the graph with `Command(resume={decision})` on the same `thread_id`.

4. **Timeout sweep — 5th DBOS scheduled trigger in `scheduled_triggers.py`** (CL-240, NOT a parallel poller): cron `*/30 * * * *`; scans `pending_approvals WHERE timeout_at <= now() AND resolved_at IS NULL`; resolves with `decision='timeout'` and resumes the affected run via `Command(resume={'decision':'timeout'})`. Body callable with injected `now` for the canary (mirrors the existing 4 bodies).

5. **Tenant scoping**: `ctx.tenant_id` on insert; checkpoint RLS already keys `thread_id == run_id` -> `pipeline_runs.tenant_id` (graph.py:_setup_checkpoint_rls). Cross-tenant resume impossible by RLS.

6. **CL-390 (no PII)**: log `approval_id` + `tenant_id` + `approval_type` + `decision` only — never message body, never owner phone. `owner_message_sid` is a SID, allowed.

## Architectural rules enforced
- Pillar 1: pause primitive only; resume lives in the webhook + sweep. No polling inside the tool.
- Pillar 7: owner approval is authoritative; no code path auto-approves on the agent's behalf. `timeout` is an explicit non-approval terminal decision.
- Pause is durable: PostgresSaver checkpoint survives process restart; resume re-enters at the interrupting node (LangGraph re-executes the node from its start on resume — see plan risk R3).
- CL-175: handoff/resume uses `langgraph.types.Command` directly (no langgraph_supervisor).

## Acceptance criteria
- Tool subclasses `MCPTool`; registered in `tool_registry`.
- Migration `052_pending_approvals.sql` applies cleanly; RLS scoped via `app_current_tenant()`.
- Happy path: tool sends template (VT-45 dry-run), persists row, raises interrupt; dispatch returns a typed pause result.
- Resume tested via synthetic inbound (classify reply -> resolve -> `Command(resume)`); agent run completes.
- Timeout sweep tested with injected clock: 49h-old pending -> `decision='timeout'` + resume.
- Multiple concurrent approvals: resolving one doesn't affect the other.
- Cross-tenant: tenant_A cannot resolve tenant_B's pending approval (RLS).
- Template-send failure -> ErrorEnvelope, no orphan pending_approvals row.
- Checkpoint resume works across process restart (canary kills + reloads substrate mid-pause).
- mypy --strict + ruff pass.

## Canary (Rule #15) — real pause->resume cycle on a seeded synthetic run

Fail-not-skip if the pause never fires or resume doesn't complete. Separate sweep canary: insert a synthetic timeout_at-in-the-past row, run the sweep body, assert `decision='timeout'`.
