# VT-198 — Owner feedback surfaces (3 tiers, coexisting)

## Why this row exists

Fazal directive 2026-05-27 (after Cowork proposal): "Agree with your recommendations and proposal and coexistence" — all three feedback tiers coexist. Without explicit owner signal, the agent can't learn what's working in the owner's specific business. Implicit attribution (customer outcomes) is necessary but not sufficient — it doesn't distinguish "this campaign worked because the agent was clever" from "this campaign worked despite the agent making the wrong call."

## Scope (3 tiers)

### Tier 1 — Implicit attribution (already substrate; surface for context bundle)

VT-156 + VT-164 capture campaign success/fail per customer. Aggregate into Day-39 evaluator (VT-197). No new code surface needed beyond ensuring Day-39 reads the attribution data.

### Tier 2 — Emoji reactions on outbound (lightweight density signal)

- Twilio WhatsApp supports emoji reactions on messages (`message_status` callbacks carry `MessageStatus=reaction` events with `ReactionEmoji` field)
- Capture in twilio_ingress + write to `owner_feedback_reactions` table (new migration)
- Per-message context: which outbound campaign / step / customer the reaction was on
- Agent reads reaction-density signal as a context bundle field

### Tier 3 — Dashboard review prompts (explicit, weighty)

- Weekly review surface in Ops Console: "agent recommended campaign X for these 12 customers — accept / modify / reject"
- Free-form reply: "no, leave Mrs. Sharma alone for now" → Cowork interprets + writes to L1 `owner_curated_context`
- Decision record: each weighty signal stored with timestamp + context + owner verdict

## Acceptance criteria

1. Twilio reaction webhook captured + stored in `owner_feedback_reactions` (Tier 2)
2. Dashboard review surface renders weekly with N pending review items per tenant (Tier 3 UI)
3. Free-form reply parser writes to L1 owner_curated_context (Tier 3 NLP-light)
4. Agent context bundle includes both reaction density + L1 explicit-review fields (consumes by VT-195's Context Composer)
5. Privacy audit log: explicit consent in Twilio webhook handling for reaction data

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt198_owner_feedback.py` — 6 assertions covering each tier:
1. Implicit: attribution row exists for synthetic campaign outcome
2. Tier 2: synthetic reaction webhook lands as `owner_feedback_reactions` row
3. Tier 3 UI: dashboard query returns weekly review list
4. Tier 3 reply: free-form text → L1 update applied
5. Context bundle merges all 3 tiers
6. RLS isolation: tenant A's feedback not visible to tenant B

## Out of scope

- Automatic agent-behavior changes based on feedback (just CARRY into context; don't auto-tune)
- Telegram review surface (Twilio + Ops Console only this row)
- AI-summarization of free-form replies (per CL-390 vendor consent locks)

## Dependencies

- VT-195 (L1 substrate — Tier 3 free-form writes go to L1 owner_curated_context)
- VT-194 (cost optimization — Tier 3 dashboard reviews may show agent decision rationale which is cached)
- Twilio account configuration for reaction webhooks (Fazal-side setup)
