# CL-32 — LangGraph supervisor library chosen for orchestrator's specialist-spawning pattern

Use LangGraph's langgraph_supervisor library (Python, MIT-licensed, ~57 code snippets, Context7 ID /langchain-ai/langgraph-supervisor-py) for the orchestrator's specialist-spawning pattern. The supervisor library auto-generates handoff tools per specialist (e.g. assign_to_sales_recovery_agent, eventually assign_to_reputation_agent, etc.), manages message history between agents, and integrates natively with LangGraph state graphs. Custom handoff tools via create_handoff_tool() let us name them whatever we want (e.g. spawn_sales_recovery instead of assign_to_sales_recovery). Configurable: add_handoff_messages=False to control history pollution, handoff_tool_prefix= to set naming. Phase 1 has exactly one specialist; the supervisor pattern still applies because Phase 2-6 will register additional specialists via the same mechanism. Refactor to manual implementation only if we hit a library limit.
