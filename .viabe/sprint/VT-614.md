# VT-614 — stop the manager fabricating the business's own identity

## Root cause (VT-611 read-only trace, 2026-07-07)
The owner reply is free LLM text from `dispatch_brain`, sent verbatim. Business identity is **never injected as
authoritative structured context**, so the model fills the vacuum with plausible fabrications:
- `business_type` degrades to the literal string `"(unknown)"` (`knowledge/business_context.py:298`).
- **city/region, platform, store-domain have NO structured block at all** — `_read_identity` (`business_context.py:214`)
  doesn't even SELECT locality; no L1 city field.
- **Smoking gun:** `agent/dispatch.py:327` `_build_onboarding_state_block` literally injects
  *"it looks like yourstore.myshopify.com"* into the manager's own context → the fabricated Shopify domain is a **copy
  of that placeholder**, not independent invention. Same literal in `prompts/integration_agent_system.md:35,48`.
- Owner-stated facts ("Verma Kirana, a provisions store") are **never persisted** (the conversational path never calls
  `upsert_business_profile` — only apify/ingestion do) → they scroll out of the 24h/≤20-turn window → the identity block
  says `"(unknown)"` → the model re-invents (→ "sweets in Chennai").
- The only honesty guard (`orchestrator_agent_system.md:33`) is scoped to *"a specific fact (e.g. exact pricing)"* — it
  never forbids asserting the business's OWN type/location/platform.

Observed: `onboarding_longhaul_resume_no_reask` (context_retention 1 — "sweets in Chennai" vs stated Verma Kirana),
`onboarding_privacy_skeptic_asks_why` (honesty 2 — invented `yourstore.myshopify.com` for a hardware shop).

## Fix (plan-first; prompt + context injection, no send-path)
1. **Prompt honesty rule** (`orchestrator_agent_system.md`): broaden the guard — never assert the business's OWN
   type/category/city/platform/store-address/name unless it is in context or the owner stated it; a `"(unknown)"` field
   is genuinely unknown → ask or omit, never guess a plausible specific. **[highest lever, prompt-only]**
2. **Kill the copyable placeholder** (`dispatch.py:327`): reword so `<name>.myshopify.com` is unmistakably a FORMAT
   example, never a value to state. Same for `integration_agent_system.md:35,48`.
3. **Frame `(unknown)` as do-not-fabricate** + surface locality/platform where present (`business_context.py`
   `_read_identity`/`render_business_context_block`).
4. (Deeper, separate row) persist owner-stated identity from chat into `tenants.business_type` / L1 so it durably
   anchors past the 24h window.
