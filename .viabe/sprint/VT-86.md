## What CoderC must produce
The monthly impact report generator: aggregates the previous month's metrics for a tenant, renders a PDF, emails it via Resend. Triggered by VT-3.5's monthly impact scheduled trigger on the 1st of each month.
1. **Report generator module** at `apps/team/src/owner_surface/monthly_report.py`:
	- `generate_monthly_report(tenant_id, year_month: str) -> MonthlyReport`. `year_month` format `YYYY-MM` for the reporting period.
	- Aggregates from canonical tables (NOT through the agent; deterministic SQL):
		- Campaign summary: count proposed/approved/sent/cancelled in the month.
		- ARRR for the month: sum of attributed transactions for campaigns whose attribution closed in the month.
		- Customer ledger growth: count of new customers added vs prior month.
		- Top 5 campaigns by ARRR contribution (ranked).
		- Approval-decision split: % approved / % rejected / % needs-changes / % defer.
		- Subscription cost: fees paid in the month.
		- Net value: ARRR − fees.
2. **PDF rendering** at `apps/team/src/owner_surface/monthly_report_pdf.py`:
	- Uses a separate PDF library (e.g., `weasyprint` or `reportlab`) — NOT shared with Reports' `pdf_generator.py` (different product, different brand).
	- Template at `apps/team/templates/monthly_report.html` + CSS.
	- Renders:
		- Header: business name + month + year.
		- Hero metric: ARRR for the month (large, color-coded — green if positive, gray if zero, NOT red).
		- Summary cards: campaigns sent, customers reached, attribution rate.
		- Top 5 campaigns table.
		- Approval-decision pie chart (matplotlib or similar; static image embedded).
		- Subscription summary: fees paid + net value.
		- Footer: data residency + DPDPA disclosure + contact link.
3. **Honesty in low-activity months**:
	- If ARRR for the month is 0: report says explicitly "This month had no attributed revenue. Possible reasons: no campaigns sent (check approval-decision split), low attribution rate, ingestion gap."
	- If campaigns approved < 2: report flags the low engagement.
	- No glossing-over. CodeX rejects PRs that hide failure modes.
4. **Email delivery via Resend**:
	- From address: `noreply@viabe.ai` (DMARC-verified).
	- Subject: "Your Viabe Team Impact Report — \[Month Year\]".
	- Body (Hindi/English/mixed per tenant.preferred_language): brief 2-paragraph summary + PDF attachment + portal link.
	- Failure handling: Resend bounce → retry once after 1 hour via VT-3.5 rescheduled trigger; second failure → Telegram alert to Fazal.
5. **Tenant skip conditions**:
	- Tenant signed up <30 days ago: skip (no full month of data; matches VT-3.5's skip rule).
	- Tenant in `cancelled` or `refunded` phase: skip.
	- Tenant in `onboarding` or `trial`: send anyway with honest framing ("You're in trial; here's what happened.")
6. **Storage**:
	- PDF stored in S3 / Supabase Storage with retention 12 months. Owner can re-download from portal (VT-9.7) at any time.
	- `monthly_reports` table (migration `031_monthly_reports.sql`): `id`, `tenant_id`, `year_month`, `pdf_storage_url`, `arrr_paise`, `fees_paid_paise`, `net_value_paise`, `generated_at`, `email_sent_at NULL`, `email_failure_count`. Tenant-scoped RLS.
7. **Tests** at `apps/team/tests/owner_surface/test_monthly_report.py`:
	- Synthetic tenant with full month of activity → PDF rendered with all sections; metrics match SQL aggregates.
	- Zero-ARRR month: report shows honest framing, no glossing.
	- Trial-phase tenant: report sent with appropriate framing.
	- <30-day-old tenant: skipped.
	- Refunded tenant: skipped.
	- Resend bounce: retried; 2nd failure alerts Fazal.
	- PDF storage: retrieved 12 months later; older deleted.
	- Cross-tenant: tenant_A's report cannot be accessed via tenant_B's tenant_id.
	- Hindi rendering: tenant with `preferred_language='hi'` gets Hindi-language body + section labels.
## Architectural rules to enforce in this subtask
- Pillar 7: honesty in low-activity reporting. CodeX rejects PRs that omit failure context.
- Pillar 8: Reports' PDF generator NOT reused; this is a separate module for a separate product. CodeX confirms via import-graph.
- Aggregations are deterministic SQL. CodeX rejects agent invocations from this path.
- PDF rendering uses `weasyprint` (or chosen lib) pinned in pyproject.toml.
- Resend is the email delivery; alternative providers are Type 2 governance.
- 12-month PDF retention is a Phase 1 commitment; Phase 2 may extend on owner request.
- Hindi content respects script + numerals (Devanagari numerals optional, Latin acceptable).
