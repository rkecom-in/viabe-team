## BUILT 2026-06-04 (dev-queue row 1) — pre-push fail-loud on empty/degenerate range
The hook silently `exit 0`'d (PASSED) when the push range computed an empty change-set → unverified code
through (the heredoc-consumed-stdin bug we hit). Fix in `scripts/git-hooks/pre-push`: empty `CHANGED` now (1)
recovers via the HEAD-vs-origin/main diff; (2) if even that is empty while HEAD is ahead of origin/main →
**FAIL LOUD (exit 1)**; (3) genuine no-op (HEAD not ahead) still passes cleanly. `PREPUSH_RANGE_ONLY` dry-run
short-circuits after range resolution for the regression test. Test `scripts/test_prepush_range.sh`: source
guard + degenerate-range recovery + empty-only fail-loud (exit 1) + genuine-no-op — ALL PASS. Done first so
every subsequent dev-queue row's pre-push gate is trustworthy.

# VT-316 — pre-push hook fail-loud on empty ref-range

## Problem (observed in the VT-73 push, 2026-06-04)
The local pre-push hook (`scripts/git-hooks/pre-push`) produced NO output on a
BACKGROUNDED `git ... commit <<'EOF' ... && git push` compound command — yet the
push succeeded. The hook silently no-op'd: it computes changed files from the
ref-range on stdin; the heredoc / background execution apparently left the hook's
ref-range stdin empty, so `changed_match` found nothing → `ORCH_CHANGED=0` → every
check block (ruff / smoke / team-web / DB jobs) was skipped → `FAIL=0` → exit 0.

CLAUDE.md makes the local pre-push hook THE safety gate (CI is the non-blocking
backstop). **A gate that can silently skip all checks undermines that.** VT-73 was
pushed unverified by the hook (compensated by a manual full-suite run — 919
passed — + the CI orchestrator job; no code impact, but the gate failed its job).

## Fix
The hook must FAIL-LOUD (or run the full orchestrator job anyway) when the
ref-range computes empty/missing on a REAL push (a non-zero local_sha with commits
ahead of the remote). Never silently pass. Options:
- If `RANGE_BASE..local_sha` yields zero changed files BUT `local_sha` is a real
  commit not on the remote → fall back to diffing against `origin/main` (or run
  the full suite unconditionally), rather than treating "no changed files" as
  "nothing to check".
- Distinguish a genuine no-op (pushing an already-pushed ref) from a stdin/range
  failure; on the latter, `err "could not determine push range — running full
  suite"` + run, or abort.

## Acceptance criteria
- A push where the ref-range computation fails/empties but real orchestrator
  changes exist → the hook RUNS the suite (or aborts loudly), never silently passes.
- A genuine no-op push (no new commits) still short-circuits cleanly.
- Verified by simulating the empty-ref-range stdin case.
