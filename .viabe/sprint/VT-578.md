# VT-578 — the manager owns the month plan

## Flow (the ruling, made concrete)
1. Agent self-readiness: before planning, a specialist checks VT-577 `readiness(agent)` — not ready ⇒ it states what's missing (drives the integration asks), never emits a hollow plan.
2. Specialist composes its plan freely (existing sales_recovery reasoning loop + VT-573 tool belt).
3. MANAGER VALIDATION: the Team Manager brain reviews the proposal against tenant objective, data readiness, memory/lessons (VT-566 read-back), cost/risk → accept / revise-request / escalate. Reasoning recorded (tm_audit decides + provenance).
4. ACCUMULATION: accepted plans compose into the tenant's managed month-plan (durable, versioned, owner-visible).
5. MANAGEMENT: the manager mints manager_tasks/steps (VT-565 spine) for plan items and tracks execution through the existing ladder/redrive machinery.
6. Guardrails unchanged: owner approval for consequential actions, consent/send/money rails, budget caps. Freedom in thinking; determinism at effects.

## Depends
VT-577 registry (readiness), VT-565 spine, VT-566 memory read-back. Build after VT-576/577 land.
