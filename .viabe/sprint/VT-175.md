## Scope

### 1. Migration `023_attributions_and_cadence_columns.sql`

```sql
-- attributions table
CREATE TABLE attributions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    campaign_id UUID NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
    customer_id UUID NULL,  -- nullable until VT-170 customers table ships
    razorpay_payment_id TEXT NULL,
    attributed_paise BIGINT NOT NULL CHECK (attributed_paise >= 0),
    attribution_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_attributions_tenant_campaign ON attributions (tenant_id, campaign_id);
CREATE INDEX idx_attributions_attribution_at ON attributions (attribution_at);

-- RLS: tenant-scoped per app.tenant_id GUC (CL-82 Standing)
ALTER TABLE attributions ENABLE ROW LEVEL SECURITY;
CREATE POLICY attributions_tenant_isolation ON attributions
    USING (tenant_id = current_setting('app.tenant_id')::uuid);

-- cadence columns on campaigns
ALTER TABLE campaigns
    ADD COLUMN attribution_close_at TIMESTAMPTZ NULL,
    ADD COLUMN attribution_closed_at TIMESTAMPTZ NULL,
    ADD COLUMN total_arrr_paise BIGINT NULL CHECK (total_arrr_paise IS NULL OR total_arrr_paise >= 0);

-- cadence column on tenants
ALTER TABLE tenants
    ADD COLUMN paid_conversion_at TIMESTAMPTZ NULL;
```

### 2. Day-39 evaluator at `apps/team-orchestrator/src/orchestrator/billing/day39_evaluator.py`

**DETERMINISTIC SQL ONLY. ZERO LLM CALLS.** Enforced by the `gate-no-llm-in-deterministic-triggers` CI gate.

```python
def evaluate_day39(tenant_id: UUID, conn: Connection) -> Day39Verdict:
    """
    For a tenant whose paid_conversion_at + 39 days <= now():
    - Aggregate ARRR from attributions table for the tenant's attribution window
    - Aggregate cumulative fees from subscriptions / payment_events
    - Verdict: 'continue' if ARRR >= 2x cumulative_fees; else 'refund_triggered'

    Returns Day39Verdict with verdict, arrr_paise, cumulative_fees_paise, decided_at.
    No LLM. No agent invocation. Pure SQL aggregation + Python comparison.
    """
```

### 3. Attribution-close aggregator at `apps/team-orchestrator/src/orchestrator/billing/attribution_close.py`

**DETERMINISTIC SQL ONLY.**

```python
def close_attribution(campaign_id: UUID, conn: Connection) -> AttributionCloseResult:
    """
    For a campaign whose attribution_close_at <= now() AND status='sent' AND attribution_closed_at IS NULL:
    - SUM(attributed_paise) FROM attributions WHERE campaign_id = $1
    - UPDATE campaigns SET total_arrr_paise = <sum>, attribution_closed_at = now() WHERE id = $1
    - Emit pipeline_log row with event_type='attribution_closed'
    """
```
