# CL-67 — Dev testing architecture: 3-tier

Three-tier dev testing architecture confirmed for Phase 1:

## Tier 1 — CI (every PR)
Unit + integration tests via pytest + vitest against pgvector/pgvector:pg16 Postgres in GitHub Actions. Already working. Catches schema bugs, function-level errors, RLS violations, DBOS workflow correctness, type errors. No changes needed.

## Tier 2 — Local synthetic webhook fixtures (after each meaningful PR merge)
Script-based testing of the orchestrator pipeline without external dependencies. Script constructs Twilio-shaped HTTP POST payloads and fires them at a running local orchestrator. Tests webhook receive → signature verification (against test secret) → pre_filter routing → direct_handler execution → response logging.

Decision: Tier 2 setup gets added to VT-3.3 scope. Trivial addition (~30 min brief expansion). Brief drafting for VT-3.3 must include:
- apps/team-orchestrator/scripts/synthetic_webhook.py — script that fires Twilio-shaped POST to local orchestrator with configurable body, sender, message_type
- Documentation in README or docs/team/dev-testing.md on how to run
- Test scenarios: opt-out keyword, DSR keyword, status callback, status ping, substantive message → brain stub

## Tier 3 — Live Twilio sandbox via ngrok (after VT-3.4 ships)
Real end-to-end testing with a real WhatsApp message reaching the orchestrator via webhook. Components:
- Orchestrator running locally
- ngrok tunnel exposing port 8000 (free tier; static subdomain ~$10/month optional)
- Twilio sandbox configured to hit ngrok URL
- Fazal + 1-2 trusted testers text the sandbox number

Decision: Set up AFTER VT-3.4 (langgraph_supervisor wiring) ships. Reason: setting up Tier 3 before VT-3.4 means watching a webhook arrive and immediately error because downstream components don't exist. Wasted setup time.

Not covered by Tier 3 (Phase 2+ concern): production-like staging environment with separate Supabase project + separate Twilio number. Phase 1 soft launch IS our staging (10 design partners as friendly testers).
