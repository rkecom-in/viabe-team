# VT-281 — PII encrypted from the VTR (de-identified view)

Agent-operation-layer row from CL-426 (VTR standing decision, Fazal 2026-06-01).
Fazal locked: **"customer PII information encrypted so the VTR cannot see it."**

Enforce that the VTR sees de-identified / business-level data ONLY — never raw
customer names / phones. Mechanism: RLS + crypto so the VTR role cannot decrypt
customer PII (a distinct role from owner/app_role). This removes the human-PII-
access surface, which is WHY no separate human-access consent is needed beyond
CL-425 — so the enforcement must be real, not advisory.

Any escalation that genuinely needs customer identity routes to the OWNER (VT-279),
not the VTR. High-priority within the VTR layer (it's the privacy guarantee the
CL-425 framing leans on). Ties to VT-231 (Mumbai prod) + VT-272 (privacy notice).
