## Contract (CL-437 ruling 3, Fazal 2026-06-12 — near-verbatim policy)

1. Outbox message bodies are retained ONLY while needed for delivery, retry, replay, or
   drain. On successful drain / terminal completion (delivered, terminally failed,
   cancelled), **redact the body fields in the outbox row, keeping metadata and hashes**
   (timestamps, status, template refs, hashed tokens — enough for ops/idempotency, zero
   content).
2. Exact owner-facing text is retained ONLY in the tenant-scoped owner conversation/audit
   surface, under normal tenant retention + DSR. STEP-0: verify that surface actually
   captures the text today; if it does not, capturing it there is IN scope (the ruling
   presumes it exists).
3. Sweep/backfill: existing terminal outbox rows get the same redaction.
4. Tests: redaction fires on each terminal path; retry/replay still works pre-terminal;
   DSR purge unaffected; a forced mid-retry redaction does NOT occur (retain-while-needed
   is part of the policy, not just the redaction).

## Memory note (standing lesson)

Outbox/event payloads retain raw PII at rest even when projections hash it — check the
EMIT payload and purge-on-drain, not just the downstream store. This row is that lesson's
closure for the owner outbox.
