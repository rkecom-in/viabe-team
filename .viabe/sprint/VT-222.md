# VT-222 — Sheet integration redesign (Drive Push + polling)

## Why this row exists

VT-212 walk surfaced that the Apps Script paste flow (VT-207 PR-1) is customer-hostile. Target persona = SMB owner / VP-level senior person; they will not paste code into Google Apps Script + add triggers. Fazal called this out 2026-05-29 04:00 IST.

CL-421 (Standing): "All Integration Agent connectors MUST be zero-manual-paste after OAuth."

This row replaces the Apps Script substrate with a customer-correct flow:

- **Drive Push Notifications (Files.watch API) = primary path.** Owner does OAuth once → orchestrator registers a Drive change webhook channel → file edits notify the orchestrator in <1s → row delta pulled via Sheets API → routed through field-mapping + dedupe → DB.
- **Polling (every 10 min) = fallback path.** Kicks in when push channel is unregistered, expired (Google's channels expire every ~7 days max), or webhook delivery is failing. Also triggerable manually by seller/admin for re-sync.

## Scope

### A. Drive Push Notifications

1. **`apps/team-orchestrator/src/orchestrator/integrations/connectors/google_sheet.py`** — add:
   - `register_drive_push_channel(tenant_id, spreadsheet_id) -> ChannelDescriptor` — calls Google Drive `files.watch` API with our `https://vt-orchestrator-service-development.up.railway.app/api/orchestrator/integrations/sheet/drive_push` as the webhook target. Persists channel_id, resource_id, expiry into a new `tenant_drive_channels` table.
   - `unregister_drive_push_channel(channel_id)` — calls Drive `channels.stop`
   - `renew_drive_push_channel(channel)` — stops the old channel, registers a new one. Called by the renewal scheduler before expiry.

2. **New migration `038_tenant_drive_channels.sql`** — table to track active channels:
   ```sql
   CREATE TABLE tenant_drive_channels (
     id UUID PRIMARY KEY,
     tenant_id UUID REFERENCES tenants(id) ON DELETE CASCADE,
     connector_id TEXT NOT NULL,  -- 'google_sheet'
     resource_id TEXT NOT NULL,   -- the Sheet ID
     channel_id TEXT NOT NULL UNIQUE,
     channel_token TEXT NOT NULL,  -- random; verified on incoming notification
     expires_at TIMESTAMPTZ NOT NULL,
     created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
     last_notification_at TIMESTAMPTZ
   );
   CREATE INDEX idx_tenant_drive_channels_expires ON tenant_drive_channels(expires_at);
   ```
   RLS: tenant_id app_current_tenant().

3. **Webhook handler `apps/team-orchestrator/src/orchestrator/api/drive_push.py`**:
   - `POST /api/orchestrator/integrations/sheet/drive_push`
   - Verifies `X-Goog-Channel-Token` header matches stored `channel_token` for the channel
   - On `resource_state = 'sync'` (initial setup ping): 200 OK, no-op
   - On `resource_state = 'update'`: enqueue `pull_sheet_delta_workflow(tenant_id, spreadsheet_id, since=last_notification_at)` via DBOS
   - Updates `last_notification_at`

4. **`pull_sheet_delta_workflow`** (`@DBOS.workflow`) — uses Sheets API to fetch rows modified since `since`. Routes through field-mapping (VT-209) + phone-hash dedupe (VT-184) + DB insert/update tagged `acquired_via='google_sheet'`.

### B. Channel renewal scheduler

5. **`@DBOS.scheduled("0 */6 * * *")` workflow `renew_expiring_drive_channels`** — every 6 hours, finds channels expiring within 48 hours and renews them (stop old + register new).

### C. Polling fallback

6. **`@DBOS.scheduled("*/10 * * * *")` workflow `poll_unwatched_sheets`** — every 10 min:
   - For each tenant where `tenant_drive_channels` has no active row for this connector, OR last_notification_at older than 30 min (presumed webhook delivery failed)
   - Call Sheets API to fetch all rows
   - Diff against last-known state (track via tenant-level checkpoint)
   - Process delta through the same ingestion pipeline

### D. Drop Apps Script substrate

7. **Mark `setup_push()` + `apps_script_template.render_apps_script()` as deprecated** with a comment block pointing at this row. Don't delete — leave for future migration story. Remove all references from integration agent prompts (VT-206) + onboarding flow (VT-211).

8. **Update `apps/team-orchestrator/src/orchestrator/prompts/integration_agent_system.md`**: remove Apps Script mention; integration agent's phase 4 (push setup) now describes Drive Push registration as automatic-after-OAuth.

### E. Onboarding flow update

9. **Update VT-211's `/team/onboard` page** to describe the simplified flow: "Connect Sheet → grant access → done." No Apps Script step.

## Acceptance criteria

1. Owner completes OAuth consent for Google Sheet connector
2. Orchestrator auto-registers Drive Push channel within 5s of OAuth callback
3. Owner edits any row in the Sheet
4. Within <2s, orchestrator receives Drive push notification
5. Within <5s, the row delta is pulled, field-mapped, deduped, and written to DB tagged `acquired_via='google_sheet'`
6. Channel renewal runs every 6h; no channel ever expires in production
7. When push channel is expired/missing, polling worker pulls rows within 10 min
8. Owner sees ZERO mention of Apps Script anywhere in onboarding (web or WhatsApp)
9. CL-421 acceptance: no manual paste required after OAuth

## Rule #15 Canary (mandatory)

`apps/team-orchestrator/canaries/vt222_sheet_drive_push.py` — 6 assertions:
- A1: OAuth callback → channel registered + persisted
- A2: Synthetic Drive push notification → row delta pulled + written
- A3: Channel renewal — synthetic channel set to expire in 1h → renewal runs → channel replaced
- A4: Polling fallback — delete tenant_drive_channels row for a tenant → polling worker pulls rows within 10 min synthetic-tick
- A5: HMAC / channel_token rejection — replay a notification with wrong token → 401, no DB write
- A6: Apps Script substrate not invoked — assert `setup_push` is not called anywhere in the new onboarding flow

## Out of scope

- Multi-Sheet per tenant (Phase 2)
- Real-time vs polling preference toggle per tenant (Phase 2)
- Drive change notifications for non-Sheet file types
- Migration from Apps Script (existing tenants would need to re-OAuth or have channels registered retroactively; defer)
