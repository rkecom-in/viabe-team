# CL-116 — PR-fix-7 shape locked

Fazal locked three calls for PR-fix-7 architecture:

1. WRAPPER SHAPE: Both primitives. (a) Context manager `tenant_connection(tenant_id)` is the load-bearing primitive — every DB writer must enter through it. (b) Decorator `@tenant_scoped` wraps DBOS steps that take SubscriberState (or any object exposing tenant_id) and applies the wrapper implicitly. Decorator is sugar; context manager is canonical. Both implemented, both tested.

2. TEST BAR: Switch CI Postgres role from superuser to app_role with FORCE RLS enabled on all tenant-scoped tables. This proves every existing test goes through the wrapper — if a writer bypasses, the test fails with permission denied. No separate E2E test needed; the entire 63-test suite becomes the proof.

3. SCOPE: All 10 call sites in one PR. Half-migrated state is worse than a slightly larger PR because any non-wrapped writer left in main reproduces the original C1 failure under app_role CI. Atomic.

CALL SITES (10 from CL-71): transitions.apply_transition (VT-3.2); twilio_ingress._record_inbound (VT-3.3a); runner.open_run + close_run (VT-3.1); runner.open_webhook_run + record_webhook_received + close_webhook_run (VT-3.3a); opt_out_handler + dsr_handler + status_ping_handler (VT-3.8). The 10th depends on what's surfaced in current main — Claude Code re-greps before drafting the diff.

OUT OF SCOPE: JWT/COALESCE policy variants. Per CL-88, those activate with first dashboard read, not now. PR-fix-7 stays GUC-only.
