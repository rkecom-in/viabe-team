# CL-269 — DECISION (Type 2, Fazal): draft_message_variants DEFERRED to Phase 1.5. v1 LLM-b

Type 2 governance decision, Fazal, 2026-05-20 (recorded in-session 2026-05-21). draft_message_variants (the optional LLM-backed tool that would generate alternative phrasings of campaign messages, parent-relative VT-5.13) is DEFERRED to Phase 1.5 — NOT in the v1 LLM-backed tool set. The v1 LLM-backed set stays at exactly two: self_evaluate and classify_owner_message, both of which have hard Phase 1 consumers. RATIONALE: the SR agent already produces message_plan (template_id + parameter substitution) inside CampaignPlan v1.0 — that covers v1 messaging. Variant generation has no Phase 1 consumer and, more importantly, no production signal to choose between variants against; it is correctly a Phase 1.5 concern once real campaign-outcome data exists. Deferring also holds the LLM-backed surface (Pillar 2 justification cost, Opus spend) to two tools instead of three. Decision page: 366387c2-cc5a-8159-b726-e7ce3ec6f4f3. CONSEQUENCES: (1) VT-5.13 sprint row status → Deferred (Clau follow-up, post PR #36 merge). (2) docs/team/llm-backed-tools-rationale.md must state this defer with attribution — handled by a fix commit on PR #36 branch before merge. (3) VT-39 framework is decision-agnostic (is_llm_backed() + registry audit) so no framework code change needed. The VT-39 page's item-7 instruction to 'lock the decision in this subtask' is satisfied by this entry + the doc commit; the framework PR itself does not encode the decision.
