# VT-245 — CI cost-control + local pre-push gate

main has **no branch protection** (ruleset empty, Fazal-confirmed) — CI is
a non-blocking backstop, not a merge gate. The cost is workflows running on
every trigger. Posture: no protection + a local pre-push gate as the safety
net.

## Phase A — trigger-diet (THIS PR, shipped directly — urgent cost fix)
- **ci.yml** `paths-ignore` on `pull_request`: skip the 17-job suite when a
  PR touches ONLY `**/*.md`, `docs/**`, `.viabe/**`, `cowork_sessions/**`,
  or `.github/workflows/deploy-dev.yml`. (ci.yml is already
  pull_request-only + `concurrency: cancel-in-progress` — those diet items
  were already in place.)
- **deploy-dev.yml** `paths-ignore` on `push`: don't redeploy dev when a
  merge touches ONLY docs / sprint / session-exports / ci.yml.
- Validated with `actionlint` (exit 0; 2 pre-existing shellcheck infos
  unrelated to this change).

**Expected reduction:** docs/sprint/CI-only and deploy-dev-only PRs (a large
share of this session's volume — session-close, snapshot, substrate
reconcile, the deploy-dev fix loop) now trigger **zero** ci.yml jobs instead
of 17, and docs/sprint merges trigger **no** dev redeploy. Rough order: the
~6 docs/substrate PRs + ~3 deploy-dev fix PRs this session would have run 0
heavy suites instead of ~9×17 ≈ 150 job-runs.

## Phase B — local pre-push gate (PLAN-FIRST, follow-up)
Commit the local-CI-parity (established 2026-05-30) into a versioned
`pre-push` git hook + installer (`scripts/git-hooks/pre-push` +
`scripts/install-hooks.sh`). Runs the FAST suite (ruff + mypy + fast pytest
excluding live-DB/canary; team-web tsc + lint + vitest), aborts the push on
failure, documents `--no-verify`. Plus a **docker build** of the
orchestrator image (Railway deploys via Docker) to catch image/dep/startup
failures pre-push — NOT auth failures (the RAILWAY_TOKEN deploy failure is a
secret a local build can't catch; confirmed: the Railway job is
`railway up` and died on auth in 0.4s before any build).

## Phase C — workflow maintenance (PLAN-FIRST, follow-up)
- Fix the 2 known flakes: RLS `service_count` + chrono-order.
- Node-20 action deprecation (force-migrated to Node 24 on 2026-06-16):
  bump `actions/checkout@v4`, `actions/setup-node@v4`, `astral-sh/setup-uv@v5`,
  `pnpm/action-setup@v4` to Node-24-compatible versions; validate with

- **Docs correction:** CLAUDE.md + docs/clau/latest-snapshot.md falsely claim
  "main is protected — 11 required checks." Update to: no branch protection;
