# VT-266 — dsn fixtures swallow apply_migrations failure result

From the VT-262/263 adversarial review (dropped-low). The real-DB test `dsn`
fixtures call `apply_migrations.apply(dsn)` but discard the returned result. If a
migration fails partway (e.g. the local mig-015 CREATE ROLE non-idempotency
halt), tests surface as confusing UndefinedTable cascades instead of a clear
"migration X failed" error. Assert `result["failed"] == []` in the fixtures so a
partial-schema apply fails loudly. Low priority, post-Sprint-2. (Relatedly, the
mig 015/027 CREATE ROLE non-idempotency is the root cause of the local-cluster
halt — separate hygiene item.)
