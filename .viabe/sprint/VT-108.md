## What CoderC must produce
All 22 Meta WhatsApp Business templates authored, Fazal-reviewed, Meta-submitted, and Meta-approved. Plus the canonical template registry that maps template names to Meta-approved IDs.
1. **Template inventory** (22 templates total; full list above):
	- Owner communications: 13 templates (welcome, weekly_approval, campaign_send_confirmation, attribution_close, day39_evaluation, refund_offer, monthly_report, support_handoff, support_resolved, trial_extension_offered, trial_max_reached, refund_processing, refund_completed, opt_out_confirmation/reconstitution_completed if owner-facing variants).
	- Customer communications: 6 templates (clarifying_question, paper_book_summary, contacts_import_confirmation, upi_export_summary, breach_notification_customer, dsr_verification — when sender is consumer-side).
	- Workflow: 3 templates (dsr_deletion_completed, breach_notification_owner, optional dsr_export_ready).
2. **Authoring process**:
	- Each template authored in English + Hindi (Devanagari script); Meta supports per-language variants of the same template name.
	- Variables: documented per template (e.g., welcome has `{{1}} = business_name`, `{{2}} = owner_name`).
	- Tone: Indian SMB-appropriate; respectful, non-pushy, regionally-aware. Fazal reviews every template before submission.
	- Authored at `apps/team/data/whatsapp_templates/<template_name>.{en,hi}.md` with metadata header (variables, category, justification for category).
3. **Submission to Meta**:
	- Submitted via Twilio Console (Twilio is the BSP — Business Solution Provider).
	- Categorization: `MARKETING` for promotional (most templates), `UTILITY` for transactional (welcome, refund_completed, dsr_deletion_completed), `AUTHENTICATION` for OTP-style (dsr_verification).
	- Per-template review by Meta: 24-72 hours typical; first-submission rejection rate 30-50%; revisions add 2-5 days.
	- Realistic timeline: 6-8 weeks for all 22 templates including revisions.
	- Submitted in batches of 5 to manage revision overhead.
4. **Canonical registry** at `apps/team/config/whatsapp_templates.yaml`:
	- Maps logical name → Meta-approved template SID + language variants.
	- VT-5.7 `send_whatsapp_template` reads from this registry. CodeX rejects PRs hardcoding Meta SIDs in code.
	- Sandbox SIDs (test mode) vs production SIDs separated via env var.
5. **Verification + go-live**:
	- Each template tested against sandbox sender ID (Twilio test sender) before adding to registry.
	- Templates marked `ready_for_production: true` only after Meta approval + sandbox verification.
	- Pre-launch: full registry verified — no template missing approval; deployment fails if any required template is unapproved.
6. **Maintenance**:
	- Template revisions post-launch require new Meta submission + approval cycle.
	- Template rejections by Meta documented in `docs/runbooks/meta-template-revision.md`.
	- Quarterly review: retire unused templates, refresh active ones.
## Architectural rules to enforce in this subtask
- Pillar 7: every customer-facing template is honest, non-pushy, culturally appropriate. CodeX rejects manipulation tactics (false urgency, fake scarcity).
- Pillar 8: one registry. CodeX rejects hardcoded Meta SIDs.
- Templates authored AND Fazal-reviewed BEFORE Meta submission. Fazal personally reviews every word + Hindi translation.
- Categorization (MARKETING/UTILITY/AUTH) chosen per Meta's rules; mis-categorization is a rejection cause.
- Sandbox + production SIDs separate.
- Launch gated on full registry approval; deployment fails on missing.

## Out of scope
- Multi-language beyond English + Hindi (Phase 2: Marathi, Tamil, Bengali if owner demand).
- Dynamic template content (Phase 2; Meta now allows dynamic body but Phase 1 uses static templates with variables).
- Rich media templates (images, buttons, lists) — Phase 1 uses text-only with simple variables; Phase 2 may add rich media for monthly_report.
- Template A/B testing (Phase 2; Meta supports template variants but quality control is more important Phase 1).
- Cross-product template sharing with Reports product (separate; Reports has its own Twilio templates).
