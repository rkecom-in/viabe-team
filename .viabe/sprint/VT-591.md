# VT-591 — the manager's reply is always complete

## Finding (deployed-dev 77c1083, reading replies)
With compose_owner_output_tool removed (VT-590), the manager writes its reply as text and [2]/[5] are Claude-grade. But explainer questions still truncate: [3] "how does this actually work?" → "Great question — here's how it works end to end:" then nothing; [4] "is my data safe?" → "Your data is safe — here's exactly how:" then nothing. The agent emits the intro (no tool call → ReAct loop ends) and stops — a model behavior three prompt passes did not reliably fix.

## Fix
In dispatch_brain's transmit seam (VT-589 _maybe_send_manager_reply), detect an INCOMPLETE trailing message and, only then, run ONE focused no-tools compose-completion LLM call that emits the COMPLETE owner reply; send that. Preserves the already-good complete replies (no extra call on those). Fail-soft: on compose error/empty, send the raw trailing text if non-empty, else nothing (D1 fallback remains the net). Validated EMPIRICALLY on deployed dev across the 6-probe set.
