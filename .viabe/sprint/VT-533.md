# VT-533 — OC1: customer-send policy rail shadow (observe-only)

## Gap
`business_policy.assert_within_policy` is BUILT and `customer_send_choke` calls it — but ONLY under `enforce_policy=True`, which defaults False everywhere, so the rail **never runs**. B5's `requires_policy_rail` exists with no consumer. There was zero operational signal on whether flipping enforcement on would block real sends (it would — no policy grants are seeded yet), so the flip was un-de-riskable.

## Built (0 migrations) — additive, observe-only
- `assert_customer_send_allowed` gains **`observe_policy: bool = False`**. When set (and NOT enforcing) it evaluates the SAME `assert_within_policy` check and, on a would-be block, records a `policy_shadow` tm_audit row (`event_layer='decides'`, `status='observed'`) — then returns **allowed=True** (never blocks). Fully fail-soft (a shadow error never touches the compliance pre-gate).
- The **`enforce_policy` branch is byte-for-byte unchanged** (added as a sibling `elif observe_policy` — the enforcing path keeps its exact fail-closed behavior).
- **Wired live at `campaign/execute.py`** (the winback campaign send) with `observe_policy=True` — every real campaign send now shadows the rail, surfacing "enforcement would block here / no policy grant yet" data.
- `policy_shadow` added to the tm_audit event_kind vocab (advisory).

## Verify
ruff clean; **19/19** — the OC1 test proves all three modes: `observe_policy=True` + no grant → **allowed=True** + a shadow recorded; `enforce_policy=True` + no grant → **blocked** (`SKIP_OUT_OF_POLICY`); default (neither) → no policy eval + no record (byte-for-byte) — PLUS the pre-existing shared-pregate + campaign_execute tests unchanged (the wired caller didn't alter the campaign path).

## NOT in scope
The `enforce_policy=True` FLIP (deliberate — needs seeded policy grants + the shadow data first; a Fazal/launch decision). VT-533's broader VTR+owner-policy CONTRACTS (the `requires_policy_rail`→capability-key coupling; the VTR-side policy authoring) — this slice is the send-side observe rail only.
