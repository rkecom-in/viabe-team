# VT-577 — the needs registry that drives everything

## Ruling
Each business agent (Sales Recovery, Marketing, …) declares WHAT DATA it needs to (a) compose a month plan and (b) execute it, toward revenue/engagement/sales/customers/leads (retarget/reactivate dropouts, inactive, window-shoppers) + cost optimisation. The onboarding integration menu, its ordering (easiest × most-enabling first), and each ask's "why" copy derive from this registry — never a hardcoded connector pitch.

## Scope
Declarative registry: per agent → needed data classes (customers+contactability, transactions/sales history, catalog, traffic/engagement, ad accounts…) → which INTEGRATIONS supply each (Shopify, Google Sheets, file upload pdf/csv, GSC, GA, Google Merchant, WABA, Meta marketing, FB/Insta) → per-integration owner instructions (plain, how to find the thing). Consumed by VT-576's paced flow + later by the agents' own readiness checks ("can I plan yet? what's missing?").
