# VT-453 — Shopify webhook-registration: bounded retry + actionable surface (no silent miss)

Built. `api/shopify_oauth.py` OAuth callback: the `setup_push` (webhook registration) now (1) gets ONE bounded
RETRY on a transient failure before giving up, and (2) on a final miss surfaces an ACTIONABLE state — the
response `status` becomes `connected_webhooks_unregistered` (no longer a bare `ok`) + `action_required:
reregister_webhooks`, and an ERROR (not just exception) log. The token still persists (install survives). So a
quiet registration miss is VISIBLE to the caller / a re-register path, not hidden. Tests: failure→retry(2)+
actionable state+token persisted; transient-then-success→retry recovers→webhooks_registered=true/status ok;
9 shopify-oauth-endpoint tests green. Risk row (deploy path) → Cowork gate → Fazal push.
