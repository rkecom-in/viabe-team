# VT-345 — Lighthouse-CI on the owner portal (post-launch)

Deferred from VT-340 (Fazal/Cowork 20260606T021500Z, option c). A CI job that builds team-web
+ serves it + runs `lhci` asserting >85 perf/a11y on the dashboard.

**The wrinkle (why post-launch):** the dashboard index is `requireOwnerSession`-gated → `lhci`
hits the login redirect, not the dashboard. So it needs either (a) a CI-only signed owner
session cookie minted for the lhci run, OR (b) target a PUBLIC page (marketing/signup) that
doesn't exercise the real dashboard. Plus it adds per-PR build+serve+lhci time for a portal
NOT yet live to real owners — perf-gating a pre-launch surface is premature. NOT a launch gate
(the Jun15 milestone doesn't depend on it).
