## Contract (2026-06-23 — RECURRING secrets-hygiene breach)

CC has now leaked a live secret VALUE into model context **twice** via broad `railway variables`
output: `RESEND_API_KEY` (golive readiness scan) and a **PROD `ANTHROPIC_API_KEY` fragment** (VT-402
Conductor scan). Each = a secret entering CC's turn context (→ Anthropic that turn), violating CL-431.
A verbal "don't grep raw variables" guardrail did not hold. Need a tooling-level fix.

## Build
1. A tiny helper — e.g. `scripts/env_presence.py` (or a shell wrapper) — that takes a list of var
   NAMES and prints ONLY presence-booleans / equality-by-reference (set/unset, MATCH/MISMATCH), and
   **never** emits any value to stdout. Internally it may read values but only to compute a boolean.
2. **Binding discipline (add to CLAUDE.md / discipline-rules):** CC NEVER runs raw `railway variables`
   / `--json` / `| head` against an env that holds secrets. All env inspection goes through the helper
   (names → booleans). Pipe-to-head/cat/echo of any secret store is forbidden.
3. (Optional) a lint/grep CI check flagging raw `railway variables` in any committed script/signal.

## Acceptance
- A documented, used helper that returns presence/equality without ever printing a value.
- Discipline rule recorded so future sessions don't repeat it.

## Immediate (Fazal, separate from this build)
**Rotate PROD `ANTHROPIC_API_KEY`** (and `RESEND_API_KEY` from the earlier incident if not already).
