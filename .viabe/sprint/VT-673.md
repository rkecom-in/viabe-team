# VT-673 — plan/roadmap read tool

## Gap (`KNOWN_CAPABILITY_GAPS[plan_roadmap_read]`)
`get_active_plan` / `items_for_agent` are dispatch MACHINERY — the Manager assembles the context and
hands it down. They are NOT a callable tool: a specialist cannot ask "what is my plan / what's next
on my roadmap" mid-loop. It only ever sees the slice the Manager pre-baked.

## Build
Add a first-class READ tool (`read_active_plan` or `read_agent_plan`) — tenant+agent scoped, PII-safe
(returns plan items / step status, not customer rows) — into the common read set. Catalog it. Wire it
so a specialist loop can pull its current plan on demand.
