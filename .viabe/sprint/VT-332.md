# VT-332 — Trial-end token issuance + single-use (HARD GATE on the deep-link path)

VT-91 shipped the deep-link VERIFY (`verify-trial-end-token.ts`) + the /subscribe auth
path, but the path is DORMANT — no token is minted yet. This row makes it live, and
single-use replay protection is NON-NEGOTIABLE here (Cowork 20260605T142200Z).

## HARD GATE (binding — not an open item that can slip)
- **Single-use consumption MUST ship WITH or BEFORE token minting.** Issuance must NOT go
  live before replay protection exists — the moment a token is minted without a consume
  step, the full 7-day TTL becomes a replay window on a PAYMENT deep-link.
- **The deep-link path stays DORMANT until BOTH land.** No trial-end template wires a real
  `?token=` until single-use is in.

## Scope
1. **Minting** — issue an audience-`'trial-end-subscribe'` HS256 token (`OWNER_JWT_SECRET`,
   7-day TTL, a unique `jti`). Lives in the team-web `owner-jwt.ts` family; the orchestrator
   gets it via an internal mint endpoint (the trial-end nudge embeds the deep-link).
2. **Single-use consumption** — on a successful subscribe via the token path, CONSUME the
   `jti` (a workspace-wide `consumed_subscribe_tokens(jti PK, consumed_at)` — service-role,
   NO tenant_id → no DSR; the orchestrator consumes it atomically in razorpay-subscribe).
   A replayed (already-consumed) token → 403.
3. **Trial-end template wiring** — the trial-end WhatsApp nudge embeds
   `viabe.ai/team/subscribe?plan=<tier>&token=<jwt>` (template copy = NEEDS-FAZAL).
4. **Nit (VT-91 #339 subagent):** pin `algorithms: ['HS256']` explicitly in
   `verifyTrialEndToken` (defense-in-depth; safe today via the symmetric key, cheap to harden).

## Canary
expired → 403, wrong-audience → 403, cross-tenant → 403, **replayed-after-use → 403** (the
single-use keystone), valid first-use → subscribes exactly once.
