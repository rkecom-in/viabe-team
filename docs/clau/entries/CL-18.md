# CL-18 — Correction to prior log: update_properties WORKS for single-page targeted update

Earlier 2026-05-12 log entry stated 'update_properties' was denied at the tool-approval layer. That was wrong. update_properties DOES work when targeting a single page with a focused property update (e.g., changing Status on one entry). What gets denied is BULK update_properties spanning many pages at once, or update_properties with very large property payloads. Implication: for targeted single-page property changes (e.g., updating one subtask's Status or Target Date), use update_properties directly. For multi-page batch changes, fall back to creating overlay subtasks or marking-Superseded patterns. Discovered when clearing 6 inference entries from ClauReports_Session_Log on 2026-05-12.
