# VT-626 — first "connect my Sheet/Shopify" ask has no deterministic net

Root-caused (i_sheets gate cluster): the manager holds no direct mint tool; the FIRST connect ask rides on the
LLM emitting `spawn_integration`, no fallback → intermittent D1 stall / fake-narrated handoff (~2/5 runs). The
only deterministic integration gate — `maybe_resume_connector_onboarding` (runner.py ~953) — is RESUME-only
(fires after a mint set phase_2_auth). First-contact falls through to `dispatch_brain`. Same failure class as
[[VT-623]] (SR lane) — LLM-gated specialist handoff with no deterministic net on the critical path.

## Fix (deterministic, zero new LLM classifier — design-verified)
New `onboarding/connector_first_contact.py::maybe_start_connector_onboarding(tenant_id, body, sid, recipient)`,
mirror `connector_resume.py` (fail-open, dict-or-None). Slots in `runner.py` **between the resume gate (line
974) and pre_filter (line 976/1002)** — same inbound + non-dupe + `not integration_loop_owns_turn` guard, same
`close_webhook_run(...,"completed")` return shape as the resume gate (runner.py:966-974).

- **Intent (BUILD-TIME CORRECTION):** the design said reuse `_INTEGRATION_INTENT_RE` — but verified against the
  ACTUAL i_sheets scenario messages it is too NARROW: it needs the provider noun immediately after "my/the", so
  it MISSES "connect my **Google** Sheet", "can we get this **connected**?", and the Hinglish "Google Sheet …
  kaise **jodu**" — exactly the asks that stall. Real fix = a connect-VERB + provider detector (still fully
  deterministic, no LLM): `_CONNECT_VERB_RE(connect|connected|link|integrate|sync|setup|jodu|…)` AND a single
  unambiguous provider (`\b(google\s*sheet|sheets?|spreadsheet)\b` vs `\bshopify\b`; both/neither → brain).
  Empirically catches all real English+Hinglish asks, no false-fire on non-connect ("my sheet has 500 rows",
  "send a message to my shopify customers"). Residual (documented): a verb-LESS connect ask still uses the brain.
  NOTE: the LLM detects this intent fine — the flaky part is it sometimes fails to EMIT spawn_integration; this
  deterministic net removes the clear asks from that flaky path entirely.
- **Sheets** → call `start_sheets_oauth(tenant_id)` directly (sheets_oauth.py:28) — mints the real authorize URL
  AND arms `(phase_2_auth, awaiting=oauth_completion)` in one call; `_send` the link-out; return routed=minted.
- **Shopify** → CANNOT return a URL first-contact (needs the shop domain; start_shopify_setup validates it) →
  call `begin_shopify_onboarding(tenant_id, recipient)` (shopify_onboarding.py:455) — deterministic discovery
  kickoff ("reply with your store address"); the existing resume gate mints once the owner replies. Still off
  the LLM path. **Flag for Fazal:** shopify first-contact asks for the domain rather than minting inline.
- **Guardrails mirror the resume gate exactly:** opt-out/DSR first (`matches_opt_out_or_dsr` → None), loop-owns
  reuse `integration_loop_owns_turn` (runner.py:941-951), consent NOT needed (deterministic mint never
  transmits to Anthropic — runs before the consent gate like the resume gate), onboarded via placement (journey
  gate owns non-onboarded first), idempotency: live-pending (`_pending_is_unexpired`) → return None → no double
  mint, fail-open (except → None → normal pipeline).
