# VT-234 — Ops Console V2 phase-1A: read-only debug view

## Why

VT-189 is the Critical-priority parent for the full Ops Console V2 — debug/trace/track/train/override/monitor/resolve. Full surface is multi-week. Phase-1A carves out the smallest read-only slice that adds operator value WITHOUT requiring product decisions on override/train/monitor semantics.

Substrate is in main:
- VT-122 — pipeline_runs + pipeline_steps schema
- VT-179 — typed envelope schemas
- VT-180 — write_step writer (load-bearing)
- VT-181 — @tool_step decorator
- VT-201 — operator-aware live stream + history view + per-run navigation

This row extends VT-201's per-run view at `/team/ops/runs/[runId]` with a **debug drill-down**: full step-by-step envelope inspection.

## Brief contract

CC implements:

### 1. New page at `apps/team-web/app/(app)/team/ops/runs/[runId]/debug/page.tsx`

Server-rendered. Auth-gated via existing `requireFazal()` pattern. Per-step expandable cards showing:

- step_kind + status + timing (start_at → end_at, duration_ms)
- input_envelope JSON pretty-printed (collapsed by default; click to expand)
- output_envelope JSON pretty-printed (collapsed by default)
- error_envelope JSON if status='error'
- Tool calls (from tool_step substrate) — model_id + tokens_input + tokens_output + tool_name + raw_tool_input + raw_tool_output
- agent reasoning_trace if available

### 2. Read-only — no override/edit/replay/retrigger affordances

This is debug-view-only. Override/replay = phase-1B (separate row, future). Phase-1A locks: no buttons that change state. Only navigation links + JSON expand/collapse + copy-to-clipboard on JSON blobs.

### 3. Use existing data-access pattern

Read from pipeline_steps via the existing `apps/team-web/lib/ops/data-access.ts` server queries. Add new function `fetchRunStepsForDebug(runId)` that returns the full step list with all envelope fields populated. Use the secret-key client for cold-read aggregations (CL-52).

### 4. JSON pretty-print component

`apps/team-web/components/ops/json-pretty.tsx` — client component for collapsible JSON tree. Use `react-json-view-lite` (small, tree-shakeable) OR roll a minimal one. Prefer rolling minimal to avoid new deps.

If a dep is needed, ensure it has no React 19 incompatibility. Otherwise reject the dep + roll minimal.

### 5. Link from per-run page

Update `apps/team-web/app/(app)/team/ops/runs/[runId]/page.tsx` to add a "Debug view" link in the page header that navigates to `/team/ops/runs/[runId]/debug`.

### 6. Canary `apps/team-orchestrator/canaries/vt234_ops_debug_view.py`

3-4 assertions:
- A1: GET `/team/ops/runs/<seed_run_id>/debug` with operator JWT → 200 + page body contains step_kind labels for each step in the seeded run
- A2: GET same URL without operator JWT → 302 to /team/ops/login (auth gate works)
- A3: Page body contains pretty-printed input_envelope + output_envelope JSON for at least one step
- A4: Page body does NOT contain any element with text "Replay" or "Override" (phase-1B affordances NOT present)

## Acceptance

- New debug page renders + auth-gates correctly
- Step-by-step envelope visibility for any run in pipeline_runs
- Tool calls visible per step
- No state-changing affordances
- Canary 4/4 PASS

## Out of scope

- Run replay / retrigger (phase-1B)
- Override / edit step (phase-1B)
- Trace correlation across runs (phase-1C)
- Training / feedback signals (phase-2)
- Per-tenant filtering (already in VT-201 history view; reuse)
- Monitor / health dashboard (phase-2 monitor)

## Scope locks

- **DO NOT** add state-changing buttons or POST handlers — phase-1A is strictly read-only
- **DO NOT** introduce new dependencies unless minimal JSON pretty-print is genuinely impossible (likely it's not — roll minimal)
- **DO NOT** touch VT-233 auth surface files (`/team/ops/login/*`, `/api/ops/login/*`, `lib/auth/*`)
- Use existing requireFazal() for auth — DO NOT widen it
- Existing pipeline_steps RLS + operator-JWT pattern applies (CL-88)
- No new env vars

## Rule #15 canary

Per acceptance above. Real-mode via mocked seeded run; CI-friendly.
