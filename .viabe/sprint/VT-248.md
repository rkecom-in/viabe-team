# VT-248 — dedicated count-bearing campaign-rejected owner template

## Why (follow-up from VT-241, 2026-05-30)
VT-241 wired fail-closed cohort rejection: when a campaign's cohort holds an
unresolvable / cross-tenant `customer_id`, the campaign is rejected (txn rolls
back) and the owner gets the generic Tier-A `team_unable_to_complete_request`
template — "couldn't complete your request" — with NO count.

Cowork ruling (a), 2026-05-30: the count-bearing surface (telling the owner
"N targets couldn't be validated, campaign not sent") is **vendor-gated** — it
needs a Meta-approved Twilio Content template SID, which is the VT-108 / CL-5
path and is currently blocked. So VT-241 shipped the safe generic surface +
the real security guarantee now, with no vendor dependency.

## Scope (do NOT start until VT-108 unblocks)
1. Author a Meta WABA template (via the VT-108 approval path) that surfaces a
   COUNT of invalid targets to the owner — e.g. "Couldn't send your campaign:
   {{count}} target(s) couldn't be verified." NO ids, NO cross-tenant
   distinction (the VT-241 privacy guard holds — owner sees count only).
2. Register `template_name → content_sid` in `twilio_templates.yaml`.
3. Re-route `campaign_not_sent_invalid_cohort` in `template_routing.yaml` from
   `team_unable_to_complete_request` to the new template.
4. The orchestrator already emits `rejected_count` in the
   `campaign_rejected` terminal dict (VT-241) — the composer just needs to map
   it into the template_params. No collapse/dispatch change expected.

## Dependencies
- **VT-108** (Meta template approval pipeline) — hard blocker. Until it lands,
  there is no approved SID to route to; VT-241's generic surface stays.

## Notes
- This is a UX nicety, not a correctness gap. The security guarantee
  (fail-closed, no unbacked campaign, no cross-tenant oracle) already shipped
  in VT-241 (PR #157). Priority Low / gate on VT-108.
- Privacy invariant (carried from VT-241 + Cowork): owner message reports the
  COUNT only; the full rejected-id list + reason lives in the operator audit
  log, never the owner surface.
