# VT-552 — B1 part-2b

## Gap
A run that reached `status='completed'` with NO `final_outcome` and NO effect (a **silent terminal**) left no durable, escalatable record — the owner never heard, ops never saw. There was no incident spine and no owner→VTR escalation path for it.

## Built (mig 156, reuses pipeline_runs + escalations)
- **Terminal-outcome enum + classifier** (`observability/terminal_outcome.py`, pure) — `classify_terminal(status, final_outcome, has_effect)` over the existing pipeline_runs fields → `COMPLETED_WITH_OUTCOME` / `COMPLETED_SILENT` / `ESCALATED` / `ABORTED` / `REJECTED` / `RUNNING`; `is_silent_terminal`.
- **`incidents` table** (mig 156) — durable incident spine, one per (run, kind) (idempotent), tenant-RLS+FORCE + operator SELECT + DSR. `escalation_tier` = the ladder (0 detected → 1 owner-contacted → 2 vtr-escalated), `version` CAS.
- **Escalation ladder** (`observability/incident_store.py`) — `create_incident` (idempotent), `escalate_incident` (CAS tier advance; at tier ≥ 2 creates a VTR `escalations` row — the mig-073 queue, via the SERVICE path, idempotent on `uq_escalations_run` — and links it back), `resolve_incident`, `get_incident`.
- **`silent_terminal` detector** (`orphan_reaper.detect_silent_terminal_runs`, wired at boot next to the other reapers) — finds completed runs with no `final_outcome` older than 30m and no existing incident → opens a `silent_terminal` incident + fires the `silent_terminal` alert. Best-effort, never raises.

## Verify
ruff clean; dep-less-safe (classifier + orphan_reaper import without heavy deps); mig 156 applies; **23/23** — classifier (all outcomes + silent detection), incident idempotency, CAS escalation ladder + the VTR escalations row at tier 2 + CAS no-double-advance, resolve, the detector opens an incident + alerts for a silent run / ignores a run WITH an outcome / is idempotent, + incidents DSR erasure.

## NOT in scope
Wiring `escalate_incident` tier-1 to the live owner-notification send (the owner-contact tier records intent + `owner_contacted`; the actual owner message reuses `owner_notification` — a follow-up wire). Auto-resolve of an incident when a later run for the same tenant succeeds.
