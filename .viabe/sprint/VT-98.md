## What CoderC must produce
A structural social-proof section on the landing page with placeholder content that is HONEST (not fake testimonials, not fabricated metrics). Real proof gets added post-launch as customers convert and provide testimonials.
1. **Component** at `apps/team-web/components/team/SocialProof.tsx`:
	- Renders 4 sub-sections:
		- **Quote testimonials** (max 3): each has owner_name, business_type, locality, quote, optional_photo. Phase 1 placeholder content: "Coming soon. We're launching with founding members; their stories will appear here as they share results." Honest framing.
		- **Customer logos** (max 8 logos in horizontal scroll): Phase 1 empty. Component renders nothing in this section if empty array.
		- **Aggregate metrics** (e.g., "₹X recovered for our customers"): Phase 1 placeholder "Metrics will appear after launch as we accumulate customer results. We don't fabricate numbers." Honest framing.
		- **Press logos** (e.g., featured-in coverage): Phase 1 empty.
	- All 4 sub-sections content driven from `apps/team-web/data/social-proof.json`. Updates to add real testimonials/logos/metrics happen via that JSON; no code changes needed.
2. **Update workflow**:
	- Adding a real testimonial: PR that updates `social-proof.json` + uploads photo asset; CodeX reviews; Fazal signs off. Type 1 governance.
	- Adding fabricated content: forbidden. Pillar 7 enforcement at PR review.
3. **Aggregate metrics calculation**:
	- When the metrics section goes live (post-launch with sufficient data): a build-time script `apps/team/scripts/compute_landing_metrics.py` computes aggregate metrics from production data with k-anonymity ≥10 enforcement (reusing VT-8.3 admission gate).
	- Phase 1 deliverable: scaffolding the script; not running it (no data yet).
	- Aggregates: total ARRR recovered (across tenants), tenants serviced, average day-39 success rate. ALL with k-anonymity gating.
4. **Privacy + consent**:
	- Testimonials require explicit owner consent (signed release form; documented in `docs/team/testimonial-consent-process.md`).
	- Customer logos: brand permissions required (logo licensing).
	- Phase 1 has no real testimonials; consent process scaffolded for Phase 1.5.
5. **Accessibility**:
	- Quotes screen-reader-readable.
	- Logos with alt text.
	- Empty state communicated to assistive tech (don't show empty boxes).
6. **Tests** at `apps/team-web/__tests__/team-landing/social-proof.test.tsx`:
	- Empty state: synthetic JSON with no testimonials/logos → honest placeholder text shown; no empty boxes.
	- With content: synthetic JSON with 2 testimonials → rendered correctly; alt text + ARIA labels present.
	- Metrics scaffold: aggregate compute script runs without error on synthetic empty production data.
