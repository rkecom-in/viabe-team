=== L1 — PLACEHOLDER DISCLOSURE ASYMMETRY ===

Subagent finding: orchestrator_agent_system.md:24-30 has a `Note:` flagging `spawn_sales_recovery` as a placeholder, but says nothing about `escalate_to_fazal`, which is also skeleton placeholder per code docstring at orchestrator_agent.py:54. Subagent recommended extending the Note to cover both for parity.

Disposition: NOT (VT-3.9-fix-1) AS PROPOSED. The finding surfaces a real design question that needs Fazal's call before any prompt change ships.

The question: what behavior do we want from the agent regarding placeholder tools?

Two principles in tension:

(A) The agent should call placeholder tools as if they work — so we can verify the orchestrator's routing logic. Telling the agent 'this is a placeholder' may make it avoid the tool, defeating the routing test.

(B) The prompt should be honest about system state — lying to the model in the system prompt compounds; every edit becomes 'is this lie still useful?' instead of 'is this accurate?'

The existing prompt picks (B) for spawn_sales_recovery and (A) for escalate_to_fazal — incoherent and probably accidental, not designed. Three coherent options:

OPTION X — Remove the placeholder note entirely. Both tools described as functional in the prompt. Code docstrings (orchestrator_agent.py:54) still document skeleton state for human readers. Agent calls both normally. Skeleton truthiness lives in code, not in agent-facing prompt. Clau's lean.

OPTION Y — Add the note to both tools (subagent's recommendation). Parity but risks agent avoiding both placeholder tools.

OPTION Z — Status quo. Coherence cost; explicit reason needed to keep it.

Awaiting Fazal's call. Next session opens with this decision pending. Once decided:
- If X or Y: ship the prompt change as (VT-3.9-fix-1) PR by Claude Code, parent (VT-CI), CL-144 + CL-147 binding.
- If Z: document the reasoning here and close.

This is also a forcing function for Fazal's post-hoc VT-3.9 review (CL-143/CL-148 Track B) — the prompt design question is exactly the kind of thing the human reviewer should weigh in on.
