# VT-530 — C2a recovery_attempted audit event

## Gap
The manager's self-handling is real but INVISIBLE. The VT-484 seam (`_tool_error_to_tool_result`) turns a raised tool error into an error `tool_result` and the brain "either recovers (re-routes / picks another tool) or terminates cleanly" — but the audit spine only shows two disconnected `tool_invoked`/`tool_result` rows. There was no explicit, greppable "tool X failed → the manager was handed the error to recover" event.

## Built (0 migrations)
- At the VT-484 tool-error seam, emit one **`recovery_attempted`** `tm_audit_log` row (`event_layer='decides'`, `status='pending'` — the recovery outcome is decided on the brain's NEXT turn), scoped by `run_id` (from the ambient `ObservabilityContext`, the same VT-181 ContextVar this file already reads), carrying `{failed_tool, error_type}` in the redacted `decision`.
- **Purely additive + fully fail-soft** — the emit is in its own guard; an audit failure OR an absent observability context can NEVER affect the VT-484 error-to-`tool_result` conversion the brain depends on (no behavior change to the recovery path). Deterministic (middleware instrumentation), not a model self-report.
- `recovery_attempted` added to the `emit_tm_audit` event_kind vocab docstring (advisory, no CHECK → 0 migrations). The deterministic routers (`error_router`/`strategies`/`escalation`) stay authoritative for classified failures; this captures the manager's handling of novel/unclassified tool errors.

## Verify
ruff clean; 2/2 on fresh DB — an emit inside an observability context writes exactly one run-scoped `recovery_attempted` row (`decides`/`pending`/`warning`, failed_tool + error_type present); with no ambient context it is a silent no-op (no row, no raise).

## NOT in scope (follow-ups)
Linking via `parent_audit_id` to the failed tool's own audit row (needs that row's id at the seam — a refinement; run_id correlation suffices now). Resolving the `pending` outcome to recovered/terminated on the brain's next turn (a second emit). C2b confidence calibration.
