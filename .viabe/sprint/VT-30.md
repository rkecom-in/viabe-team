## What CoderC must produce
The DETERMINISTIC composer that takes a specialist agent's structured output (e.g., `CampaignPlan`) plus current orchestrator state (phase, escalation flags, pending approvals, error annotations) and produces ONE owner-facing message ready to send via WhatsApp.
**Architectural shift from prior framing:** the composer is now a TOOL invoked by the orchestrator-agent (or by deterministic state-machine paths like attribution-close confirmation). It is NOT a separate LangGraph node that fires automatically after every specialist invocation. The orchestrator-agent decides when to invoke it.
1. **Composer** in `apps/team-orchestrator/src/orchestrator/output_composer.py`:
	- Function `compose_owner_output(specialist_result: AgentResult | None, state: SubscriberState, intent_or_trigger: str) -> ComposedOutput`.
	- Registered as one of the orchestrator-agent's tools in VT-3.9's tool inventory.
	- Returns `ComposedOutput`: message body, message_type (`free_form_24h | template`), template_id, template_params, urgency, follow_up_required, follow_up_intent.
	- The composer is the SOLE producer of owner-facing message text.
2. **Composition rules** (deterministic Python, NOT LLM-generated):
	- If `state.escalation_pending`: prepend honest "the agent encountered an issue; here's what happened" framing.
	- If specialist_result has `terminated_by` (hard-limit breach): explain which of 5 limits (tokens/tools/depth/wall-clock/₹50 cost) was breached in plain language; offer Fazal escalation path.
	- If `state.phase == 'paid_at_risk'`: include soft retention nudge but NEVER apply pressure (Pillar 7).
	- If `state.phase == 'refunded'`: messages MUST acknowledge refund decision; cannot continue as if it didn't happen.
	- If outside 24-hour Twilio window: must use Meta-approved template (`message_type = template`); free-form rejected at tool level.
	- If inside 24-hour window AND no template applies: free-form with specialist content + orchestrator framing.
3. **Honesty rules** enforced in code (CodeX reviews these specifically):
	- The composer NEVER overstates ARRR. Uncertainty in attribution → message says so.
	- The composer NEVER hides a failure. Agent escalated → owner is told.
	- The composer NEVER applies retention pressure. "Are you sure you want to cancel?" allowed once; "Are you SURE? Look at all this value!" forbidden.
	- The composer NEVER claims certainty about customer intent ("this customer wants X") — always frames as "this customer's pattern suggests X."
4. **Language selection**:
	- Phase 1 supports Hindi + English via `tenant.preferred_language` (set during sign-up; defaults to English).
	- Templates pre-approved for both languages (VT-13.3). Free-form composed in preferred language.
	- Mixed-language replies (Hindi + English in same message, common Indian SMB pattern) ACCEPTED. Composer never enforces monolingual output.
5. **Template selection logic**:
	- Tier-A templates approved in VT-13.3: 5 essential. Tier-B and Tier-C added in later subtasks.
	- Composer maps `(intent_or_trigger, state.phase, escalation_pending)` → template_id when applicable.
	- Mapping table in `apps/team-orchestrator/config/template_routing.yaml`.
6. **Tool registration for orchestrator-agent**:
	- `compose_owner_output` registered as a tool the orchestrator-agent can call.
	- Tool description: "Compose an owner-facing WhatsApp message from a specialist result + current state. Returns structured ComposedOutput. Always call this BEFORE invoking send_whatsapp_message or send_whatsapp_template."
	- CodeX enforces: `send_whatsapp_message` / `send_whatsapp_template` tools (VT-5) reject calls where the message text didn't pass through `compose_owner_output` (via a hash signature in the call payload).
7. **Tests** in `apps/team-orchestrator/tests/orchestrator/test_output_composer.py`:
	- All Tier-A trigger→template mappings produce correct templates.
	- 24-hour window enforcement: synthetic state where last_owner_message_at > 24h forces template; <24h allows free-form.
	- Honesty rules: synthetic specialist_result with overstated ARRR gets corrected.
	- Refunded phase: messages always acknowledge refund decision.
	- escalation_pending: messages prepend honest framing.
	- Hindi + English: composer produces correct language for each preferred_language value.
	- Pressure avoidance: cancellation flow regex check rejects high-pressure copy.
	- Tool integration: orchestrator-agent can invoke `compose_owner_output` via supervisor; output flows back to agent's reasoning.

## Architectural rules to enforce in this subtask
- **Pillar 1 (revised):** composer is in orchestrator-agent's tool inventory. Specialists produce structured `AgentResult` objects, NEVER WhatsApp message text. CodeX rejects any specialist code that constructs message body strings.
- **Pillar 7 (owner-truth):** honesty rules enforced in code. CodeX reviews against explicit checklist.
- **Pillar 8 (no patchwork):** composer is SOLE owner-message producer. Other paths (Fazal escalations) use Telegram, not WhatsApp.
- Template routing config-driven. Adding new trigger→template = YAML change, not code change. New templates require Meta approval (VT-13.3) first.
- Composer is DETERMINISTIC. Same inputs → same output. No LLM call (the agent has already called LLMs to produce `AgentResult`; composer just packages).
- Mixed-language inputs from owners NEVER "corrected" by composer.

## Acceptance criteria
