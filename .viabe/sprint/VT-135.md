# VT-135 — CampaignPlan v0.1→v1.0 consumer migration + campaigns write-path reconciliation

## Summary
Migrate all v0.1 CampaignPlan consumers on `main` to the v1.0 discriminated-union schema shipped by VT-37 (PR #32). `main` is currently split-brain: v1.0 schema lives at `orchestrator.agent.schemas.campaign_plan`; all six consumers still import v0.1 at `orchestrator.types.campaign_plan`. The mismatch crosses the campaigns WRITE path — Priority Critical for that reason.

## Secondary Area
Primary Area: Specialist Agent. Secondary: Database (the campaigns table INSERT/column reconciliation is part of this subtask).

## Six v0.1 call sites to migrate
**Production (4):**
- `apps/team-orchestrator/src/orchestrator/supervisor.py` — :31, :37, :92 (sales_recovery_node parses stub JSON via model_validate)
- `apps/team-orchestrator/src/orchestrator/collapse.py` — :32, :50 (tenant guard + INSERT INTO campaigns)
- `apps/team-orchestrator/src/orchestrator/agent/sales_recovery_stub.py` — :23 (hardcoded_campaign_plan returns v0.1 shape)
- `apps/team-orchestrator/src/orchestrator/state/agent_graph_state.py` — :23, :47 (AgentGraphState.campaign_plan field type)
**Tests (2):**
- `apps/team-orchestrator/tests/orchestrator/test_supervisor.py` — :42
- `apps/team-orchestrator/tests/orchestrator/test_collapse.py` — :79

## Why this is not a type-swap
- v1.0 is a discriminated union — `CampaignPlan.model_validate(...)` sites must switch to the `parse_campaign_plan(...)` TypeAdapter accessor.
- v1.0 has no top-level `template_id` / `body_params` / `proposed_at` / `proposed_by` / `subscriber_id`. `template_id`→`MessagePlan`; `proposed_by` dropped (run_id covers correlation); single subscriber→`target_cohort.customer_ids` list.
- `collapse.py`'s `INSERT INTO campaigns` writes v0.1-shaped columns. Migrating the type alone breaks the write path — the campaigns table schema (or the INSERT) must be reconciled to the v1.0 shape in THIS subtask.

## Scope line
IN: six call-site migrations to the v1.0 accessor + campaigns INSERT/column reconciliation to v1.0 shape.
OUT: the lifecycle-state field (`approved`/`rejected`/`sent`/`failed`) — separate campaigns-schema / owner-surface subtask. If the INSERT reconciliation HARD-depends on the lifecycle field existing first, Claude Code surfaces that at pre-flight; do not pre-hunt it.

## Expected Outcome
`main` is no longer split-brain: every CampaignPlan consumer imports and parses the v1.0 discriminated union, and the campaigns write path persists v1.0-shaped rows. One PR.
