# CL-206 — CL-202 keyless-stub probe was hasattr mislabeled

=== WHAT HAPPENED ===
CL-202 (STATE SNAPSHOT, 2026-05-19 evening) recorded: 'Keyless-stub path probe-confirmed viable. Real probe this session: langchain_core.language_models.fake.GenericFakeChatModel has bind_tools, and create_agent constructs successfully with it.'

This claim was synthesized from a hasattr check that returned True (the method exists, inherited from BaseChatModel) PLUS the observation that create_agent construction did not raise (because bind_tools is called lazily, not at construction time).

Neither check exercises the actual surface. BaseChatModel.bind_tools is `raise NotImplementedError`. GenericFakeChatModel does not override it. Calling .bind_tools([]) raises immediately.

=== HOW IT WAS CAUGHT ===
The next-session brief (drafted this session by Clau) included an explicit pre-flight step 4: 'in a Python REPL or scratch file, import GenericFakeChatModel from langchain_core.language_models.fake, call .bind_tools([]) on an instance, confirm no exception. If this fails, STOP and report.'

Claude Code ran the probe. It raised NotImplementedError. Claude Code stopped, reported honestly, did not proceed to Test 1. This is CL-205 non-negotiable discipline working correctly.

=== ROOT CAUSE (CL-183 class) ===
Adjacent signal — hasattr returning True, plus construction-time no-raise — was treated as verification of runtime callable behavior. The signals are adjacent to the real question but do not transfer.

Meta-point: prior-session pre-compaction summaries are themselves captured prose. They are subject to CL-194 (ground-truth verification of repo-state assumptions) — and now extended to verification-claim assumptions. Any 'probe-confirmed' claim that gets carried forward across sessions must be re-callable-probed in the new session, not trusted on text.

=== AMENDMENT TO CL-202 DESIGN ===
Test design unchanged in semantics. Add a 4-line test-local subclass at the top of test_supervisor.py:

class ToolBindableFake(GenericFakeChatModel):
    def bind_tools(self, tools, **kwargs):
        return self

Usage substitutes ToolBindableFake for GenericFakeChatModel in the two pre-canned sequence instantiations. bind_tools is a no-op because tool_calls are baked into the pre-canned AIMessages — the fake never 'decides' to emit a tool call from a schema; it just yields the next pre-canned message.

Real build_supervisor_graph + real create_agent + real Command.PARENT + real conditional edge still exercised. Landmine-1 observation (Command.PARENT vs conditional-edge precedence) determined by langgraph executor, not by model layer.

=== STANDING IMPLICATION ===
Not a new standing rule — covered by CL-183 (adjacent verification anti-pattern). Adding to mental model: 'probe-confirmed' carried across session boundary = captured prose = must be re-probed at the actual callable surface in the new session.
