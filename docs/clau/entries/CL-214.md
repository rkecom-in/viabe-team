=== ASSERTION-BY-ASSERTION ===

(1) DBOS_STEP_PENDING_OBSERVED=True: DBOS marked the workflow PENDING between SIGKILL and relaunch. Confirmed independently by the DBOS recovery log: 'Recovering 1 workflows from application version 5c9f5a8a56a7067b928e99426393406b' at 22:11:35 in the relaunched worker. PENDING-tracking surface is functional.

(2) CHECKPOINTER_STATE_AT_STEP_BOUNDARY: pre_crash checkpoint sequence preserved verbatim in post_resume (first three UUIDs match exactly). Fourth checkpoint in post_resume is the replay's new state — appended on top of preserved pre-crash state, not overwriting. PostgresSaver behaves correctly under DBOS replay. No state drift.

(3) NODE_EXECUTION_COUNT: node_a=1, node_b=2.
  - node_a (outside crash window) executed exactly ONCE. DBOS replay did NOT cause langgraph to re-execute the already-completed node. This is the load-bearing assertion.
  - node_b (containing the SIGKILLed step) executed TWICE — original attempt + replay attempt. Expected behavior.

DUPLICATE_SIDE_EFFECTS=False + WARNINGS=[]: no silent state corruption, no langgraph runtime warnings.

=== WHAT THIS PROVES ===
@DBOS.step boundary aligns with langgraph node boundary. DBOS PENDING-state tracking, langgraph checkpointer state preservation, and replay-once-not-twice semantics all hold under SIGKILL.

The coarser composition that already shipped (runner.run_pipeline wrapping a langgraph graph in a DBOS workflow) was the outer surface. Landmine 2's specific question — the @DBOS.step boundary one level inside the workflow — is what this test verifies. Both granularities now have empirical confirmation.

=== WHAT THIS DOES NOT PROVE ===
- Behavior under crash AFTER the step body completes but BEFORE the step result is committed.
- Behavior across DBOS app-version changes mid-workflow.
- Production-scale concurrency. Single-workflow, single-step replay.

These gaps are acceptable for PR 2/3 close. The specific composition pattern handoffs.py + supervisor.py builds on — single workflow, mid-step crash, single replay — is verified.
