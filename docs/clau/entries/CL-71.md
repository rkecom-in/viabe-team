# CL-71 — Correction: 6th brief oversight (cross-app assignment)

Pattern: Clau's brief drafting has overreached 6 times in this session, all of similar shape — assuming cross-app data flow works without verifying each app has the pieces required.

This time: VT-3.3b brief assigned tenant lookup + rate limiting to team-web (per Notion VT-3.3 body). Claude Code surfaced the gap: team-web has no DB credentials in env, no Postgres client dep, and the per-app env lint rule (PR #5) explicitly forbids non-NEXT_PUBLIC_* vars in team-web except a tight whitelist.

Resolution: Option 2. Move tenant lookup + rate limiting to the orchestrator. Re-touch PR #9's /api/orchestrator/twilio-ingress contract (private internal API with no production callers; only 12 test cases need updating). Preserve Pillar 8 single-DB-access-path (only orchestrator has TEAM_SUPABASE_DB_URL). team-web stays thin: signature verification + forward + 200 to Twilio.

This means Notion VT-3.3 body's claim that team-web does the lookup is OVERRIDDEN by Pillar 8. Precedent established earlier this session: when Notion called for AGE on Supabase (impossible), we overrode; when Notion called for dev branch (out of date), we overrode; this is the same pattern. Pillars > Notion specificity.

Discipline rule to add to my brief-drafting checklist:

For every cross-app assignment in a brief, verify ALL of:
1. Target app has the env vars it needs (and they're allowed by the lint rule)
2. Target app has the client libraries/dependencies in its pyproject.toml or package.json
3. The data flow shape matches existing contracts (e.g., merged endpoint signatures)
4. If assignment requires breaking Pillar 8 (multiple apps with direct DB access, multiple migration mechanisms, etc.) — flag and consider an alternative architecture BEFORE drafting the brief
