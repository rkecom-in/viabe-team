# CL-34 — Architecture diagrams on a separate Notion page from concept diagrams

Architecture diagrams live on a separate Notion page (NOT inside the concept diagrams page) titled 'Viabe Team — Architecture Diagrams'. Concept page is for high-level orientation (what the system is). Architecture page is for engineering reference (how the system is built). Different audiences — different scroll context. Per-page focus is cleaner. Architecture page will contain three diagrams: (1) System Architecture (components + boundaries + harness vs loop separation), (2) Runtime Flow (single end-to-end pipeline run with all happy/failure/exception paths), (3) Data Flow + Step Tracking (where data lives, step record persistence, L0 distinct write flow).
