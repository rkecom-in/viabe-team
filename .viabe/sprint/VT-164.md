# VT-164 — Per-tenant attribution recovery-target wiring — replace inline *1.1 / ₹500-floor default

## Scope (written 2026-05-30 from brief 20260530T184226Z-brief-ready-e2e-batch.md, Wave A)

Replace the two hardcoded constants behind the agent's `target_recovered_paise`
with per-tenant configuration, read from the DB. Today's behaviour lives in
`apps/team-orchestrator/src/orchestrator/context_builder.py`:

- line 430: `_DEFAULT_TARGET_RECOVERED_PAISE: int = 50_000  # ~₹500 baseline target`
- lines 457-461: `target_recovered_paise = max(int(last_7d_recovered_paise * 1.1), _DEFAULT_TARGET_RECOVERED_PAISE)`

These feed the SR-agent prompt (`serialize_bundle_for_prompt`, rendered at
`context_builder.py:553`) and downstream attribution + projected-recovery —
the weekly_approval `{{3}}` figure.

### What changes
1. **Migration 051** (ASSIGNED up-front, CL-424 — do NOT run the allocator):
   add two per-tenant config columns to `public.tenants`:
   - `recovery_target_multiplier NUMERIC NOT NULL DEFAULT 1.1` (CHECK > 0)
   - `recovery_target_floor_paise INTEGER NOT NULL DEFAULT 50000` (CHECK >= 0)
   Backfill is the column DEFAULTs — every existing tenant gets exactly the
   current 1.1 / ₹500 behaviour, so NO behavioural change until a tenant
   overrides. Decision: **columns on `tenants`, NOT a new `tenant_config`
   table** (rationale in plan.md — config is 1:1 with tenant, low-cardinality,
   read on the same hot path that already opens `tenant_connection`).
2. A per-tenant config read — `_build_recovery_target_config(tenant_id)` in
   `context_builder.py` — via the canonical `tenant_connection` RLS path, with
   `assert_tenant_scoped` belt-and-braces. Returns `(multiplier, floor_paise)`,
   falling back to the dataclass defaults (1.1 / 50_000) when no row / NULL.
3. `serialize_bundle_for_prompt` computes the target from the per-tenant
   values instead of the module constants. The `_DEFAULT_*` constant stays as
   the in-code fallback default (matches the column DEFAULT) so a missing read
   never changes the number.
