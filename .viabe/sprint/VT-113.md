Fully configured email deliverability for Viabe Team's transactional emails. SPF + DKIM + DMARC records on [viabe.ai](http://viabe.ai) DNS; Resend sender domain verified; reputation warmed via gradual ramp; deliverability monitoring.
1. **DNS records** (Fazal owns DNS edits at registrar):
	- **SPF** record on [viabe.ai](http://viabe.ai): `v=spf1 include:_spf.resend.com ~all` (allows Resend to send from [viabe.ai](http://viabe.ai)). Soft-fail (~all) Phase 1; hard-fail (-all) Phase 2 once stable.
	- **DKIM** record(s): Resend provides 2-3 CNAME records (e.g., `resend._domainkey.viabe.ai → resend.cname-resend.dkim`). Added to DNS.
	- **DMARC** record on [viabe.ai](http://viabe.ai): `v=DMARC1; p=none; rua=mailto:dmarc@viabe.ai; ruf=mailto:dmarc@viabe.ai; sp=none; ri=86400`. Phase 1 = `p=none` (monitor only); Phase 2 = `p=quarantine`; Phase 3 = `p=reject`.
	- DMARC reports parsed via a third-party service (e.g., Postmark DMARC) or self-hosted parser; Phase 1 uses free-tier Postmark.
2. **Resend sender domain verification**:
	- In Resend dashboard, add `viabe.ai` as a verified domain.
	- Verification confirms DKIM CNAMEs resolved correctly + SPF includes Resend.
	- Verification status: green check before any production email sends.
3. **Sender addresses**:
	- Production: `noreply@viabe.ai` (transactional), `support@viabe.ai` (replies routed to Fazal), `dmarc@viabe.ai` (DMARC report inbox).
	- Configured in `apps/team/config/email_senders.yaml`.
4. **Reputation warming**:
	- Phase 1 starts with low volume (10-50 emails/day at launch).
	- Gradual ramp documented at `docs/runbooks/email-reputation-warmup.md`: Week 1 = 50/day; Week 2 = 200/day; Week 3 = 500/day; Week 4+ = production volume.
	- Resend's deliverability dashboard monitored for spam reports / bounce rates.
5. **Deliverability monitoring**:
	- Daily 10 AM IST scheduled trigger: queries Resend's deliverability API for the past 24 hours; flags bounce rate > 5% OR spam complaint rate > 0.1%.
	- Telegram alert to Fazal on threshold breach.
	- DMARC reports reviewed weekly (Phase 1 manual; Phase 2 automated alerting).
6. **Production cutover checklist** at `docs/runbooks/resend-cutover.md`:
	- DNS records verified via `dig` + Resend dashboard green status.
	- Sender domain verified.
	- Test email sent to Fazal's Gmail + corporate addresses (5+ providers); inbox placement verified.
	- DMARC monitoring active.
	- Rollback: revert to dev Resend account.
