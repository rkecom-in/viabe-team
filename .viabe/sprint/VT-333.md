# VT-333 — Founding-slot release on the cancelled-transition path (audit-only)

VT-94 wired `release_founding_slot` (audit-only — stamps `released_at`, NO counter
decrement; no-reopen policy) into the REFUND path (`refund_executor`). The CANCELLED
transition path was deferred — Cowork 20260605T143300Z: audit-only release carries ZERO
integrity risk (a missed release just leaves `released_at` NULL, never corrupts the
counter), so this is audit-completeness only.

## Scope
When a tenant transitions to `cancelled` (manual_cancel / cancellation_requested), call
`release_founding_slot(conn, tenant_id)` so the `founding_tier_claims` audit row is stamped
`released_at`. NO counter decrement (the no-reopen policy stands). `transitions.py` is the
generic SOLE phase-mutator — hook the release at the cancelled-transition completion point
(a post-transition hook, NOT inside apply_transition's deterministic core). Service-role
(founding_tier_claims has no app_role UPDATE policy).

## Canary
A tenant cancels -> `founding_tier_claims.released_at` set; `founding_tier_counter.claimed_count`
UNCHANGED.

## Origin
VT-94 #341 — the deferred cancelled-path release (Cowork Q2, 20260605T143300Z).
