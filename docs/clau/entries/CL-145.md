# CL-145 — VT-CI check for stale merge-base

Defense-in-depth for CL-144 (branching discipline). Primary mitigation is the four-command boilerplate in every brief; this CI check is the backstop.

Proposed check: GitHub Actions job on every PR that computes `git merge-base HEAD origin/main` and fails if the merge-base commit is more than 24h old (configurable) OR if the merge-base is not on main's first-parent history.

Would have caught PR #22: its actual base (feat/vt-orchestrator-agent-skeleton's tip) was not on main's first-parent history; merge-base with main would have been the commit BEFORE VT-3.9 started, which was 24h+ stale by the time PR #22 opened.

Scope: ~30 lines of YAML + a small shell script. (VT-CI) parent per CL-139. Phase 1 timing: after sarvam rename + langgraph migration + tailwind-merge bump are in. Not urgent because four-command rule is the primary defense and Claude Code reads briefs carefully now.

Closing condition: CI check live on main + at least one synthetic test PR (deliberately stale base) confirms it fires.
