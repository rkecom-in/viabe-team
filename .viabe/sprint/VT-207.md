# VT-207 — Google Sheet connector

## Why this row exists

Fazal directive: RKeCom doesn't have a digital stack yet → bootstrap with a Google Sheet of dummy customer + drop-off data. Sheet is also the simplest connector to ship (OAuth flow well-documented, no rate-limit headaches, no destructive writes). Prove the framework here; subsequent connectors slot in.

## Scope

### Auth flow

- Google OAuth 2.0 (PKCE), scope `https://www.googleapis.com/auth/spreadsheets.readonly`
- Token refresh handled; refresh_token stored per-tenant encrypted (re-use Fernet substrate from VT-191)
- New table `tenant_oauth_tokens` (tenant_id, connector_id, encrypted_refresh_token, scopes, last_refreshed_at)

### Connector adapter

`apps/team-orchestrator/src/orchestrator/integrations/connectors/google_sheet.py`:
- Implements `ConnectorBase` interface (defined in VT-205)
- `start_setup(tenant_id) -> auth_url`
- `complete_setup(tenant_id, auth_code) -> ConnectorConfig`
- `pull_sample(tenant_id, spreadsheet_id, range='A1:Z50') -> SamplePayload`
- `pull_full(tenant_id, since=last_sync_timestamp) -> list[CanonicalRow]`
- `setup_push(tenant_id, spreadsheet_id) -> Apps Script payload` (Apps Script template the owner copy-pastes into the Sheet's Extensions menu — calls `/api/orchestrator/integrations/sheet/push` on edit)

### Canonical schema mapping

Input: arbitrary Sheet columns. Output: `customers` + `drop_offs` table rows with `acquired_via='google_sheet'`.

Field-mapping reasoner (VT-209) handles the translation. This row provides the raw pull; mapping is shared substrate.

## Acceptance criteria

1. OAuth flow works end-to-end: owner clicks link → grants permission → token stored
2. `pull_sample` returns first 50 rows + column names
3. `pull_full` is incremental (uses `since=` cursor); doesn't re-fetch unchanged rows
4. Apps Script push works: owner pastes script into Sheet → adds row → orchestrator receives webhook within 5 min
5. Re-auth flow works when refresh_token expires (90-day default)
6. RKeCom dummy Sheet: 47 customer rows + 12 drop-offs → 47 + 12 canonical rows land in DB

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt207_sheet_connector.py` — 5 assertions: OAuth callback handling, sample pull, incremental full pull, Apps Script webhook receive, re-auth flow.

## Out of scope

- Bidirectional sync (Sheet → DB only; not DB → Sheet)
- Sheet creation (assume owner provides existing Sheet)
- Service account auth (only OAuth in Phase 1)
