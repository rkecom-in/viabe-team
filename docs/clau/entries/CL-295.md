# CL-295 — BLOCKER: PR #40 real-Opus acceptance test reported PASSED but did not execute — 

PR #40's real-Opus acceptance test (test_cl288_real_opus_emit_shape_round_trips_through_parse, added per CL-290) reported PASSED but DID NOT EXECUTE a real Opus call. Evidence: the 2026-05-21 verification run completed 5 tests in 7.05s total; the 4 pre-existing *construct_variant_payload unit tests run in 0.73s, so the new fifth test contributed ~6.3s — far below a real Opus agent-loop inference (15-40s+ with tool turns). ~6s is consistent with heavy-dependency import + collection, NOT a live model call.

ROOT CAUSE — two candidates, not yet disambiguated: (a) Fazal re-ran the OLD exploratory -k command (RUN_INTEGRATION_TESTS=1 uv run pytest -k 'sales_recovery and (...emit)') instead of the explicit ::test_cl288*... command Claude Code supplied. The -k command sets RUN_INTEGRATION_TESTS=1 but does NOT set ANTHROPIC_API_KEY or DATABASE_URL. If those are unset in shell, the test SHOULD have skipped on its skipif — but it reported PASSED, meaning the skipif gate is wrong (RUN_INTEGRATION_TESTS=1 alone satisfied a gate that should require the two credential vars). (b) If the credential vars ARE exported, the test ran but in ~6s with no real call — meaning a mock leaked in or the SDK-boundary patch swallowed the call. Either way: a real-Opus test that passes in 6s did not call Opus.

This is the THIRD dead-canary occurrence in the PR #39/#40 thread (CL-272 PR #39 canary skipped-but-reported-green; CL-286 canary skip-gated by undocumented third env var; now CL-291). Recurrence pattern: tests that can report PASS on a path they did not execute.

REQUIRED RESOLUTION: (1) Fazal confirms echo $ANTHROPIC_API_KEY / echo $DATABASE_URL non-empty, then runs the EXPLICIT pytest ::test_cl288_real_opus_emit_shape_round_trips_through_parse command (not -k). (2) Claude Code surfaces the test's exact skipif condition AND its assertion block — the test MUST contain an assertion that fails if no real Opus call occurred (call-count on the patched Anthropic client, latency floor, or equivalent). A real-model test that can pass while skipped or mocked is not a real-model test. Per CL-286: a canary that passes while skipped/mocked is worse than no canary.

PR #40 stays HELD. Sequence unchanged: #40 (with a VERIFIED-executing real-Opus test, green) -> #39 rebase -> #39 canary -> #39 merge.
