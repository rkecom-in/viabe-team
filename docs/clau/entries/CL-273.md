# CL-273 — BLOCKER: PR #37 held from merge — (1) real-API canary makes no live call, report

PR #37 (VT-50 self_evaluate tool) held from merge — two defects found at manual pre-merge verification, neither catchable by CI.

DEFECT 1 (Critical): the real-API canary test test_canary_real_haiku_run_returns_verdict does not make a live Anthropic call. Two manual runs by Fazal — first without the env flag, second WITH VIABE_RUN_SELF_EVALUATE_CANARY=1 set and ANTHROPIC_API_KEY exported — both completed in ~2.4-3.1s and reported PASSED. A real Opus 4.7 call evaluating a full CampaignPlan draft cannot complete in 2.4s; no API traffic occurred. The canary's env-gating logic is broken: when gated ON it must take a genuine live path and assert on the live response; when OFF it must SKIP (not PASS). As shipped the test reports PASSED whether or not the gate actually works — a green check with no verification behind it. CI never caught this (no key in CI → skips). Additionally the model discrepancy is unresolved: test named '...haiku...' but VT-50 models.yaml pins Opus 4.7 for self_evaluate; the canary must exercise the production model (Opus 4.7), not Haiku.

DEFECT 2 (High): Pydantic field-name shadowing. self_evaluate.py:118, class *FeedbackPayload — field 'schema' shadows BaseModel.schema(). The four feedback categories are schema/pillar/consistency/legal, and 'schema' collides with Pydantic's own method. Currently only a UserWarning but it sits on the verdict-carrying model of the quality gate; Pydantic field/method shadowing can break model_json_schema()/serialization depending on version. Fix: alias the Python attribute (e.g. schema_ with Field(alias='schema') + populate-by-name) keeping the JSON wire key 'schema'; confirm the VT-36 gate's SelfEvaluateFeedback consumer still reads the four fields.

RESOLUTION: one fix-commit cycle on branch feat/vt-mcp-self-evaluate before merge — Claude Code briefed with both defects. CI must stay 12/12. ROOT CAUSE on Defect 1: the VT-50 brief asked for an env-gated real-API integration test; what shipped reports PASSED instead of SKIPPED when ungated and does not make a live call when gated — mocked-test green did not surface it, and the in-session naming flag (haiku vs Opus pin) raised earlier was the visible thread that led here. Manual pre-merge verification did its job; the canary must be genuinely fixed, not just re-run.
