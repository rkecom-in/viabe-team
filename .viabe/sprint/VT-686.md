# VT-686 — every specialist carries its own identity card

## The ask (Fazal, verbatim intent)
GST agent example: category "Compliance", tags identifying GSTN capabilities (gstr1/gstr3b/…),
plus a full capability brief — what it can do, what actions/business activities it performs —
rich enough that the MANAGER understands the agent and knows when to delegate to it.

## Design
1. **AgentManifest gains** (agent_framework/manifest.py, validate() extended):
   - `category: str` — ONE of a finite registry set (AGENT_CATEGORIES: Compliance / Sales /
     Marketing / Finance / Accounting / Onboarding / Integration / Tech / CostOpt) — boot-fails
     on an unknown category (the CapabilityMode discipline).
   - `tags: frozenset[str]` — free-vocabulary lowercase identifiers (e.g. {"gst","gstr1",
     "gstr3b","returns","filing-readiness"}); validated shape only (lowercase, no spaces).
   - `brief: AgentBrief` — a STRUCTURED dataclass, not prose blob:
     `what_it_does` (1-2 sentences) · `actions` (tuple — concrete verbs it performs) ·
     `business_activities` (tuple — owner-recognizable outcomes it completes) ·
     `when_to_use` (delegation guidance for the Manager) · `limits` (what it does NOT do —
     honesty-first, mirrors the capability registry's disabled entries).
2. **Conformance check #10**: `brief_complete` — category valid, ≥1 tag, every brief field
   non-empty, limits non-empty (an agent that claims no limits fails).
3. **Manager consumption**: `render_agent_directory()` — a compact per-agent card
   (category/tags/what/when/limits) rendered into the Manager's per-turn context alongside the
   VT-681 capability-truth block (post-prefix SystemMessage family, cache holds), so delegation
   decisions are grounded in the briefs, not spawn-tool docstrings.
4. **Retrofit**: SR module + onboarding-conductor module + common/integration tools modules +
   the VT-685 compliance skeleton all gain category/tags/brief. EXTERNAL-BUILDER-ONBOARDING §2
   documents the new fields as REQUIRED.
5. Registry linkage: CapabilitySpec.lane stays per-capability; manifest.category is per-AGENT —
   the directory card cross-references its capability keys so promise-truth and agent-identity
   never diverge.

## Gate
Unit (manifest validation + conformance #10 + renderer) · retrofit modules conformance-green ·
brain-touch (directory block in context) → ×3 full-pack re-drive before trusting.
