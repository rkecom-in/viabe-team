# VT-527 — B4 generic pending-questions

## What
The manager's clarify/confirm loop for ARBITRARY tasks — the mechanism B3's CLARIFY decision reaches for (record_decision parks a task `waiting_owner`; this carries the actual question to the owner + correlates the reply). Deliberately separate from `onboarding_journey` (singular-per-tenant, reset-on-restart); a general mechanism needs multiple concurrent open questions across tasks.

## Built (mig 153)
- **`pending_questions` table** — soft `task_id`/`run_id` pointers (no FK, VT-521 lesson), REDACTED `question_text` + `answer_text`, status `open→(answered|expired|cancelled)`, `last_message_sid` (idempotency), `expires_at` (TTL). One-open-per-(tenant,task) partial unique + an expiry sweep index (mirrors `pending_approvals`). RLS+FORCE + operator SELECT + in `dsr_purge._PURGE_ORDER` + erasure-proven (same PR).
- **`pending_questions.py`** — `ask` (idempotent per task), `correlate_reply` (redelivery-guarded by `message_sid` — a duplicate Twilio webhook is an idempotent no-op regardless of status; terminal-safe, first answer wins), `expire_stale` (service-role best-effort sweep), `get_open`. Reuses the journey idempotency + pending_approvals expiry patterns — no third mechanism invented. Redact-at-write; tenant-scoped via `tenant_connection`.

## Verify
ruff clean; mig 153 applies; 21/21 on fresh DB — ask redacts + is per-task idempotent, reply answers + redacts the owner text, redelivered sid is a no-op success, terminal-safe (a late reply finds nothing open → None, first answer preserved), expiry sweeps past-TTL only, RLS isolation, + pending_questions DSR-erasure proof.

## NOT in scope (follow-ups)
Cross-task **running-context** (owner prefs / confirmed facts / recent decisions) — a separate net-new substrate (VT-527b). **VTR-in-the-loop LIVE resume** — inverts CL-426 async-VTR, **blocked on Fazal**; this table serves the owner-ask + keep-async default, a VTR answer path layers on later. Wiring `ask` into B3's CLARIFY branch + the reply into the inbound webhook = the B3 graph-wiring slice.
