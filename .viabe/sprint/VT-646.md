## Problem (j01 Tier-1: ignored_speech_act)
"how many customers haven't ordered in a while" → answered with the TOTAL ledger count (10), not the
LAPSED count (8). Root: `status_query.classify` routed the behavioural-dormancy phrasing (no explicit
"lapsed"/"dormant" token) to `customer_count` (via the "customers" token) — a deliberate prior pin
(test_edge_cases.py:44 → brain), but the brain answered TOTAL (the ignored_speech_act breaker).

## Fix (Cowork applying Fazal's 45d ruling CL-2026-07-10 — deterministic)
New `_has_count_cue(norm, tokens)` (how many / how much / kitne / kitni / कितने) + the existing
`_has_inactivity_cue` (adjacency-bound "haven't ordered" / lapsed token / recency) → route a
behavioural-dormancy COUNT to `lapsed_count` (45d), BEFORE customer_count and only when NOT a list ask.
Corrected the pin (test_edge_cases.py:44) to expect `lapsed_count` — this APPLIES the 45d ruling
("haven't ordered in a while" honestly means dormant), it is not gaming. BOUND: a plain "how many
customers do I have" (no dormancy cue) still = customer_count (total).
