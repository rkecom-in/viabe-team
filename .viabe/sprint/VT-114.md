## What CoderC must produce
LangSmith billing tier upgrade from free to a paid plan that supports production trace volume. Cost forecast + Fazal sign-off. Activation timed to soak run start (so soak doesn't get throttled).
1. **Trace volume forecast**:
	- Phase 1 launch target: 100 tenants × ~5 dispatches/week × ~10 spans/dispatch = ~5,000 spans/week = ~700/day average.
	- Soak run: 50-100 synthetic tenants × 500-1000 events/day = ~5,000-10,000 spans/day during soak.
	- Production growth (Phase 1): 100→500 tenants in 6 months = ~25,000-50,000 spans/day at 500 tenants.
	- Free tier: 5,000 traces/day. Phase 1 will exceed within ~2-4 weeks of launch.
2. **Paid tier selection**:
	- LangSmith paid tiers (as of 2026):
		- **Plus** (~$39/month): 50K traces/month, 14-day retention. Insufficient for production scale.
		- **Team** (~$99/month base): 500K traces/month included, 28-day retention, additional traces ~$0.0008 each.
		- **Enterprise** (custom): unlimited traces, custom retention, SOC 2 access.
	- **Phase 1 recommendation: Team tier**. Rationale: covers Phase 1 launch volume + 6-month growth; 28-day retention enables debugging; cost predictable.
	- Cost forecast: Team tier base $99/month → ~$1,200/year. At 500-tenant scale, additional trace fees ~$200-400/year. Total ~$1,500-2,000/year.
3. **Activation timing**:
	- Activate Team tier 1 week before soak run start (give billing time to process).
	- Verify quota uplift via test trace burst.
4. **Quota monitoring**:
	- Daily 9 AM IST scheduled trigger: queries LangSmith billing API for current month's trace count vs quota. Alerts Fazal if usage >80% of monthly quota mid-month.
	- Soft cap: trace sampling at 90% of quota (drop low-severity traces; retain critical-severity always).
5. **Cost dashboard integration**:
	- LangSmith cost added to VT-12.3 cost dashboard as a new category (`langsmith_observability`).
	- Per-tenant cost allocation: workspace LangSmith cost ÷ tenant trace share.
6. **Production cutover checklist** at `docs/runbooks/langsmith-tier-upgrade.md`:
	- Billing tier upgraded.
	- Production project (`viabe-team-prod`) confirmed in paid tier.
	- Quota monitoring active.
	- Sampling logic deployed but inactive (activates at 90% quota).
	- Rollback: drop to Plus tier if costs exceed forecast.
