# CL-135 — Context7 validation required for every brief

Fazal's standing instruction 2026-05-18: every brief Clau writes that references a library, framework, or API surface MUST pass through Context7 validation BEFORE the brief ships to Claude Code. Validation is on the specific call signatures and patterns in the brief, not just 'does the library exist'.

Why: Clau's training data has a cutoff. Libraries deprecate APIs (see CL-134 — create_react_agent deprecated in langgraph 1.2.0, removed in V2.0; Clau's brief used it because training data didn't know). Briefs built on stale APIs cause Claude Code to ship working-but-deprecated code, which compounds tech debt and risks bumps blocking us later.

Validation pattern (mandatory):
1. Identify every external library/framework/API surface referenced in the brief.
2. For each: Context7:resolve-library-id then Context7:query-docs on the specific call surfaces (function names, kwarg signatures, import paths, deprecation status).
3. If Context7 surfaces a deprecation, a newer canonical API, or a breaking change since Clau's training data — ADJUST the brief to use the current API, log a Correction entry, and proceed.
4. If Context7 surfaces a meaningfully better current approach (not just the API still working but a new pattern that's now canonical), surface to Fazal as a Type 2 trade-off, do not silently adopt.
5. The brief includes a note to Claude Code naming which Context7 lookup informed the API choice, so Claude Code can re-verify if uncertain.

Skip-allowed cases (narrow): pure internal modules with no external dep surface (e.g., a brief that just moves a function across our own files); pure documentation or config tweaks; brief is a follow-up patch to a Context7-validated brief touching the same surfaces (re-validation only if a week+ has passed or the deps were bumped).

Violation cost: stale APIs in shipped code, compounding migration tech debt, risk of being blocked when a dep bump becomes urgent.

This rule applies to ALL future briefs, audit work, second-set-of-eyes work, and any architectural recommendation that references external tech. Failure to validate is a Correction-worthy lapse.
