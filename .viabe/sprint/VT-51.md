## What CoderC must produce (IF approved in v1)
The `draft_message_variants` MCP tool. LLM-backed. Given a campaign context, produces 2-3 alternative phrasings of the message_params for an approved template. The agent picks one to ship.
1. **Tool implementation** at `apps/team/src/agent/tools/draft_message_variants.py`:
	Input schema:
	- `template_id: str` (must be in approved registry).
	- `cohort_summary: CohortSummary` (cohort_label, recency_band, dominant_acquired_via, customer_count).
	- `business_profile: BusinessProfileSummary` (business_type, locality, preferred_language).
	- `tone_target: Literal['warm_conversational', 'concise_professional', 'celebratory_event'] | None` (agent-specified; null means evaluator picks).
	- `variant_count: int = 3` (max 5).
	Output schema:
	- `variants: list[MessageVariant]` — each: `template_params: dict[str, str]` (matching the template's spec), `tone: str`, `rationale: str` (<200 chars; why this variant fits the cohort/profile).
	- `model_used: str` (Sonnet 4.6 pinned; cheaper than Opus, more capable than Haiku for nuanced phrasing).
	- `tokens_used: int`, `cost_paise: int`.
2. **Honesty enforcement at variant generation**:
	- Prompt instructs the model: "Each variant MUST avoid high-pressure language ('this is your last chance', 'don't miss out'). Avoid false urgency. No fabricated discount amounts. No claims that aren't backed by the cohort context."
	- Post-generation: each variant's `template_params` are checked against the same regex patterns used by the orchestrator's composer (VT-3.7 Pillar 7 honesty layer). Variants failing the check are dropped from output; the tool returns fewer variants if needed.
3. **Template parameter compliance**:
	- The template's parameter spec is fetched from `approved_templates.yaml`. Each variant's `template_params` keys must match exactly. Missing required params → variant dropped.
	- Parameter values must respect Meta template constraints (max length, no URL, no emoji-spam, etc.).
4. **Prompt** at `apps/team/src/agent/tools/prompts/draft_message_variants_v1.md`:
	- System prompt: "You are a copywriter for an Indian SMB's WhatsApp re-engagement campaign. Produce 2-3 phrasing variants for the given template parameters. Variants should sound authentic to small-business owners writing to their dormant customers — not corporate, not overly formal. Hindi/English/mixed-language as appropriate to the tenant's preferred_language."
	- Honesty constraints listed verbatim.
	- Examples of good variants vs bad (high-pressure rejected).
	- Token budget under 2,500.
5. **Cost handling**:
	- Sonnet 4.6 estimated ~₹1-3 per call. With 5-10 calls per agent run and ~50 runs/day: ~₹500-1500/day. Within Phase 1 budget if approved.
6. **Tests** at `apps/team/tests/agent/tools/test_draft_message_variants.py`:
	- Happy path: known template + cohort + profile → returns 3 variants, all pass honesty regex, all have correct param keys.
	- Pressure language filtering: synthetic prompt that tries to elicit pressure language → variants dropped from output; fewer than requested returned.
	- Template not in registry: error envelope.
	- Hindi-language preference: tenant `preferred_language='hi'` produces Hindi-language variants.
	- Mixed-language: tenant in mixed market produces mixed-language variants if appropriate.
	- Cross-tenant: ctx.tenant_id has no effect on variant content (stateless tool); logged only.
	- Real-API integration test (env-gated).

## Architectural rules to enforce in this subtask
- Pillar 2: LLM-backed; rationale documented. Sonnet (not Opus, not Haiku) is justified by cost/quality balance for phrasing tasks.
- Pillar 7: honesty regex layer is identical to the composer's (VT-3.7). Variants that pass this tool but fail the composer would be a Pillar 7 contradiction — prevented by sharing the regex set.
- The tool is stateless on tenant data; ctx.tenant_id logged only.
- Variants are SUGGESTIONS; the agent chooses one. The tool does not ship a variant directly.
- Output is strict JSON.
- Prompt versioned.
- This tool counts as one tool call against the agent's 25-call cap.
