# VT-531 — C3 reviewer-correction store (capture half)

## Gap
When an owner edits/rejects an agent draft, the raw `owner_feedback` lands on `agent_draft_batches` and — the instant the batch goes terminal — `redact_batch_close` overwrites it with `redacted:sha256:<hex>` (**content destroyed**). Only an aggregate regression counter survived; the single most valuable learning signal ("what the owner actually said to fix it") was gone.

## Built (mig 154)
- **`agent_corrections` table** — append-only, one row per correction, keyed by (tenant, batch) with soft run_id/batch_id pointers (no FK). `correction_text` is **PII-REDACTED (pii_redactor), NOT sha256'd** — the substance survives ("make it softer") while customer PII is stripped. `correction_kind` reuses `RegressionKind` (`edit`/`reject`). **Retrieval-gate placeholder cols default-CLOSED** (`retrieval_eligible=false`, `authority`, `expires_at`) — capture-now, retrieve-later; Phase-2 needs no ALTER. RLS+FORCE + operator SELECT + in `_PURGE_ORDER` + erasure-proven (same PR).
- **`correction_store.record_correction`** — writes on the caller's resolution conn inside a **SAVEPOINT** (atomic with the batch transition when it succeeds; a capture failure rolls back only the savepoint + is swallowed — the Pillar-7 approval resolution never breaks on this write).
- **Wired 3 capture points in `apply_agent_decision`**, each **BEFORE** its `redact_batch_close`: the edit_requested (1st needs_changes → `edit`), the terminal-reject (2nd needs_changes → `reject`), and the direct `rejected` branch. Timeout/defer (no owner correction) skipped.

## Verify
ruff clean; mig 154 applies; **27/27** — the 4 new capture tests (edit captured, reject captured, PII redacted, retrieval default-closed) PLUS the key **survives-redact** test (after the terminal close the batch's `owner_feedback` is `redacted:sha256:…` but `agent_corrections` still holds the readable "make it softer") — AND every pre-existing approval_glue test unchanged (the 3 wires didn't alter the resolution flow) + agent_corrections DSR erasure.
