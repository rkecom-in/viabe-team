# CL-58 — uv and Ruff retained — OpenAI ownership accepted

uv and Ruff retained as Python toolchain despite OpenAI's March 19 2026 acquisition of Astral (parent company).

Reason for acceptance:
1. Tools remain MIT/Apache 2.0 licensed — forkable if priorities shift badly
2. No alternative comes close on speed/UX (uv is 10-100x faster than pip; Ruff is 1000x faster than legacy linters)
3. Anthropic acquired Bun (JS runtime used by Claude Code) in Dec 2025 — same dynamic exists in our JS toolchain anyway, can't fully insulate
4. Switching would impose real productivity cost for theoretical strategic benefit

Accepted risks:
- Roadmap drift toward OpenAI Codex priorities over time (community-facing features may deprioritize)
- Strategic optics: Anthropic-powered product running on OpenAI-owned dev toolchain (operationally fine, messaging-uncomfortable)
- 'Open source capture' pattern: source stays open, but who-the-roadmap-serves shifts

Mitigations:
- Pin uv and ruff versions in pyproject.toml; treat upgrades as deliberate decisions, not auto-bumps
- Set GitHub notifications on astral-sh/uv and astral-sh/ruff repos to spot priority shifts early
- Pre-evaluated fallbacks: Poetry (Python pkg mgmt), Pyright/mypy (typecheck) — inferior today but viable if needed
- Trigger to revisit: if community-filed issues start getting noticeably slower responses while Codex-integration issues get instant attention, OR if uv/Ruff ships Codex-exclusive features

No immediate changes to repo or stack. Decision is conscious risk acceptance.
