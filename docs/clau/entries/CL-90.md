# CL-90 — Correction to CL-82: Status set by Fazal pre-merge

CORRECTION to CL-82 (Notion status sync via Claude Code MCP). Simpler and more reliable mechanism.

FINAL MERGE RITUAL (Fazal 2026-05-18 after pushback):
1. Before approving merge: Fazal manually sets the Notion subtask's Status property to 'Done' (~15 seconds, one click in the ViabeTeam_Sprint board).
2. Run `gh pr merge N --squash --admin --delete-branch`.
3. If the PR fixes a Blocker entry in Clau_Session_Log, Fazal also marks that entry Status='Resolved' (another ~15 seconds).

Why this is better than the prior 'Claude Code does it post-merge' approach:
- No dependency on whether Claude Code's Notion MCP server is connected with update_properties write access.
- No tool-approval-layer risk (bulk update_properties denied per Layer 1 instructions; single-page works but Claude Code's exact MCP config is unverified).
- Status update happens in Fazal's review flow when the PR context is already loaded.
- ~15 seconds total vs. a tool-call cascade.
- Avoids the failure mode of merge happening + Notion sync failing silently.
