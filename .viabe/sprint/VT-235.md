# VT-235 — Ops Console styling pass

## Why

Fazal manual walk 2026-05-29 18:40 IST: magic link → finalize → operator JWT cookie → /team/ops opens. AUTH WORKS. But the Ops Console workspace renders as unstyled HTML:

- Banner: "Escalations 24h: 0Hard-limits 24h: 0Errors 24h: 0refreshed 11:23:32 PM" — labels + values run together with no separators
- KPI block: "In-flight runs 11", "Total runs today 0", "Escalations today 0", "Cost burn today (paise) 0" — no card hierarchy, no spacing
- Tables (Top tenants by activity, In-flight runs) — bare HTML tables with no styling

Root cause: pages shipped during VT-123 (Ops UI MVP), VT-201 (sticky banner + history view), and VT-202 (proactive alerts UI surface) used no Tailwind utility classes. Same architectural bug as VT-203 that VT-232 fixed for /team/ops/login. The fix at /team/ops/login was isolated — surrounding pages still unstyled.

## Brief contract

CC implements a **styling-only pass** across the entire Ops Console surface. NO functional logic changes. NO new auth surface. NO new env vars.

### Pages to style (Tailwind utility classes only)

1. `apps/team-web/app/(app)/team/ops/page.tsx` (workspace)
2. `apps/team-web/app/(app)/team/ops/runs/[runId]/page.tsx` (per-run detail)
3. `apps/team-web/app/(app)/team/ops/stream/page.tsx` (live stream)
4. `apps/team-web/app/(app)/team/ops/stream/history/page.tsx` (history view)
5. `apps/team-web/app/(app)/team/ops/tenants/[tenantId]/page.tsx` (per-tenant view)
6. `apps/team-web/app/(app)/team/ops/layout.tsx` (the shell wrapping the banner)

### Components to style

1. `apps/team-web/components/ops/sticky-banner-live.tsx` — the banner overlay
2. Any pill / KPI / table component under `apps/team-web/components/ops/*`

### Design spec (baseline)

Match the VT-232 login card aesthetic:
- Page background: `bg-gray-50`
- Cards/sections: `bg-white rounded-lg shadow-sm border border-gray-200 p-6`
- Typography: `text-2xl font-semibold text-gray-900` for h1, `text-lg font-medium text-gray-800` for section heads, `text-sm text-gray-600` for body
- Banner: pill-style metric chips with spacing — `flex gap-4 items-center px-4 py-2 bg-amber-50 border-b border-amber-200`; each metric `inline-flex gap-2 items-center text-sm`
- KPI grid: `grid grid-cols-2 lg:grid-cols-4 gap-4`; each KPI card `p-4 bg-white rounded-lg shadow-sm` with `text-3xl font-bold text-gray-900` for the number + `text-xs uppercase tracking-wide text-gray-500` for the label
- Tables: `divide-y divide-gray-200`; header row `bg-gray-50 px-4 py-2 text-xs font-medium uppercase text-gray-500`; body rows `px-4 py-3 text-sm`
- Run/tenant ID columns: `font-mono text-xs text-gray-700`
- Timestamps: `text-sm text-gray-500`
- Hover states on table rows: `hover:bg-gray-50 transition-colors`

CC may adjust within the spirit — but every element must have either a Tailwind utility class OR a component class that maps to Tailwind utilities. No bare `<table>`, `<div>`, `<h1>` without classes.

### Sticky banner specific spec

```tsx
<div className="sticky top-0 z-10 bg-amber-50 border-b border-amber-200 px-4 py-2">
  <div className="flex flex-wrap gap-6 items-center text-sm">
    <span className="flex gap-2 items-center">
      <span className="text-xs uppercase tracking-wide text-amber-700">Escalations 24h</span>
      <span className="font-semibold text-amber-900">{counts.escalations_24h}</span>
    </span>
    <span className="flex gap-2 items-center">
      <span className="text-xs uppercase tracking-wide text-amber-700">Hard-limits 24h</span>
      <span className="font-semibold text-amber-900">{counts.aborted_hard_limits_24h}</span>
    </span>
    <span className="flex gap-2 items-center">
      <span className="text-xs uppercase tracking-wide text-amber-700">Errors 24h</span>
      <span className="font-semibold text-amber-900">{counts.errors_24h}</span>
    </span>
    <span className="ml-auto text-xs text-amber-600">refreshed {timeFmt(counts.refreshed_at)}</span>
  </div>
</div>
```

If any banner count > 0, intensify the visual (e.g., `bg-red-50` if errors_24h > 0).
