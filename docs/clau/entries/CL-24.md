# CL-24 — Orchestrator-as-agent locked: Opus 4.7 brain + own memory + spawns specialists; 

Orchestrator is an agent (not a state machine). Opus 4.7. Its own context bundle, its own memory tier (L0 workspace-level, separate from L1-L4 which are tenant-scoped knowledge). LangGraph is the substrate the orchestrator-agent runs on, not the orchestrator itself. Two-stage filtering: cheap deterministic pre-filter (regex/signature checks for opt-outs, DSRs, status pings) handles ~70% of events without invoking the brain; orchestrator brain handles the remaining ~30% requiring judgment. Spawning specialists becomes a tool call: spawn_specialist(name, context). Orchestrator decides: spawn specialist, run simple pipeline (deterministic), respond directly with own context, or listen-and-remember. Phase 1 still ships only Sales Recovery specialist; orchestrator's spawn-tool has exactly one specialist option. Phase 2-5 grow the specialist set without changing the orchestrator pattern. Ingestion scope unchanged: 9 methods per concept doc Section 5. Connected systems (ERP, Accounting, Marketplaces, Amazon Seller Central) remain Phase 3+ per concept doc roadmap.
