# CL-322 — CORRECTION + discipline rule #12: a session escalated an AGE-hosting crisis off 

## What happened

A session investigating VT-7.1 read the subtask row whose TITLE was 'Apache AGE setup + L1 KG schema', took AGE as the current stack, and escalated a manufactured crisis to Fazal: 'AGE can't run on managed Supabase, this could mean migrating off your database.' Fazal challenged it; the session searched Clau_Session_Log, found the 2026-05-16/17 decision retiring AGE in favour of Mem0 OSS + Postgres + pgvector, and corrected itself.

## The real root cause — stated precisely

This was NOT a logging failure. The AGE retirement was logged (decision 363387c2-cc5a-813d-8a4b-dc07866b6119).

It was NOT a propagation failure. The VT-7.1 row ALREADY carried a prominent SUPERSEDED banner as its first content block: '⚠️ SUPERSEDED 2026-05-16 — Apache AGE REPLACED by Mem0 OSS library', with the full new Mem0 approach spelled out. Discipline rule #9 had been applied correctly back on 2026-05-16.

The actual failure: the session acted on the row's TITLE and PROPERTIES without reading the row BODY, where the correction sat two lines into the content. It then built an escalation to Fazal on a premise the row itself already refuted.

## Two fixes applied

1. Row title corrected. The VT-7.1 row title was changed from 'Apache AGE setup + L1 KG schema' to 'L1 KG schema (Mem0 OSS — AGE retired, see body)'. A SUPERSEDED banner in the body is insufficient when the TITLE still asserts the dead technology — titles are what sessions and search hits see first. Superseding corrections must reach the title, not only the body.

2. Discipline rule #12 (below).

## Discipline rule #12 — verify row bodies, not just titles

Before briefing work against a ViabeTeam_Sprint subtask row, OR escalating any problem derived from one, the session MUST read the row's full BODY — not only its title and properties. Specifically: scan for a SUPERSEDED / superseding / 'out of date' banner before treating any technology choice, scope claim, or dependency in the row as current. A subtask row title can lag the architecture by weeks; the body (and the decision log) is ground truth. A title is a label, not a fact.

This is the escalation-side companion to rule #7 (pre-brief audit-blocker check) and rule #9 (supersession discoverability). Rule #9 ensures the correction is WRITTEN where it can be found; rule #12 ensures the session actually READS it before acting.
