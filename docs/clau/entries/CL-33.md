# CL-33 — Phase 1 durability posture: accept LangGraph checkpointer-only gap

Phase 1 accepts the LangGraph checkpointer-only durability gap. Concrete consequences: (a) If Railway crashes mid-run, no automatic failure detection — Fazal manually re-triggers via workflow_dispatch or replay tool. (b) No automatic resumption — stuck runs require human intervention to invoke with the saved checkpoint_id. (c) No duplicate-execution prevention if two processes try to resume the same thread_id simultaneously (low risk Phase 1 with single Railway service + low concurrency). Mitigation in Phase 1: (i) Daily Telegram alert (VT-121) flags any run with status=running and started_at \> 10 minutes ago (stuck-run detection). (ii) Postgres advisory locks on run_id at orchestrator entry to prevent concurrent execution attempts. (iii) Manual replay tool (VT-12.4 + Ops UI Phase 2 mutation actions). Phase 2 trigger (re-evaluate at ≥200 subscribers): consider layering DBOS or Temporal on top of LangGraph for automatic failure detection + auto-resume + distributed coordination.
