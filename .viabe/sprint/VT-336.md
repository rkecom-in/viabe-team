# VT-336 — VT-84 PR-1 hardening (4 subagent nits; none blocking — reversible action + test gaps)

VT-84 PR-1 #347 Cowork subagent nits:
1. **Confidence floor for exclusion_request** — `route_edge_case` ignores Haiku confidence;
   a low-confidence misroute hits the exclusion handler. Mitigated (deterministic resolution
   + reversible flag + owner ack), but add a confidence floor for `exclusion_request`
   specifically (status_query is read-only — fine without).
2. **Tighten `_extract_phone` (exclusion.py)** — the `\d[\d\s-]{8,13}\d` regex grabs a 10-14
   digit invoice/order number as a phone; fail-safe to not_found UNLESS it collides with a
   real customer phone → a wrong (reversible) exclude. Require a name/phone keyword context
   or validate the E.164 shape more strictly.
3. **Same-phone-cross-tenant test** — the current cross-tenant test uses DIFFERENT phones;
   add tenant A + tenant B SHARING a phone, assert A's exclude can't touch B's row (proves
   the tenant-predicate, not just unrelated rows).
4. **Consumer-STOP-then-owner-exclude test** — the precedence test used
   `customers.opt_out_status='opted_out'` (wrong store). Add the REAL path: consumer STOP →
   `record_of_consent.opted_out_at` set, owner then excludes → assert the VT-45 consent gate
   still fail-closes the send. The guarantee is structurally correct (separate tables) but
   unproven by a green test.

5. **End-to-end adhoc-no-bypass keystone test** (Cowork, VT-335 review 20260605T170900Z) —
   the adhoc-never-sends-without-approval invariant is currently locked by UNIT tests
   (route_after_approval ignores trigger_reason + the router doesn't send). Add ONE
   integration test that drives the supervisor graph with `trigger_reason='owner_initiated'`
   and asserts it PAUSES at the approval `interrupt()`, NOT reaches `campaign_execute`. For
   THE Pillar-7 send invariant, green-locked beats trace-reasoned.
6. **Owner-portal IDOR/clamp regression tests** (Cowork, VT-338 #354 review) — add a
   `page_size=500 → clamped-to-100` assertion on dashboard_customers + a page-level
   session-tenant test (a crafted request can't read another tenant) to lock the IDOR
   boundary against regression.
