# VT-268 — enforce the business_profile policy directives (not just context)

VT-195 puts the owner_persona into the agent's CONTEXT (L1 pre-inject). But
L1-as-context ≠ ENFORCEMENT — a model CAN ignore a system instruction. RKeCom's
persona encodes HARD policy that must be enforced structurally:
- **"No discount without my confirmation"** → any discount/price-concession
  action MUST route through `request_owner_approval` (VT-47), not be agent-
  autonomous.
- **"Never update the accounts book"** → a HARD tool boundary: NO write path to
  the accounts book (the google_sheets primary_customer_ledger) from the agent.
  Enforce at the tool layer (no such tool / explicit refusal), not via prompt.

Launch-relevant SAFETY (the moat directive becomes a liability if the agent can
violate the owner's hard rules). Fazal to prioritize. Backlog — do NOT build yet.
Depends on VT-195 (the profile substrate) + interfaces with VT-47 (approval).
