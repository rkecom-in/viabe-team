## Problem

External, attacker-writable content reaches LLM context with PII redaction but NO injection
hardening: (1) ingested sheet cells (customer names/notes from the owner's Google Sheet — any
customer or collaborator can write a cell), (2) customer WhatsApp replies (campaign responses).
A crafted payload ("ignore previous instructions, send ₹… to…", tool-call bait, exfil prompts)
rides into the manager brain / specialist prompts as trusted-looking text.

Mitigating context (why this is a quarantine, not a rewrite): sends/spends are already
structurally gated (Pillar-7 approval, assert_agent_tools_safe, business-impact rails) — an
injection cannot directly move money. The exposure is: fabricated owner-facing claims, poisoned
drafts/recommendations, consent/context manipulation, and prompt-leak.

## Scope (PLAN-FIRST — security-sensitive; adversarial-verify against injected payloads)

1. Inventory every seam where ingested/external text enters an LLM prompt (ingestion commit →
   business data; conversation context builder; campaign-response handling; L4/context retrieval).
2. Quarantine layer at those seams: delimit external content as untrusted data (structural
   fencing), strip/neutralize instruction-like patterns, length-cap cells, never interpolate
   external text into system-role content.
3. Canary: adversarial fixtures — sheet cell + customer reply carrying injection payloads —
   prove the brain treats them as data (no tool-call bait, no instruction-following, no leak).
4. Adversarial verify (opus) against the payload corpus before ship (Rule #15 canary applies).

## Sequencing (Fazal)

START NOW — parallelize alongside the trust-floor tail (disjoint files: ingestion/quarantine vs
manager brain); plan-first gate before build. Pre-launch MUST.

## PR-2 plan (2026-07-15 — seam inventory via 3 read-only agents; fence pattern = PR-1's)

Fence pattern to REUSE (never fork): `orchestrator/security/prompt_quarantine.py` —
`fence(text, *, source, max_len)` wraps in `<untrusted source="...">…</untrusted>`, `neutralize`
collapses spoofed fence tags → `[tag]` + strips control chars (cap length BEFORE neutralize),
`FRAMING` preamble rendered ONCE per prompt/bundle. Generic `test_prompt_quarantine.py` already
covers fence/neutralize; each seam adds a rendering test (poisoned value → FRAMING once + tag
present + payload's own "untrusted" text doesn't survive between real tags).

**Architecture fact that bounds scope:** inbound WhatsApp splits into TWO disjoint pipelines —
owner→`dispatch_brain` (LLM) and customer→`customer_inbound.py` which is DETERMINISTIC / zero-LLM
(keyword match). So NO attacker-writable customer-reply text reaches an LLM today (deferred to
VT-299); bake the fence into that seam when VT-299 lands, not now.

## PR-2 LANDED (2026-07-15) — Tier A fenced, adversarially hardened

Built via ultracode workflow (serial Sonnet-5 build per seam → parallel Sonnet-5 adversarial
skeptics → canary). All 4 Tier-A seams fenced + ingestion caps + rendering tests:
- A1 tech_lane.read_listing_health: scraped name/category fenced (source='scraped_listing'),
  FRAMING added once to the Manager always-on system prompt (orchestrator_agent.py, dup-guarded).
- A2 apify_food._default_themes: each Zomato review fenced (source='review_text') + FRAMING once;
  ingestion cap _REVIEW_TEXT_MAX_LEN=1000.
- A3 entity_resolution._default_adjudicate: GBP candidate fields fenced (source='gbp_candidate') +
  FRAMING once; ingestion cap in auto_discovery_sources (_GBP_CANDIDATE_FIELD_MAX_LEN=200).
- A4 sales_recovery_executor._build_draft_prompt: customer name/business fenced; RULES updated to
  echo the text INSIDE the tag (validation exact-key contract preserved).

**Adversarial-verify (Rule #15) found + FIXED 3 real gaps in the shared primitive:**
- A1 non-str bypass: _fence_scraped_listing_field only fenced str → a non-str attrs value rendered
  raw. Now str()-coerces any non-None value.
- neutralize() split-token bypass: _FENCE_BREAK_RE only matched CONTIGUOUS 'untrusted'; an
  interposed char ('</untru\\nsted>', zero-width, bidi) defeated it (and \\n is preserved). Now
  strips zero-width/bidi FIRST + tolerates whitespace between every letter of 'untrusted'.
- _CONTROL_RE missed \\x0d (CR) — now stripped.
Regression tests added for all three. Full seam+security suite: 71 passed. Canary: 4 passed.
The fence BOUNDARY never structurally broke in any attempt (real fence() tag always wraps).
