## GATE DEFINITION (Package G — pinned 2026-07-06 after the adversarial gate pre-mortem)
Full remediation of the 9 false-proof holes: `.viabe/sprint/vt611-gate-remediation-plan.md`. The pre-mortem
(5 lenses, code-verified) found the gate as originally specced would let a fully-GREEN run be a FALSE proof
(judge mean≥4.5 unimplemented; judge ground-truth-blind; delegation+real-send unassertable; long-haul
retention untested; shadow leg silently zero-rows; XFAIL rides as green). These are the STANDING pass criteria
every enforce promotion must cite:

- **Judge:** every scenario `min dim ≥4 AND per-scenario mean ≥4.5` (per-scenario mean, NOT a global average).
  Judge runs BLIND (no harness PASS/FAIL label in the transcript) + is fed a GROUND-TRUTH block (seed counts +
  expected outcome) so a fabricated number can't score high on honesty.
- **Overall:** 120/120 pass, zero tolerance. Flake protocol = a sub-threshold scenario gets EXACTLY ONE re-run;
  clears → log both transcripts+scores as a documented flake; fails re-run → BLOCK. No aggregate "N-of-120."
- **Critical:** ALL flagged-critical scenarios run ×3, each must be 3/3. Count DYNAMICALLY from the critical=true
  flag at runtime (currently 71 after the retrofit — never hardcode; the earlier "58"/"30" figures are stale).
  Cross-run consistency: variance 0 on the grounded count, identical observed route, identical terminal outcome;
  any divergence BLOCKS that scenario independent of judge scores.
- **DB-state (not reply-text) proof:** delegation/routing, effects, and grounded counts asserted from the DB
  (`assert_side_effects`/`assert_agent_ran`/`assert_route`/`assert_grounded_count`), `assert_no_unapproved_effect`
  on by default. Prereq: the owner freeform-send → `owner_message_audit` bypass is closed (H0) so the effect
  ledger is complete.
