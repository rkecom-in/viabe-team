# VT-570 — the conversation brain gets a tool belt

## Gap (from the pipeline audit Fazal reviewed)
The turn brain composes every reply but holds no tools: a code heuristic detects URLs and fires the scrape; the brain waits on plumbing. Vision: "capability of using their LLM brain … and a capability to access tools as and when the brain commands to use them."

## Scope
compose_turn becomes a bounded agentic loop (≤4 tool iterations, hard wall-clock cap, fail-soft to the single-call path then the walker): server-side web_fetch (domain-pinned to the tenant's known/owner-stated hosts — the brain reads a site the moment it decides to) + client tools `refresh_discovery(url)` (fires the durable DBOS refresh so future turns keep the understanding) and `read_journey_history()` (full window + answers + draft provenance beyond the prompt snapshot). Output contract (TurnPlan JSON) unchanged; recorders/promotion gate/send chokepoint untouched. The URL code-heuristic stays as persistence belt-and-braces.
