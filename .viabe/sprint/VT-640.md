## Problem

`multi_field_single_message` flagged `ignored_speech_act` 3/3 (sonnet judge, model-independent —
both judges historically agree). Owner sends 3 facts in one message, then asks "did you get all
that?". The manager's reply was the contradictory "I haven't started on that yet — tell me to go
ahead" — right after it had said "Great, I found your store, tap to connect."

**Root cause (dev diag VT-640-DIAG, captured pre-gate text + classification):** the BRAIN reply
is CORRECT — "Yes, I have it: Probe Hardware, a hardware and tools shop, with store
probe-hardware.myshopify.com. The Shopify connection isn't complete yet — approve it in the
browser using **the link I sent**, then reply 'done'." The **emission gate** wrongly swapped this
perfect answer: the `("i","sent")` completion bigram matched "the link I **sent**", and the
owner-directed exemption (b) requires an explicit "you" in the SAME sentence — "…using the link I
sent, then reply 'done'" has none — so the gate read a truthful *link*-to-owner delivery as a
fabricated *customer* send and replaced the whole message with the `not_started` stall.

(Classification of "did you get all that?" = `other`, confidence 0.98 → it never reaches
`answer_status_query`; a pre-brain net there would not fire. The fix is squarely the gate.)

## Fix

`src/orchestrator/agent/emission_gate.py` — exemption (d) in
`_sentence_has_blocking_completion_claim`: a subject-ful `i`/`ive` "sent" of an owner-FACING
artifact (`_OWNER_ARTIFACT_TOKENS` = link/links/invite/invitation/oauth/approval) with **no
customer reference** is a delivery to the OWNER, not a customer send. A real customer send names
customers (`has_customer_ref`) or uses the "sent to N" trigram — both STILL block. Scoped to the
i/ive bigrams that over-match; campaign/messages bigrams unchanged.

Merge-blocking regressions kept green: "I sent the offer link to 40 customers" (customer ref),
"Sent to 45 customers via the link" (trigram), "I sent it just now" (no artifact) all still block.
