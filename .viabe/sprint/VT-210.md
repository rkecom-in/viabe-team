# VT-210 — Recurring ingestion mechanism

## Why this row exists

Phase 3 of the use-case diagram: after first-time share, agent offers push OR pull options; runs hands-off forever. This row builds both mechanisms; per-connector adapters declare which they support (via VT-205 registry).

## Scope

### Pull (scheduled)

- DBOS scheduled workflow per (tenant, connector) — uses existing VT-3.5 substrate (4 scheduled triggers already in repo)
- Cron expression from `tenant_integration_state.pull_cadence` (default = connector spec default = 9 AM IST daily)
- Each run: connector.pull_full(since=last_sync) → field mapping → dedupe → land canonical rows
- Failures logged as `error_envelope` step; agent escalates to Fazal if 3 consecutive runs fail (e.g., token expired, sheet permissions revoked)

### Push (webhook)

- `apps/team-web/app/api/orchestrator/integrations/[connector]/push/route.ts` — generic push receiver
- Auth: per-connector. Shopify HMAC signature; Sheet Apps Script shared-secret; etc.
- On receive: validate → connector.parse_push_payload → field mapping → dedupe → land canonical row
- Same observability surface: pipeline_runs + pipeline_steps

### Status surface

- Owner sees in Ops Console: per-connector last_sync_at, last_status, rows_ingested_today, next_scheduled_run
- Integration Agent can answer: "is my Sheet connected?" "when did it last sync?" → reads from this same status table

## Acceptance criteria

1. Schedule path: tenant_integration_state.pull_cadence = "0 9 * * *" → DBOS fires at 9 AM IST → connector pulls → rows land
2. Push path: synthetic Shopify webhook (HMAC-signed) → handler validates → row lands within 30s of webhook
3. Failure recovery: token-expired pull → agent dispatches "your Sheet token expired — reconnect here" to owner
4. Status surface accurate: last_sync_at within 1 min of actual sync
5. RKeCom Sheet: append a row → push receives within 5 min → customer appears in DB

## Rule #15 Canary

`apps/team-orchestrator/canaries/vt210_recurring_ingestion.py` — 5 assertions: schedule fires, push receives + validates, failure escalation works, status surface accurate, RKeCom end-to-end Sheet append → DB.
