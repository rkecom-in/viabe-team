# CL-50 — Twilio account reused from Reports

Twilio configuration for viabe-team uses Reports' existing Twilio account (parent account shared). Three env vars added to viabe-team/.env.local on Fazal's machine 2026-05-16:

- TEAM_TWILIO_ACCOUNT_SID (from Reports config)
- TEAM_TWILIO_AUTH_TOKEN (from Reports config)
- TEAM_TWILIO_FROM_NUMBER (Reports' current number, temporary)

TEAM_ prefix enforces Pillar 8 cross-product separation via the no-cross-product-env-vars lint rule scaffolded in VT-17. Sub-account or separate Twilio account NOT created — single Twilio account suffices since: (a) Phase 1 sender is per-number not per-account, (b) DLT registration is per-entity not per-Twilio-account, (c) env vars already enforce separation at the code layer. When Team gets its own approved sender number + name, Fazal will: (1) register new number in same Twilio account, (2) update TEAM_TWILIO_FROM_NUMBER, (3) Reports' number stays unchanged. Estimated effort: 30 minutes config change.

TEAM_TWILIO_FROM_NUMBER added proactively (Option A) rather than deferred (Option B). Reasoning: outbound messaging code in VT-3.7 composer + VT-5.6 send tools needs this env var; defining it now prevents discovery-at-runtime later. Pillar 8 prefers defined env vars to ad-hoc discovery.
