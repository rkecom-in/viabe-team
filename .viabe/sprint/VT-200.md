# VT-200 — Hygiene bundle (3 small fixes from E2E findings)

## Why this row exists

Sprint 1 E2E smoke test surfaced 3 small issues, none blocking but all worth closing:
- Finding 2: Twilio outbound creds absent → status_ping path crashes. Workaround was changing canary body to substantive (routes to brain instead). Need a real fix.
- Cleanup: `twilio_inbound_events` table FK to tenants blocks DSR-purge order (ForeignKeyViolation). Need ON DELETE CASCADE.
- Finding 4: DBOS scheduler emits `purge_workflow_inputs_scheduled is not a registered workflow function` warning every minute. Pre-existing; bookkeeping fix.

Bundled together because all three are small (~1 file each) and architecturally unrelated to anything else.

## Scope

### Fix 1 — `TEAM_TWILIO_MOCK_MODE` env flag

- Add env var `TEAM_TWILIO_MOCK_MODE` (default: `0` / disabled)
- In `twilio_send.py:_client()` — when `TEAM_TWILIO_MOCK_MODE=1`, return a mock client that logs the send (no real API call) instead of raising on missing creds
- Document in `.viabe/secrets/README.md`
- Enables E2E canaries to exercise the status_ping path without real Twilio creds

### Fix 2 — `twilio_inbound_events` ON DELETE CASCADE

- Migration: alter FK on `twilio_inbound_events.tenant_id` to add `ON DELETE CASCADE`
- DSR-purge tests pass without explicit delete-order workaround
- Verify cascade doesn't affect other tables via inadvertent shared FK chains

### Fix 3 — DBOS purge_workflow_inputs_scheduled registration

- Grep for `purge_workflow_inputs_scheduled` — find the DBOS scheduler config referencing it
- Either register the function (if it exists somewhere) OR remove the schedule entry (if it's stale config)
- Verify scheduler boot is warning-clean

## Acceptance criteria

1. `TEAM_TWILIO_MOCK_MODE=1` allows status_ping path to complete without Twilio creds (canary verifies)
2. DSR-purge on a test tenant cascades cleanly through `twilio_inbound_events`
3. Orchestrator boot logs show no `DBOSWorkflowFunctionNotFoundError` for `purge_workflow_inputs_scheduled`
4. No regression on existing twilio_ingress / DSR-purge / DBOS scheduler tests

## Rule #15 Canary (mandatory)

`apps/team-orchestrator/canaries/vt200_hygiene_bundle.py` — 3 assertions, one per fix:
1. With `TEAM_TWILIO_MOCK_MODE=1`, synthetic webhook routing to status_ping completes (no crash)
2. DSR-purge of synthetic tenant removes associated twilio_inbound_events rows (cascade verified)
3. Orchestrator boot stderr does not contain `DBOSWorkflowFunctionNotFoundError`
