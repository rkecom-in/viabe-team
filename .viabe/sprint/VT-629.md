## Arm 1a — dispatch brain has no anti-loop rule (FIXED here, cheap)
The dispatch/handle-directly prompt (`orchestrator_agent_system.md`) had brevity + honesty rules but NO
"always advance / never re-ask" rule. Added one: never re-ask a fact already given, never repeat a prior
message/link, on an impatient repeat give the result / concrete options / best-effort / escalate (never re-send
"I'm on it"), a specialist hand-off reply must carry the RESULT. Cheap prompt lever; expected to lift some
dispatch-side narrow misses. NOT deployed solo — bundling with the structural fixes to measure once.

## Arm 1b — REVISED: it's the "I'm on it" STALL reflex, NOT a recording bug

Karol Bagh, closed Tuesdays" from s0) and DOES record the name correction (uses "Sharma Traders" at s6). The

works; the manager reflexively emits "I'm on it" for things it should just DO/confirm inline. This is the SAME
disease as arm 1a/2, not a separate recording bug. → The VT-629 dispatch prompt rule ("never re-send 'I'm on
it'; carry the result / confirm inline") directly targets it. (Some profile-collection scenarios may still have
a genuine confirm-lock gap — watch happy_path_profile after the prompt fix; don't assume.)
