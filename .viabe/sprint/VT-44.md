## What CoderC must produce
The `send_whatsapp_message` MCP tool. Sends a free-form WhatsApp message via Twilio. Strictly enforces the 24-hour window rule: if `customers.last_inbound_at` is older than 24 hours OR null, the call returns an error envelope and the orchestrator must use `send_whatsapp_template` instead.
1. **Tool implementation** at `apps/team/src/agent/tools/send_whatsapp_message.py`:
	Input schema:
	- `customer_id: UUID` — the recipient. Tool resolves to phone number internally; phone never appears in input.
	- `body: str` — message text. Length 1-4096 chars (WhatsApp limit). The composer (VT-3.7) is responsible for content; this tool does NOT inspect message content for honesty rules — those are enforced upstream.
	- `idempotency_key: str` — caller-supplied key to prevent duplicate sends. Required. Tool stores `(tenant_id, idempotency_key) → message_sid` and returns prior result on duplicate.
	Output schema:
	- `status: Literal['sent', 'window_closed', 'rate_limited', 'error']`
	- `message_sid: str | None` (Twilio's SID; populated on `sent`)
	- `customer_id: UUID`
	- `sent_at: datetime | None`
	- `error_envelope: ErrorEnvelope | None`
2. **24-hour window enforcement**:
	- Before any Twilio call, query `customers.last_inbound_at` for the recipient.
	- If `last_inbound_at IS NULL`: return `status='window_closed'`, `error_envelope.code='no_inbound_history'`, `message='customer has no inbound message history; use template'`.
	- If `last_inbound_at < now() - 24 hours`: return `status='window_closed'`, `error_envelope.code='window_expired'`.
	- If within window: proceed.
3. **Twilio invocation**:
	- Use Twilio Messaging API with `From=TWILIO_WA_FROM`, `To=whatsapp:<phone>`, `Body=<body>`.
	- Phone number resolution: query `customers WHERE id = customer_id AND tenant_id = ctx.tenant_id`. Cross-tenant attempt returns no row → tool returns `unauthorized` error.
	- Persist `campaign_messages` row with `send_status='sent'` + `twilio_message_sid` on success.
4. **Idempotency**:
	- Composite key `(tenant_id, idempotency_key)` indexed in a new `send_idempotency_keys` table (migration 012).
	- On duplicate key: return prior result, do NOT re-send.
	- Idempotency key TTL: 24 hours (after which the same key may be reused).
5. **Rate limiting**:
	- Per-tenant: max 1000 outbound free-form messages per day. Beyond limit: `status='rate_limited'`, `error_envelope.retry_after_ms` populated.
	- Per-customer: max 1 outbound free-form per 6 hours (anti-spam). Beyond: `rate_limited`.
6. **Tests** at `apps/team/tests/agent/tools/test_send_whatsapp_message.py`:
	- Happy path: customer with `last_inbound_at` 2 hours ago → message sends → Twilio SID returned.
	- Window-closed: `last_inbound_at` 25 hours ago → `status='window_closed'`.
	- No-inbound-history: customer never messaged → `status='window_closed'` with correct error code.
	- Idempotency: same `idempotency_key` twice → second call returns prior result, Twilio not invoked.
	- Rate limit per-tenant: 1001st call in 24h → `rate_limited`.
	- Rate limit per-customer: second call within 6h → `rate_limited`.
	- Cross-tenant: customer_id from tenant_B with ctx.tenant_id=tenant_A → `unauthorized`.
	- Twilio API error: simulated 5xx → retried via framework backoff; structured error if exhausted.

## Architectural rules to enforce in this subtask
- Pillar 2: tool is deterministic transport. No LLM, no content inspection. Content honesty enforced upstream by composer (VT-3.7).
- Pillar 3: phone resolution via tenant-scoped query. Cross-tenant attempts return `unauthorized`, never leak phone.
- Pillar 7: the 24-hour window enforcement is honest about WhatsApp's real constraint. The tool never silently substitutes template send — it returns `window_closed` and lets the composer decide.
- Idempotency keys are mandatory inputs. CodeX rejects PRs with optional idempotency.
- The tool does NOT call `send_whatsapp_template` as a fallback when window is closed. That's the composer's decision — the tool reports the constraint, doesn't work around it.
- Phone numbers never appear in `pipeline_log` payload. Logged values: `customer_id`, `tenant_id`, `message_sid`, `status`. PII redaction enforced.
