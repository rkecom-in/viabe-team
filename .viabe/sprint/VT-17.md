# VT-17 — Repo and monorepo layout

## Why
First piece of VT-Foundation. Every later parent (Orchestrator, Agent, Tools, Ingestion, Knowledge, Owner Surface, Billing) sits on top of the apps/ + packages/ + pnpm-workspace.yaml structure. Reports product had no such structure — apps and packages were tangled — which made later refactors painful. Team enforces tier separation from day one. Per CL-78 the original spec named `apps/team`; that was pivoted to `apps/team-orchestrator` to reserve `team` namespace.

## What
- `pnpm-workspace.yaml` declares `apps/team-web`, `apps/team-orchestrator`, `apps/team-ingestion-worker`, `packages/team-shared` as the workspace members.
- Each app has its own `package.json` (or `pyproject.toml` for Python apps) with project-scoped deps.
- `packages/team-shared` carries cross-app types (TypeScript) + the migrations SQL set consumed by both web + orchestrator.
- `tsconfig.base.json` at repo root is the shared TS config; each app extends.
- No cross-product imports between Reports product and Team product; CI lint (`scripts/lint-cross-product-env.mjs`) enforces.

## Acceptance criteria
- ✓ `pnpm-workspace.yaml` lists the four workspace members.
- ✓ `pnpm install --frozen-lockfile` succeeds at repo root.
- ✓ All three apps build independently (`pnpm --filter @viabe/team-web build`, etc.).
- ✓ `packages/team-shared` is consumable by `apps/team-web` via TS path aliases.
- ✓ Cross-product import lint passes on green branches.
