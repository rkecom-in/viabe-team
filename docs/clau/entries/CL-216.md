# CL-216 — test_dbos_step_resume.py timeout=60 fragile against pooler

=== DEBT ITEM ===
Landmine 2 test (test_dbos_step_resume.py) uses _wait_for_probe(timeout=60) when polling the probe table for node_b row insertion. On Supabase Session Pooler, end-to-end test run takes ~75 seconds (DBOS init in main process + DBOS init in worker subprocess + langgraph init + actual graph execution). The probe wait portion is ~20s when the test runs cleanly, well under 60s — but only because most of the 75s is consumed by setup that happens BEFORE the polling window starts.

If pooler latency increases (peak load, region degradation, schema changes adding migration overhead), the timeout=60 window can be exhausted before node_b's probe row appears. The original run with timeout=60 against the pooler timed out at 60s before the SIGKILL ever fired — demonstrated fragility.

=== CONTEXT ===
Fazal-extended the timeout to 180 for the successful Landmine 2 run, then reverted to 60 for the commit per explicit call (2026-05-19). The revert is an accepted-debt decision, not an oversight — Fazal's intent is that Landmine 2 will be re-run on local Docker PG going forward, not against the pooler. Local Docker PG removes the latency tax (DBOS init ~5s instead of ~16s), making 60s ample.

=== KNOWN FRAGILITY ===
Any future run of Landmine 2 against pooled PG (Supabase Session Pooler or similar) will likely time out at 60. The test as committed is implicitly local-PG-only for reliable runs. This is acceptable because:
  (a) Landmine 2 verdict is already captured (CL-214 BORING).
  (b) Future re-runs are diagnostic / regression — local PG is the right env for those.
  (c) CI does not run this test (gated on DATABASE_URL absence — CI environment has no PG fixture).

=== REPAYMENT TRIGGERS ===
This debt repays itself if:
  - We add a CI job that runs landmine-class tests against a provisioned PG.
  - Landmine 2 needs re-run against the pooler for production-environment verification. Bump timeout to 180 temporarily; do not re-commit unless we want pooler-tolerance permanently.
  - We extend the test to cover additional crash windows or composition surfaces and the polling logic gets factored. Add a configurable timeout via env var at that point.

=== STANDING IMPLICATION ===
None. Per-test timeout values are local concerns. The general principle (tests that depend on external infra timing should make the timeout configurable or generous) is already known; no new rule.

=== STATE AT TIME OF WRITE ===
test_dbos_step_resume.py committed with timeout=60. Fragile against pooled PG. Test passes against local Docker PG with margin.
