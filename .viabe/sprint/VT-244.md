# VT-244 — fix deploy-dev Vercel deploy (outdated CLI)

## Why
`deploy-dev.yml` runs on every push to `main` (post-merge). Both deploy
jobs were failing:

1. **Vercel (team-web)** — `amondnet/vercel-action@v25` pins the Vercel
   CLI to `25.1.0`. Vercel's API now rejects it: *"Your Vercel CLI
   version is outdated. This endpoint requires version 47.2.2 or later."*
   The action is unmaintained. **This row fixes it.**
2. **Railway (orchestrator)** — `Invalid RAILWAY_TOKEN`. The
   `RAILWAY_TOKEN` GitHub secret is expired/invalid. **NOT a code fix —
   Fazal must rotate the token in the Railway dashboard + update the
   GitHub secret.** Tracked separately; out of scope for this row.

## Change
Replace the `amondnet/vercel-action@v25` step with the current official
Vercel CLI flow:
- `npm i -g vercel@latest`
- `vercel pull --yes --environment=production`
- `vercel build --prod`
- `vercel deploy --prebuilt --prod`

Token passed via `env: VERCEL_TOKEN` (not `${{ }}` interpolation in the
run block) — GitHub Actions injection-safe. `VERCEL_ORG_ID` /
`VERCEL_PROJECT_ID` read by the CLI from the environment.

## Acceptance
- PR CI green (standard required checks).
- After merge, the deploy-dev **Vercel** job succeeds (no CLI-version
  error). Full deploy-dev green additionally requires the Railway token
  rotation (Fazal action — separate).

## Out of scope
- Railway token rotation (Fazal / secret rotation).
- `.next-id` reconcile (Cowork; VT-244 consumed from the allocator).
